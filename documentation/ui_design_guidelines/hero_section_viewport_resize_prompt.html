Layout Structure (Diagram)

The hero is split into four areas as in the Excalidraw diagram:

  • Section 1 (top left): Logo / vertical “MALAYALEES.US” style content; Kerala/backwaters image. Must fit without cutoff.
  • Section 2 (top right): Regular slideshow; do not break existing slideshow behavior.
  • Section 3 (bottom left): Unite India logo + “About Our Mission”; logo must fit inside the area without cutoff.
  • Section 4 (bottom right): Donate button and “Support Our Community”; make prominent.

Top row = Sections 1 & 2 side by side (desktop); bottom row = Sections 3 & 4 side by side. All four plus the “Browse all upcoming events” strip must fit within the intended viewport height.

Goals

  1. Reduce overall hero height by at least 150px (or more) so the entire hero fits within the viewport on typical laptops and smaller screens.
  2. Contain the full hero within the viewport so users do not need to scroll to see the bottom of the hero (bottom row + browse link). Currently the hero extends beyond the display and requires scrolling.
  3. Optionally reduce the height of sections 3 and 4 by about 20px (e.g. lower min-height / max-height of .hero-bottom-card and/or .hero-bottom-row flex share).
  4. Ensure all images in sections 1–4 are contained within their allocated areas and fill those areas appropriately (no unintended cropping; follow image containment rule).
  5. Maintain mobile responsiveness: sections stack properly (e.g. 1, then 2, then 3, then 4) and remain readable and tappable.
  6. Apply frontend design polish (spacing, proportions, visual hierarchy) so the resized hero still looks intentional and professional.

Height & Viewport Requirements

  • Overall hero container: Reduce total height by 150px or more. Consider using a max-height such as calc(100vh - 150px) or similar (account for header/nav height) so the hero never exceeds the visible viewport.
  • Top row (Sections 1 & 2): Currently flex: 0 0 80% of the hero. Reduce this percentage or the hero’s total height so the top row plus bottom row plus browse strip fit in one viewport.
  • Bottom row (Sections 3 & 4): Currently flex: 0 0 18%. Optionally reduce the height of the bottom cards by about 20px (e.g. .hero-bottom-card min-height / max-height).
  • Browse strip: Keep compact (e.g. current flex: 0 0 2% or equivalent) so it doesn’t push content off-screen.

Target: On a typical laptop/smaller screen, the entire hero (left panel, right panel, bottom two cards, and “Browse all upcoming events”) should be visible without vertical scrolling. Only content below the hero should require scrolling.

Image Containment

Follow the rule in .cursor/rules/image_containment_prevention_hero_images_cards.mdc:

  • Containers: Use flexible height where appropriate (h-auto or constrained by parent flex) so images can maintain aspect ratio. Avoid fixed heights that force cropping.
  • Images: Use object-contain for hero/banner/logo images that must be fully visible (Sections 1, 2, and 3). Ensure images fill their allocated section area (e.g. w-full h-full with object-contain and centered object-position) so the section looks filled without cropping edges.
  • Section 3 (Unite India): Diagram states “logo should fit inside this area without cutoff.” Ensure the Unite India image is contained and fills the card area (e.g. object-contain or controlled object-cover with safe object-position so the logo is never cut off).

Reference: Image containment rule specifies object-contain, w-full h-auto (or equivalent within a flex layout), and no cropping for hero/banner/logo imagery.

Mobile Responsiveness

  • On small screens, sections should stack vertically in a logical order: e.g. Section 1 (logo/Kerala image), then Section 2 (slideshow), then Section 3 (Unite India / Mission), then Section 4 (Donate), then the browse link.
  • Preserve existing responsive breakpoints (e.g. @media (min-width: 768px) for row layout) and adjust any new height values so mobile view also fits within the viewport where possible, or scrolls minimally.
  • Touch targets (Donate button, Mission card, slideshow controls) must remain large enough and properly spaced.
