1. Database Schema Changes

1.1 Add net_payout_amount Column

New Column: net_payout_amount numeric(21,2) NULL

Purpose: Store the net amount received in bank account after Stripe fees and tax are deducted.

Calculation: net_payout_amount = final_amount - stripe_fee_amount - stripe_amount_tax

4. Add Net Revenue Before Tax Calculation

4.1 Overview

Net Revenue Before Tax Calculation (Option A: Calculate On-the-Fly)

Formula: net_revenue_before_tax = final_amount - stripe_fee_amount

Alternative Formula: net_revenue_before_tax = net_payout_amount + stripe_amount_tax

Approach: Calculate in analytics service layer (no database changes needed)

Rationale: Simple calculation using existing fields, no need to store in database unless performance becomes an issue.

5. Fix Purchase Flow Calculation Logic

5.1 Problem Statement

Issue: During ticket purchase, totalAmount and finalAmount are being set to the same value.

Expected Calculation:

  • totalAmount = Sum of base ticket prices (before discounts, fees, taxes)
  • finalAmount = totalAmount - discountAmount + platformFeeAmount + taxAmount

// ✅ CORRECT Calculation Logic (to implement in service) public EventTicketTransactionDTO createTransaction(EventTicketTransactionDTO dto) { // Step 1: Calculate totalAmount (sum of base ticket prices) BigDecimal totalAmount = BigDecimal.ZERO; for (TicketItem item : dto.getTicketItems()) { BigDecimal itemTotal = item.getPricePerUnit() .multiply(BigDecimal.valueOf(item.getQuantity())); totalAmount = totalAmount.add(itemTotal); } // Step 2: Calculate discountAmount (if discount code applied) BigDecimal discountAmount = BigDecimal.ZERO; if (dto.getDiscountCodeId() != null) { DiscountCode discountCode = discountCodeRepository.findById(dto.getDiscountCodeId()) .orElseThrow(); if ("PERCENTAGE".equals(discountCode.getDiscountType())) { discountAmount = totalAmount .multiply(discountCode.getDiscountValue()) .divide(BigDecimal.valueOf(100), 2, RoundingMode.HALF_UP); } else if ("FIXED".equals(discountCode.getDiscountType())) { discountAmount = discountCode.getDiscountValue(); } // Ensure discount doesn't exceed total amount if (discountAmount.compareTo(totalAmount) > 0) { discountAmount = totalAmount; } } // Step 3: Calculate platformFeeAmount (from tenant settings) TenantSettings tenantSettings = tenantSettingsRepository.findByTenantId(dto.getTenantId()) .orElse(null); BigDecimal platformFeePercentage = tenantSettings != null ? tenantSettings.getPlatformFeePercentage() : BigDecimal.ZERO; BigDecimal amountAfterDiscount = totalAmount.subtract(discountAmount); BigDecimal platformFeeAmount = amountAfterDiscount .multiply(platformFeePercentage) .divide(BigDecimal.valueOf(100), 2, RoundingMode.HALF_UP); // Step 4: Calculate taxAmount (if tax is applicable) // Note: If using Stripe Tax, tax is included in Stripe amount, not calculated separately BigDecimal taxAmount = BigDecimal.ZERO; // TODO: Add tax calculation logic if manual tax is used // Step 5: Calculate finalAmount (what customer pays) // finalAmount = totalAmount - discountAmount + platformFeeAmount + taxAmount BigDecimal finalAmount = totalAmount .subtract(discountAmount) .add(platformFeeAmount) .add(taxAmount); // Step 6: Set all calculated values dto.setTotalAmount(totalAmount.setScale(2, RoundingMode.HALF_UP)); dto.setDiscountAmount(discountAmount.setScale(2, RoundingMode.HALF_UP)); dto.setPlatformFeeAmount(platformFeeAmount.setScale(2, RoundingMode.HALF_UP)); dto.setTaxAmount(taxAmount.setScale(2, RoundingMode.HALF_UP)); dto.setFinalAmount(finalAmount.setScale(2, RoundingMode.HALF_UP)); // Initialize Stripe-related fields (will be populated by batch job) dto.setStripeFeeAmount(BigDecimal.ZERO); dto.setStripeAmountTax(null); dto.setNetPayoutAmount(null); // Will be calculated by batch job // ... save transaction ... return savedDto; }

⚠️ Important Notes:

  • Stripe Amount Verification: The finalAmount should match the amount charged by Stripe (from PaymentIntent or CheckoutSession)
  • Tax Handling: If Stripe Tax is enabled, tax is included in the Stripe amount. Don't add taxAmount again to finalAmount.
  • Platform Fee: Platform fee is typically added on top of the ticket price (not deducted from it)
  • Precision: Always use setScale(2, RoundingMode.HALF_UP) for monetary values
