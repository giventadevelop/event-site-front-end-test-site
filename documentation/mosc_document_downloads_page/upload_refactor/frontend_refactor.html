Official Document Upload Refactor - Frontend

This document describes frontend and script updates for hierarchy-aware upload and paginated tree display.

1) Implemented in Next.js app

  • Added explicit DTO fields in src/types/index.ts: hierarchyPath, hierarchyCategoryLabel, displayPriority.
  • Public tree API implemented at src/pages/api/public/official-documents-tree.ts, with priority-first ordering and pagination response.
  • Downloads redesign page now supports:
    • hierarchical tree rendering
    • category filters
    • page navigation
    • priority-first display
    Files: src/app/mosc-redesign/(syro)/downloads/page.tsx, DownloadsPageClient.tsx, ApiServerActions.ts.

2) Auto-upload script refactor

Script: scripts/mosc-in-migration/upload-mirror-to-official-docs.mjs

  • Reads mirror report and converts filesystem structure into hierarchy metadata.
  • Uploads each file via tenant official-document API.
  • Patches uploaded records with: hierarchyPath, hierarchyCategoryLabel, displayPriority, priorityRanking.
  • Keeps legacy description markers for backward compatibility.

3) NPM commands

npm run mosc:migration:upload:mirror:dry
npm run mosc:migration:upload:mirror

4) Data compatibility strategy

Rendering code uses this priority:

  1. Explicit DB fields (hierarchyPath, hierarchyCategoryLabel, displayPriority)
  2. Fallback to description markers for old data
This means no data loss during migration: old rows still render correctly until DB/API rollout completes.

5) Required backend alignment

  • Persist new columns from request payload in backend entity/DTO mapping.
  • Expose new fields in response JSON for list/detail endpoints.
  • Keep old marker-based data untouched until fully migrated.

6) Verification checklist

  • Upload dry run completes without failures.
  • Real upload inserts rows and patches hierarchy columns.
  • /mosc-redesign/downloads renders tree and paginates by priority.
  • Category filters keep hierarchy structure within selected category.

Result: hierarchy + priority are now first-class fields with backward compatibility.
