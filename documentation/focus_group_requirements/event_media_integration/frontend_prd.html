PRD: Focus Group – Event Media Integration (Frontend)

Project: mosc-temp (Next.js 15+ App Router)
Document Version: 1.0
Date: March 2026
Status: Draft

Requirement

When uploading or editing event media in the admin event media flow, allow optionally associating a focus group for that event. Where relevant (e.g. event detail page or admin event media list), display event focus groups and allow filtering or grouping media by focus group. Implementation is to be done by the frontend team when ready; this PRD is the handoff specification.

API / Proxy

  • Use existing proxy routes: /api/proxy/event-medias and /api/proxy/event-focus-groups (or equivalent).
  • Follow nextjs_api_routes.mdc: all authenticated backend calls must be made from server components or server actions (e.g. ApiServerActions.ts); use fetchWithJwtRetry from @/lib/proxyHandler for backend calls. Do not call the backend API URL directly from client components.
  • Do not add tenantId.equals when calling proxy endpoints; the proxy injects it.

DTOs & Types

Extend the event-media type (in @/types or equivalent) to include optional eventFocusGroupId (or event_focus_group_id) once the backend exposes it. Align with the backend API schema and documentation/Swagger_API_Docs/api-docs.json when the EventMedia schema is updated.

UI Standards

  • Follow ui_style_guide.mdc: page container max-w-5xl mx-auto px-8 py-8, content cards bg-white rounded-lg shadow-md p-6, and existing form/input patterns.
  • On media upload/edit: add an optional focus group selector (dropdown or multi-select) populated with the event’s focus groups from event_focus_groups (filter by eventId.equals).
  • Event detail and admin media list: optionally show focus group label per media and allow filtering by eventFocusGroupId.equals when querying event-medias.
Scope: No new components are built in the initial schema/docs phase. This PRD is for the frontend team to implement when the backend has exposed eventFocusGroupId on event media.
