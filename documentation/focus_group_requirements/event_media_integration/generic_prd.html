PRD: Focus Group – Event Details & Event Media Integration (Generic)

Project: MCEFEE Multi-Tenant Event Management Platform
Document Version: 1.0
Date: March 2026
Status: Draft

Executive Summary

This document defines the integration of event_details, event_media, and event_focus_groups so that (1) events can be associated with one or more focus groups (already supported via the event_focus_groups join table), and (2) event media files can optionally be associated with a focus group when uploaded. The change is minimal: a single optional column on event_media referencing event_focus_groups. No changes are required to event_details or event_focus_groups table definitions.

Key Concept: Events are already linked to focus groups via event_focus_groups (event_id, focus_group_id). This PRD adds an optional event_focus_group_id on event_media so that when media is uploaded for an event, it can be tagged with one of that event’s focus groups. When the event–focus group association is removed, media is unlinked (ON DELETE SET NULL).

1. Current Tables

Table Description Key Columns
event_details Enhanced event details (title, dates, location, etc.). id, tenant_id, title, start_date, end_date, event_type_id, created_at, updated_at
event_focus_groups Join table mapping events to one or more focus groups. id, tenant_id, event_id (FK event_details), focus_group_id (FK focus_group), created_at, updated_at. UNIQUE(tenant_id, event_id, focus_group_id)
event_media Media files (images, documents) for events or albums. id, tenant_id, title, event_media_type, event_id, album_id, sponsor_id, uploaded_by_id, created_at, updated_at. New: event_focus_group_id (optional FK to event_focus_groups)

2. Relationship Diagram

event_details  ||--o{ event_focus_groups : "has"
focus_group    ||--o{ event_focus_groups : "used by"
event_details  ||--o{ event_media        : "has"
event_focus_groups ||--o{ event_media    : "optional tag"

event_details has many event_focus_groups (each row links one event to one focus group). event_media belongs to an event (event_id) and may optionally reference one event_focus_groups row (event_focus_group_id) to tag the media with that event’s focus group.

3. Change Summary

  • event_details: No change.
  • event_focus_groups: No change.
  • event_media:
    • New column: event_focus_group_id bigint NULL
    • FK: REFERENCES public.event_focus_groups(id) ON DELETE SET NULL
    • New index: idx_event_media_event_focus_group_id (partial, WHERE event_focus_group_id IS NOT NULL)

4. Backward Compatibility

The new column is nullable. Existing rows and all existing queries remain valid. No change to sequence_generator or other tables. If the backend exposes the field in REST/DTOs as optional, existing API clients are unchanged.
