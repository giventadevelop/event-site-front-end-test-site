Event Registration Enhancements - Generic Requirements

This document defines the cross-team functional requirements for event registration notes and attendee attachment uploads.

Scope

  • Add attendee notes with a max length of 1500 characters.
  • Allow uploading up to 2 attachments during event registration.
  • Render notes and attachments on the registration success page.
  • Use a dedicated DB table for attachments: event_attendee_attachment.
  • Store files in S3 using attendee-specific paths.

Business Rules

  • Registration remains valid even when no attachments are provided.
  • Attachments allowed: PDF, DOC/DOCX, XLS/XLSX, CSV, TXT, JPG/JPEG/PNG.
  • Maximum file count: 2 per registration request.
  • Notes must be validated client-side and server-side at 1500 chars.
  • Upload sequence: attendee record first, then attachment uploads tied to attendee ID.

High-Level Flow

  1. User submits registration form.
  2. System creates attendee record.
  3. System uploads up to two files and persists attachment metadata rows.
  4. Success page displays attendee details, notes, and attachment links.

S3 Path Convention

Required generic path format:

{profile}/events/tenantId/{tenantId}/event-id/{eventId}/attendees/attendee-id/{attendeeId}/attachments/{base}_{timestamp}_{uuid}.{ext}

Acceptance Criteria

  • Attendee notes accept up to 1500 characters and persist correctly.
  • More than two files is rejected by validation.
  • Unsupported file types are rejected by validation.
  • Upload errors return clear UI feedback; registration error states are visible.
  • Success page shows notes and uploaded attachments when present.

Ownership

Product/PM: Confirm business rules and limits.

QA: Validate all acceptance criteria and negative upload scenarios.
