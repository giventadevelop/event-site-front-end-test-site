Malankara Association Downloads — Design & Migration PRD

Layer: Data migration + public downloads hub  |  Version: 1.0  |  Date: June 27, 2026
Gap analysis: gap_analysis_prd.html  |  Upload runbook: official_documents_upload_runbook.html  |  Public route: /mosc-redesign/downloads  |  Legacy reference: mosc.in Malankara preliminary list

Recommendation: Upload Malankara Association PDFs (including the preliminary candidate list) into the existing PostgreSQL official documents system (event_media + official_document_category). Use the unified downloads hub with category and year filters — not Strapi, not a dual-source merge. Legacy “different attributes” are HTML presentation (section headers, link colors), not a separate data model.

Product decision (confirmed): Primary UX is the unified /mosc-redesign/downloads hub filtered by category (e.g. Malankara Association 2022) and year — not separate legacy-style landing pages.

1. Problem statement

The downloads page at http://localhost:3000/mosc-redesign/downloads is populated from PostgreSQL via the Rust backend (GET /api/event-medias/public-official-documents). Some content was migrated from mosc.in/downloads/ using migration scripts under scripts/mosc-in-migration/.

Malankara Association content — including the preliminary list page and the richer Malankara Association 2022 landing page — appears missing or mis-categorized because:

  • The scrape manifest assigns all 288 files to generic mosc-in-downloads, not Malankara-specific categories.
  • PDF rows for key files (e.g. List-of-Candidates.pdf) are not present in checked SQL seed files.
  • Static download cards still use # placeholders for Malankara links (downloadsStaticItems.ts).

8. Migration runbook (Windows)

Prerequisites: Next.js app + Rust API running locally; NEXT_PUBLIC_TENANT_ID matches target tenant; categories seeded in DB; respect MOSC copyright and terms when re-fetching from mosc.in.

Environment variables
  • MOSC_DOWNLOAD_ROOT — local folder for fetched PDFs (default C:\E_Drive\code_backup\mosc_downloads)
  • NEXT_PUBLIC_TENANT_ID / MOSC_TENANT_ID — e.g. mosc_malankara_orthodox_2
  • MOSC_APP_BASE_URL — e.g. http://localhost:3000 (for upload proxy calls)
  • MOSC_AUTO_CREATE_CATEGORIES=true — if any category slug is missing
  • MOSC_MIRROR_REPORT — path to mirror report JSON for upload-mirror-to-official-docs.mjs

8.1 Regenerate URL manifest (optional refresh)

Deep scrape → url-list.mosc-in.generated.json
cd F:\project_workspace\mosc-temp && node scripts/mosc-in-migration/_scrape-downloads-deep.mjs --write-url-list
Windows single-line:
cd F:\project_workspace\mosc-temp && node scripts/mosc-in-migration/_scrape-downloads-deep.mjs --write-url-list
After regenerate, patch categorySlug on Malankara-related items before fetch (see §7.1). The scrape script alone will reset all items to mosc-in-downloads.

8.2 Fetch PDFs to disk (manifest-driven)

Fetch URLs from manifest
set MOSC_DOWNLOAD_ROOT=C:\E_Drive\code_backup\mosc_downloads && cd F:\project_workspace\mosc-temp && node scripts/mosc-in-migration/fetch-urls.mjs --manifest scripts/mosc-in-migration/url-list.mosc-in.generated.json --skip-existing
Windows single-line:
set MOSC_DOWNLOAD_ROOT=C:\E_Drive\code_backup\mosc_downloads && cd F:\project_workspace\mosc-temp && node scripts/mosc-in-migration/fetch-urls.mjs --manifest scripts/mosc-in-migration/url-list.mosc-in.generated.json --skip-existing

Files land under {MOSC_DOWNLOAD_ROOT}/{categorySlug}/{year}/{filename}. For Malankara 2022 items, use categorySlug: malankara-association-2022 and year: 2022 in the manifest entry.

8.3 Upload Malankara manifest (direct backend API — recommended)

Full procedure: See official_documents_upload_runbook.html for architecture, localhost:8080 JWT auth, thumbnail backfill, and reuse for other document sets.

Upload missing + thumbnails (Malankara repair)
set MOSC_DOWNLOAD_ROOT=f:\project_workspace\mosc-temp\.mosc-downloads && set MOSC_UPLOAD_LIMIT=0 && cd F:\project_workspace\mosc-temp && npm run mosc:migration:malankara:upload:missing

10. Strapi appendix — when to use / not use

Use Strapi for

  • Kalpana CMS — kalpana-editions, kalpana-documents via src/app/mosc-redesign/(syro)/kalpana-cms/
  • Editorial content: news, saints, institutions, publications, ecumenical articles
  • Structured entries with images, ordering, tenant-scoped slugs

Do not use Strapi for

  • Malankara Association official PDFs (use PostgreSQL official documents)
  • Duplicating the downloads hub file library
  • Merging with PostgreSQL in one paginated grid (not implemented)
