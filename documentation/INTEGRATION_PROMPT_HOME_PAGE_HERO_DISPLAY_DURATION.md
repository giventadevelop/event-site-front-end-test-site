# Integration Prompt: `home_page_hero_display_duration_seconds`

Use this prompt to integrate the new **home page hero display duration** field in the **backend** (malayalees-us-site-boot) and **batch job** (event-site-manager-batch-jobs) projects. The frontend (mosc-temp) already has the field in `EventMediaDTO` and related docs.

- **Backend project**: `E:\project_workspace\malayalees-us-site-boot` (Java/Spring Boot).
- **Batch job project**: `E:\project_workspace\event-site-manager-batch-jobs` (Java).

**New DB column**: `event_media.home_page_hero_display_duration_seconds`  
**Type**: `int4` (PostgreSQL INTEGER), **nullable**, **default** `NULL`.  
**Meaning**: Duration in **seconds** to show this image in the homepage hero slider when `is_home_page_hero_image` is true. Examples: 50 secs → `50`, 1 min 20 secs → `80`. When `NULL`, the app uses 8 seconds.  
**Constraint**: `1 <= value <= 600` when not null (1 second to 10 minutes).

**Migration SQL**: `code_html_template/SQLS/migration_add_home_page_hero_display_duration.sql` (in mosc-temp).  
**Full spec**: `documentation/HOME_PAGE_HERO_DISPLAY_DURATION.md` (in mosc-temp).
