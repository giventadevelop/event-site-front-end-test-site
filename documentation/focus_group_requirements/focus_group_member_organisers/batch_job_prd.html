Requirement

Focus group member join and executive/organising committee list are handled by the application backend and frontend. Batch jobs do not create or update focus_group_members for join or leave. This PRD documents the data model for any batch job that may read focus groups or members (e.g. notifications, reports).

Schema / Tables

No new tables or columns. Use existing focus_group and focus_group_members. The role column allows values MEMBER, LEAD, ADMIN, EXECUTIVE, ORGANISER (EXECUTIVE and ORGANISER = executive/organising committee). status: PENDING, ACTIVE, INACTIVE. If the batch project keeps a copy of the canonical schema, ensure it includes the COMMENTs on focus_group_members.role and focus_group_members.status from code_html_template/SQLS/Current_Sqls/Latest_Schema_Post__Blob_Claude_12.sql.

Jobs

If any job reads focus_group or focus_group_members (e.g. to send notifications to committee members or to report on membership), use the existing tables and role values. No changes to job logic are required for this feature unless a new use case (e.g. “email organisers when a user joins”) is added later.

No changes required: For the current scope (end-user join and committee list), no batch job changes are required. This PRD is for reference if batch jobs later need to read focus group or member data.
