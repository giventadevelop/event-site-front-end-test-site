Executive summary

Problem and solution

The v2.0 implementation delivers admin catalog CRUD, a registration wizard with Stripe checkout, and results entry. Users still experience a flat competitions hub without per-competition detail pages, structured discipline types, or team roster flows. Parents cannot easily register multiple children for specific competitions; team sports (soccer, group dance) lack named rosters despite GROUP type in the schema.

Solution (v2.1): Schema deltas + new APIs for group members and composite team registration; public competition detail routes; wizard branches for parent / self / team captain; admin guardrails (age, grade, discipline, rules); extended podium with configurable max_placements and optional 4th-place points.

Open questions — recommended defaults (v2.1)

Stakeholders may override these defaults; document decisions in event settings or tenant config.

  1. Per-competition fee vs event pass: Keep per-competition fee (current model). Event-wide pass deferred to a future phase.
  2. Stripe at launch: Required when fee_amount > 0; admin manual confirm allowed for $0 competitions only.
  3. Grade categories: Tenant-configurable via category_code + admin templates — not hard-coded four grades.
  4. Group billing: Team captain pays single combined fee (leader registration + roster members linked).
  5. Youth public names: Use initials + category code when results_display_mode = PRIVACY_SAFE.
  6. Max placements: Configurable per competition (max_placements); default 3, allow 4+.
  7. Winner photos: Optional before publish (recommended, not required).
  8. Tickets + competitions: Separate carts; use existing allow_ticket_sales flag — no mixed Stripe cart in v2.1.
  9. MIXED events actor: Wizard shows explicit step: Parent | Self | Team captain when audience_mode = MIXED.

Success criteria

  • Admin creates competitions with discipline, age/grade bounds, rules, and team settings.
  • Public hub lists competitions with links to detail pages showing regulations and register CTA.
  • Parent registers each child per competition; adult self-registers; team captain registers with named roster.
  • Backend rejects ineligible registrations (age, grade, capacity, duplicate, group size).
  • Admin enters and publishes 1st–Nth placements; public winners grouped by competition podium.
