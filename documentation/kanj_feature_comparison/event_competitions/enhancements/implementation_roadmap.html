9. Test checklist

Database & API (E1)

  • Liquibase applies cleanly on empty and populated DB
  • Group member CRUD + unique (registration, participant) constraint
  • Eligibility rejects age/grade out of bounds
  • Team POST creates leader + members atomically
  • Team POST fails when roster < min_group_size
  • Bulk POST creates N registrations for one participant

Public UX (E2–E3)

  • Hub cards navigate to competition detail
  • Detail shows rules_markdown and discipline badge
  • Register deep-link skips catalog when competitionId valid
  • MIXED event shows actor selection step
  • Parent can switch between two child profiles and register each
  • Team captain completes roster within min/max; team name required when flag set
  • Stripe checkout confirms all registrations in metadata
  • My registrations lists per-competition entries with team name

Admin & results (E4)

  • Admin form saves all v2.1 competition fields
  • Results grid blocks placement > max_placements
  • 4th place receives points_fourth when configured
  • Duplicate placement rejected on publish
  • Public winners show podium grouped by competition
  • PRIVACY_SAFE display mode masks youth names

Batch (E5)

  • Team confirmation email includes full roster
  • Winners digest sent for single competition publish
  • CSV export includes group members column

Regression

  • is_competition_event=false events unchanged (ticket flow)
  • Existing individual registration wizard still works
  • Mobile browser proxy calls succeed (middleware ignoredRoutes)
