1. What is Impeccable?

Impeccable is a design-focused agent skill (one SKILL.md + 7 reference docs + 23 sub-commands + a regex anti-pattern linter) that teaches your AI coding harness to design like a senior product designer. It builds on Anthropic’s original frontend-design skill and ships under Apache 2.0. Once installed globally, the skill is available to Cursor IDE in every project on this machine without per-project setup.

The skill includes references for typography, color & contrast, spatial design, motion, interaction, responsive design, UX writing, and a brand-vs-product register. The 23 commands include craft, audit, critique, polish, distill, animate, bolder, quieter, and more.

2. Prerequisites

⚙️ Prerequisites
Requirement Why it matters How to verify / enable
Node.js + npx The skills CLI is invoked via npx; it clones the skill repo and copies files. Run node --version and npx --version in PowerShell. Both should print versions (Node v20+ recommended; this machine has Node v22, npx 11).
Cursor Nightly channel Agent Skills is a Beta feature served on Nightly first. Cursor → Settings → Beta → switch channel to Nightly, then restart Cursor.
Agent Skills enabled Without this toggle, Cursor will not load SKILL.md files from ~/.cursor/skills/. Cursor → Settings → Rules → enable Agent Skills.
PowerShell This guide’s commands are written for PowerShell on Windows (the project’s default shell). Open Cursor terminal — it should default to PowerShell.
💡 Sanity check: If the existing global skill html-documentation-styling already shows up in your Cursor agent context (it lives at C:\Users\<you>\.cursor\skills\html-documentation-styling), then both the Nightly channel and Agent Skills toggle are already in place — Impeccable will be discovered the same way.

3. Cursor skill paths on this machine

📁 Where Cursor looks for skills
Scope Path on Windows Loaded by Use case
Global (user) C:\Users\<you>\.cursor\skills\ Cursor IDE Skills available in every project — recommended for Impeccable.
Project <project>\.cursor\skills\ Cursor IDE Project-specific skills (e.g., html-documentation-styling, ui-ux-pro-max).
Global (cursor-cli) C:\Users\<you>\.agents\skills\ Cursor CLI agent (and other .agents harnesses) Auto-detected by npx skills when the CLI agent is on the system. Not used by the IDE.
Cursor managed C:\Users\<you>\.cursor\skills-cursor\ Cursor IDE (built-ins) Skills shipped/managed by Cursor itself. Don’t put third-party skills here.
⚠️ Key point: The npx skills add CLI auto-detects cursor-cli on this machine and writes to ~\.agents\skills\, not ~\.cursor\skills\. Cursor IDE only reads ~\.cursor\skills\, so we copy the skill there in step 4.3.

4. Installation, step by step

Run all four steps in PowerShell from the Cursor terminal (or any PowerShell window). Each command block has a Template button (copies the original sample) and a Copy button (copies the customized command using your inputs). The Windows single-line version below each block is the exact string that goes to the clipboard.

4.1 Install via npx skills add

Use the official vercel-labs/skills CLI to fetch and install the Impeccable skill. The defaults below install pbakaus/impeccable globally for the cursor agent, copying files (avoids symlinks that need admin on Windows) and skipping prompts.

Setup — Install Skill
🔧 Customize parameters:
npx -y skills add pbakaus/impeccable -g -a cursor --copy -y
Windows single-line:
npx -y skills add pbakaus/impeccable -g -a cursor --copy -y

Parameter notes:

  • source — the GitHub owner/repo, full URL, or local path of any agent-skills repo (default pbakaus/impeccable).
  • agent — cursor for Cursor IDE, claude-code, codex, etc.
  • scope — -g for global (per user), omit for project-local install into ./.agents/skills/.
  • mode — --copy creates independent file copies. Recommended on Windows because the symlink alternative requires Administrator or Developer Mode.
✅ Expected result: Output ends with “Installation complete” and shows the install path. On this machine the CLI auto-detected cursor-cli and wrote to ~\.agents\skills\impeccable.

4.2 Verify where it landed

Quickly confirm the files exist and check whether they are also in the Cursor IDE skills folder yet (they won’t be after step 4.1 alone — that’s expected).

Verify — Check Install Location
🔧 Customize parameters:
if (Test-Path "C:\Users\$env:USERNAME\.agents\skills\impeccable") { dir "C:\Users\$env:USERNAME\.agents\skills\impeccable" }; if (Test-Path "C:\Users\$env:USERNAME\.cursor\skills\impeccable") { Write-Host "EXISTS in Cursor IDE" } else { Write-Host "NOT yet in Cursor IDE - run step 4.3" }
Windows single-line:
if (Test-Path "C:\Users\$env:USERNAME\.agents\skills\impeccable") { dir "C:\Users\$env:USERNAME\.agents\skills\impeccable" }; if (Test-Path "C:\Users\$env:USERNAME\.cursor\skills\impeccable") { Write-Host "EXISTS in Cursor IDE" } else { Write-Host "NOT yet in Cursor IDE - run step 4.3" }
💡 Expected output after step 4.1 only: a SKILL.md + reference/ + scripts/ listing under ~\.agents\skills\impeccable, plus the message NOT yet in Cursor IDE - run step 4.3. That’s normal — proceed to 4.3.

4.3 Copy into the Cursor IDE skills folder

Cursor IDE only reads ~\.cursor\skills\. Because --copy was used in step 4.1 (real files, not symlinks), we can safely copy the folder into the IDE’s skills directory. After this command, both cursor-cli and Cursor IDE will see the skill.

Setup — Copy to Cursor IDE
🔧 Customize parameters:
Copy-Item -Path "C:\Users\$env:USERNAME\.agents\skills\impeccable" -Destination "C:\Users\$env:USERNAME\.cursor\skills\impeccable" -Recurse -Force
Windows single-line:
Copy-Item -Path "C:\Users\$env:USERNAME\.agents\skills\impeccable" -Destination "C:\Users\$env:USERNAME\.cursor\skills\impeccable" -Recurse -Force

4.4 Final verification

List globally-installed skills and confirm Impeccable now shows up. The CLI’s “Agents:” line for impeccable may say not linked — that is fine because Cursor IDE picks up skills via the file copy at ~\.cursor\skills\, not via the CLI’s symlink registry.

Verify — List Global Skills
🔧 Customize parameters:
npx skills list -g
Windows single-line:
npx skills list -g
📊 Expected output
Global Skills

impeccable                   ~\.agents\skills\impeccable
  Agents: not linked
html-documentation-styling   ~\.claude\skills\html-documentation-styling
  Agents: Claude Code, Cursor

Plus, ~\.cursor\skills\impeccable\SKILL.md exists (verified by step 4.3). Restart Cursor or open a new chat — the agent will discover the skill.

5. Why the two-step install (cursor-cli vs Cursor IDE)

The npx skills CLI from vercel-labs/skills auto-detects which agent harnesses are present on the machine and routes the install accordingly. On this machine it sees the Cursor CLI binary and treats the install target as cursor-cli, whose global path is ~\.agents\skills\ — a generic location shared with several other CLI agents (Cline, Dexto, Warp, etc.).

The Cursor IDE, however, expects skills under ~\.cursor\skills\. They are two different agents with two different skill folders. Step 4.3 closes that gap by putting a real copy of the skill in the IDE’s folder. Both copies can co-exist safely — the CLI and the IDE simply read from different places.

💡 Could we skip the copy? Yes — if you only use the cursor-cli agent and not the IDE, the file at ~\.agents\skills\impeccable is enough. But for IDE chat usage (the most common case in this workspace), the copy in step 4.3 is required.

6. How to use Impeccable in this project (or any project)

📝 Invoking Impeccable in chat

Because the skill lives at ~\.cursor\skills\impeccable\, it is automatically available in every project on this machine, including mosc-temp. There is no per-project setup required. The agent will trigger the skill automatically when your prompt matches its description (frontend design, audit, polish, etc.), and you can also be explicit by naming a sub-command.

6.1 Sample chat prompts (copy & paste into a Cursor chat)

Prompt — audit a component
Use the impeccable skill to audit @src/components/Header.tsx. Focus on hierarchy, touch targets, accessibility, and any AI-slop tells (Inter, purple-to-blue gradients, nested cards, gray text on tinted backgrounds). Return a prioritized fix list with file/line references.
Prompt — critique a page
Run impeccable critique on the Membership page (@src/app/membership/MembershipClient.tsx). I want UX-design feedback: visual hierarchy, scan path, emotional resonance, copy clarity. Don’t change code yet — just give me the critique.
Prompt — polish before shipping
Run impeccable polish on the admin dashboard (@src/app/admin/page.tsx and the buttons it renders). Align the buttons with our admin_home_button_groups_styling rule, fix any inconsistent paddings or color uses, and prepare it for ship. List every change you made.
Prompt — first-time project setup
Use impeccable teach for this project. Read the existing components, the Tailwind config, and the .cursor/rules/mosc_styling_standards.mdc rule, then write PRODUCT.md and DESIGN.md at the repo root capturing our brand, users, tone, anti-references, color tokens, typography, and component conventions. Ask me anything you cannot infer from the code.
💡 Recommended one-time per project: Run impeccable teach once. The skill enforces a “context gate” before any design work and will prompt for PRODUCT.md if it’s missing. Having those files makes every later audit / polish / craft session sharply more on-brand.

7. The 23 Impeccable sub-commands

All sub-commands have a corresponding reference file under ~\.cursor\skills\impeccable\reference\<command>.md. Name the sub-command in your prompt (e.g. “run impeccable polish on …”) and the agent will load that reference.

8. Optional: standalone CLI for anti-pattern detection

Impeccable also ships a regex-based linter that runs without invoking the agent — useful in CI or for a quick local scan. It catches 24 issues across AI slop (side-tab borders, purple gradients, bounce easing, dark glows) and general design quality (line length, cramped padding, small touch targets, skipped headings, etc.).

Verify — Anti-Pattern Scan
🔧 Customize parameters:
npx impeccable detect src/
Windows single-line:
npx impeccable detect src/

Useful targets:

  • src/ — scan a directory (default).
  • index.html — scan a single HTML file.
  • https://example.com — scan a URL via Puppeteer.
  • --fast — regex-only, skip LLM & browser checks.
  • --json — machine-readable output for CI.

9. Maintenance: list, update, remove

9.1 Update Impeccable to the latest version

The CLI re-clones the source and refreshes its registered copy. Because Cursor IDE uses an independent file copy under ~\.cursor\skills\impeccable\, run the Copy-Item from step 4.3 again afterwards to refresh the IDE-side copy.

Expedite — Update Skill
🔧 Customize parameters:
npx skills update impeccable -g -y
Windows single-line:
npx skills update impeccable -g -y

9.2 Remove Impeccable

Removes the CLI-managed copy. To fully uninstall, also delete the IDE copy at ~\.cursor\skills\impeccable manually (see the second command below).

Expedite — Remove (CLI registry)
🔧 Customize parameters:
npx skills remove impeccable -g -y
Windows single-line:
npx skills remove impeccable -g -y
Expedite — Remove (Cursor IDE copy)
🔧 Customize parameters:
Remove-Item -Path "C:\Users\$env:USERNAME\.cursor\skills\impeccable" -Recurse -Force
Windows single-line:
Remove-Item -Path "C:\Users\$env:USERNAME\.cursor\skills\impeccable" -Recurse -Force

10. Caveats and limitations

  • No native slash commands in Cursor. Cursor IDE doesn’t support the /impeccable craft-style slash command syntax used by Claude Code or Codex. Invoke the skill via natural language (e.g. “use impeccable polish on …”).
  • Two file copies, two upgrade steps. The CLI manages ~\.agents\skills\impeccable; the IDE reads ~\.cursor\skills\impeccable. After every npx skills update, re-run the Copy-Item from step 4.3 to refresh the IDE copy.
  • Symlinks need admin on Windows. That’s why we used --copy in step 4.1. If you have Windows Developer Mode enabled (or run as Administrator), you can drop --copy for a single source-of-truth install.
  • Template placeholders in SKILL.md are intentional. Strings like {{command_prefix}} and {{scripts_path}} are resolved at runtime by the agent based on the harness; do not hand-edit them.
  • Skill auto-trigger may collide with your project rules. If Impeccable’s defaults conflict with project conventions (this repo has detailed Cursor rules in .cursor/rules/), run impeccable teach once so it captures and respects them in PRODUCT.md / DESIGN.md.

11. Troubleshooting

Symptom Likely cause Fix
Cursor agent doesn’t mention Impeccable in available skills Cursor not on Nightly, or Agent Skills toggle off, or skill not yet under ~\.cursor\skills\ Verify prerequisites in §2; re-run step 4.3; restart Cursor.
npx skills add says cursor-cli detected Expected behavior — the CLI sees the Cursor CLI binary and uses ~\.agents\skills\ Continue with step 4.3 to copy into the IDE folder. Do not try to override the auto-detect.
npx skills list -g shows Agents: not linked for Impeccable Impeccable was installed by step 4.1 but has no symlink registered to a CLI agent Harmless. Cursor IDE reads from the file copy at ~\.cursor\skills\impeccable and will still load it.
