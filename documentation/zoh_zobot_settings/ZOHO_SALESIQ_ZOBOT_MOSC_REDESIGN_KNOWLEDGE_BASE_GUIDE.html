Zoho SalesIQ Zobot — MOSC Redesign Knowledge Base Guide

Purpose. This guide explains how to feed the homepage chat widget (Zoho SalesIQ Zobot, registered under Giventa Inc.) with content from every public page under:

https://www.mosc-temp.com/mosc-redesign and all sub-paths.

Important. Zoho SalesIQ does not provide a “enter URL and crawl my whole site” feature like some third-party chatbot builders. You must extract page content, import it into SalesIQ Resources (Articles / FAQs), connect Answer Bot, and optionally plug Answer Bot into your existing Zobot flow. This document gives the full pipeline.

Related repo files: src/app/mosc-redesign/(syro)/sitemap/sitemapData.ts (curated link list), src/middleware.ts (public access), live sitemap page at /mosc-redesign/sitemap.

Alternatives: See section 12 for chatbot platforms that support paste URL → auto-crawl (SiteGPT, Chatbase, DocsBot, Tidio Lyro) and when they may be a better fit than SalesIQ alone.

Operations (OAuth, bulk delete, FAQ CSV, re-import): See the dedicated SalesIQ Operations Runbook for step-by-step instructions including obtaining ZOHO_REFRESH_TOKEN from the event-site-manager project, bulk deleting articles, and adding curated FAQ rows to salesiq-faq-supplement.csv.

cd f:\project_workspace\mosc-temp; npm run zobot:export-articles
Content extraction pipeline
  1. Fetch each URL from mosc-redesign-urls.txt (HTTP GET with realistic User-Agent, or headless browser if pages need JS).
  2. Parse main content: prefer <main>, or article body selectors used on mosc-redesign layout. Strip nav, footer, scripts.
  3. Build one row per page with at least:
    • Title — from <title> or first <h1>
    • Body — plain text or HTML (SalesIQ editor accepts rich text)
    • Category — e.g. “MOSC Redesign / Saints”, “Directory”, “Faith & Teaching” (match sitemap sections)
    • Tags — optional keywords (saint names, diocese, liturgy)
    • Source URL — canonical page URL (store in article or custom field for attribution)
  4. Deduplicate: some paths appear in multiple sitemap sections; one article per canonical URL.
  5. Review: have a content reviewer spot-check high-traffic pages before bulk publish.

Brand language & department (fix “Invalid article language”)

Article import uses the language labels configured on your brand, not ISO codes like en in the CSV alone. Confirm this before Validate:

  1. From salesiq.zoho.com/giventainc/onboard, open Settings → Brands → Giventa Inc. → General configurations.

Chat widget language API codes (en, fr, …) are documented in .salesiq.language() — article import expects the portal display name English.

Create article categories before bulk import

If Validate fails or the Edit info popup shows Select a Category with no match, SalesIQ does not recognize the category strings in your CSV. Categories must exist in the portal before bulk Validate (or be picked manually per row in Edit info).

Where categories live (Giventa portal): Articles is under the left sidebar (Resources) — not under Settings. On the Articles screen (your screenshot with Articles(0)), use one of these:

  1. Manage Categories icon — In the Articles page header (near Add and the English language dropdown), look for a folder / categories icon labeled Manage categories or similar. Click it → Add category → type the name exactly as in the CSV (e.g. General, Administration).
  2. While adding one article — Click Add → Add article (not Import). In the article form, open the Category field → choose Add new category / type a new name → save. Repeat for each name you need (or use the shortcut below).
  3. Search — On the Settings home screen, use the top Search bar and type category or article category if you cannot see the icon on Articles.
Shortcut (recommended for first import): Create one category named General, then patch the CSV so every row uses that category: npm run zobot:patch-import-csv -- --department "MOSC Connect" --language English --channels Web --category "General". You can split articles into more categories later in SalesIQ.

Full export uses ~31 flattened names (Administration, Saints, Directory, …). To keep those in the CSV, create each name via Manage categories before upload. Flattened names come from:

node scripts/zoh-zobot-settings/patch-salesiq-import-csv.mjs --department "MOSC Connect" --language English --flatten-categories

That keeps only the part after MOSC Redesign > (e.g. Administration) — create those shorter names in SalesIQ instead.

8. Configure Answer Bot

Answer Bot (knowledge-driven replies)
  1. Settings → Bots → Answer Bot — select the brand for www.mosc-temp.com.
  2. Connect knowledge base — link to Resources (Articles + FAQs) you imported.
  3. Enable “Fetch answer from resource (powered by OpenAI)” if you want concise answers instead of full article dumps (SalesIQ KB chatbot guide).
  4. Enable Self-service in the visitor chat window so users can browse articles by category.
  5. Map article categories to your website in SalesIQ brand settings.
  6. Monitor unanswered questions in Resources — add missing pages or FAQs when users ask something not covered.
Hybrid pattern (recommended): SalesIQ supports plugging Answer Bot into a rule-based Zobot so greetings and handoff stay in your flow while factual questions hit the knowledge base. See Zobot hybrid chatbot.

9a. First-time setup — “No bots found” (Giventa Inc.)

If you open Settings → Bots → Zobot and see:

No bots found
Create a new bot from scratch or import a bot

you have not created a Zobot yet. Follow the steps below. Use Create a new bot from scratch unless you already have a bot export file from another SalesIQ portal.

Step 0 — Confirm brand & website exist

  2. Go to Settings (gear icon, top right) → Brands.
  3. Verify a brand exists for www.mosc-temp.com (or the hostname visitors use). If missing, click Add brand, enter the site name and URL, and save.
  4. Open that brand → Installation tab — note the widget embed code for section 9e (you will paste this into the Next.js site later).

  1. Navigate to Settings → Bot → Zobot (direct link).
  2. Click Create a new bot from scratch (not Import unless you have a SalesIQ bot export).

  4. Platform: choose Codeless Bot Builder (recommended — visual flow + Answer Bot card; no Deluge required).

Step 2 — Bot profile
  1. Description: Short text shown to visitors, e.g. “Ask about the Malankara Orthodox Syrian Church, saints, directory, and events.”
  2. Choose brand: Select the www.mosc-temp.com brand (must match embed code domain).
  3. Associate department: Pick the department that should receive handoff chats (e.g. Support).
  4. Channels — Where bot is active: Enable Website. Leave social channels (WhatsApp, etc.) off unless configured separately.

  1. Choose working hours aligned with your brand’s business hours, or 24/7 if the public site should always offer the bot.

  3. Click Next.
Step 4 — Zobot configuration (audience & triggers)
  1. Bot audience:
    • All visitors — simplest for homepage + mosc-redesign.
    • Custom visitors — e.g. rule: Current page URL contains mosc-temp.com or path contains /mosc-redesign (use if you run multiple brands in one portal).
  2. When should the bot initiate chat? Recommended for homepage:
    • When the visitor clicks the chat widget — least intrusive; or
    • When the visitor lands on your site — proactive popup after a delay (optional).
  3. Response interval: Instant (or 1–2 seconds for a more human feel).
  4. Chat inactivity rules: Optional reminder before auto-close (e.g. warn at 5 min, end at 8 min).
  5. Handoff to operator: Enable if live agents monitor SalesIQ. Set label e.g. “Connect with a person?” → Yes. Check Hide outside business hours if no overnight staff.
  6. Forward message: Optional text while connecting to an operator.
  7. Actions: Enable Go back / Restart chat if you want visitors to recover from wrong paths.
  8. Click Create. You enter the Codeless Bot Builder canvas (section 9c).

9c.3 Connect the Answer Bot “Failure” branch (your current screen)

After you add an Answer bot Card, the canvas shows a purple Failure pill on the right side of that card, with a dashed line to an open +. That path runs when Answer Bot cannot find a match in Resources (or the lookup fails). SalesIQ still treats it as an incomplete flow until you connect it.

Yes — you need one more card on the Failure branch. Click the + after Failure and add one of these (simplest first):

B — Live support Forward (Action card) Transfers chat to a SalesIQ operator when someone is online.

12. Alternatives: auto-scrape chatbot platforms

Several products support “paste a URL or sitemap → auto-crawl → train → embed widget” more directly than Zoho SalesIQ. SalesIQ excels at live chat, agent inbox, and Zobot flows; its gap is automatic website ingestion, not chat UX.

Quick decision matrix
If you need… Better fit than SalesIQ alone
Paste domain → auto-crawl → chat widget SiteGPT, Chatbase, DocsBot, Tidio Lyro
Full support platform (tickets, agents, AI) Intercom Fin, Zendesk AI
Keep SalesIQ but fix the crawl gap Crawl tool + SalesIQ Articles import or Articles REST API

12a. Services that auto-scrape websites

Product Auto-crawl features Refresh / sync Widget + handoff
SiteGPT Website recursive crawl, sitemap.xml, URL list; depth + max pages; include/exclude path filters (e.g. /mosc-redesign) Scheduled auto-sync (hourly / daily / weekly / monthly) — Auto sync docs Embed widget; no native Zoho agent desk
Chatbase Crawl full site, sitemap, or individual URLs; review discovered links; path include/exclude — Data sources docs Website sources re-crawl about every 24 hours Embed, API, Slack; configurable email fallback
DocsBot AI Single URL, URL list (CSV/TXT), sitemap, RSS, WordPress, many cloud sources — URL list training Daily / weekly / monthly scheduled refresh on sources Widget, API, Slack, Teams; strong support-tool integrations
Tidio Lyro Scrape single page or entire site from top-level URL — Lyro data sources; API: POST /lyro/data-sources/website/scrape Manual re-sync + auto-sync fields on website sources Live chat + AI + human handoff (closest to SalesIQ model)
Crawl APIs (Firecrawl, KnowledgeSDK, Crawl4AI) Full-site or API-driven extract; you own chunking + storage Webhooks / scheduled jobs (DIY) No widget — pipe into SalesIQ Articles API or any bot
Recommended seed for MOSC: Start crawl from https://www.mosc-temp.com/mosc-redesign and restrict paths to /mosc-redesign so event homepage and admin routes are excluded. With ~272 App Router pages, set max pages ≥ 300 and verify coverage after first crawl.

12c. When SalesIQ is still the right choice

Stay on SalesIQ if you need…
  • Giventa SalesIQ portal with permission to create a Zobot (section 9a — first-time setup at Zobot settings)
  • Live operators, queues, and operator chat window
  • Zoho stack (CRM, Desk, campaigns) integration
  • Answer Bot + hybrid Zobot with human-reviewed Articles/FAQs

SalesIQ’s limitation is ingestion, not chat. Use sections 5–7 of this guide, or combine with a crawl-first tool (below).

12e. Caveats for www.mosc-temp.com/mosc-redesign

Risk Detail Mitigation
No XML sitemap yet Crawlers can recurse from seed URL but may miss deep pages Add sitemap.ts later; use URL list from section 5 as crawl seed
JavaScript-rendered content Some scrapers fail on client-only pages mosc-redesign uses SSR for most content — test sample saint/directory pages after training
AI hallucinations Bots may invent links or facts not on your site Test with real questions; constrain to retrieved content; human review for theology
Sacred / doctrinal accuracy Auto-scrape ≠ official church approval Content reviewer sign-off before publish; link to canonical page URLs in answers
Do not run two chat widgets on the same page without a clear split (e.g. SalesIQ for “talk to a person”, Chatbase for “search the site”). Overlapping bubbles confuse visitors and waste API quota.
