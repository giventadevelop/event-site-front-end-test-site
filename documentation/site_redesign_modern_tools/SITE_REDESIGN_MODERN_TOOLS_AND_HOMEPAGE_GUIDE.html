2. Recommended four-step workflow

High-level process
  1. Claude Code (or Cursor Agent) — Use an agentic CLI or IDE agent to plan refactors, keep files consistent, and apply multi-file edits with review.
  2. Framer Motion (optional) — Add when you need orchestrated enter/exit transitions, scroll-linked motion, or gesture-driven UI on the hero and sections. If you only need subtle fades, prefer existing Tailwind animation utilities first.
  3. UI/UX Pro Max skill — Run --design-system queries to avoid generic layouts and to get palette, typography, and pattern recommendations tuned to “nonprofit / cultural / community” keywords; extend queries with Malayalee diaspora / Keralites in America language once research themes are defined.
  4. 21st.dev (and similar) — Browse community React/Tailwind blocks (for example premium hero scaffolding) as reference or starting points. Port patterns manually into this codebase so they comply with MOSC tokens and accessibility rules.

Framer Motion and 21st.dev — registration and API keys

  • Framer Motion — This is an open-source npm package for React. You do not register on a site or obtain an API token to use it in your app. Install with npm install framer-motion, read framer.com/motion, and copy or adapt examples from the docs into your components (typically under 'use client'). Licensing is the library’s open-source license, not a separate “Framer login” for the motion package. (Framer’s hosted site builder product is separate; this guide refers to the motion library in code.)
  • 21st.dev — browsing the site (inspiration / copy snippets) — You can usually open 21st.dev in the browser, browse components, and copy code by hand without setting up an API key. Anything that is clearly “log in to see this” or “Pro” follows their product rules on the site.
  • 21st.dev — Magic MCP, CLI, or integrated IDE flows — If you use their Cursor MCP, official CLI (npx @21st-dev/cli), or similar, you typically need a 21st.dev account and an API key (often from their Magic Console). Store keys only in local MCP or environment configuration—never commit secrets. See current 21st.dev documentation for exact env var names and setup.

4. Homepage improvements for mosc-temp

Design and technology actions
  • Respect existing sacred / MOSC palette — Use mosc_styling_standards.mdc tokens (cream background, earth tones) unless the org approves a full rebrand.
  • Hero clarity — Keep the four-section hero narrative but tighten typography scale, contrast, and CTA hierarchy; ensure mobile containment rules in globals.css stay correct.
  • Motion policy — Prefer CSS and Tailwind for small hovers; add Framer Motion only for hero or key CTAs if you need staged reveals. Avoid heavy animation on first paint (LCP).
  • Content blocks below the fold — Apply UI/UX Pro Max output to “events teaser”, “community stories”, or “donation / membership” rows for a more SaaS-like clarity while staying on-brand.
  • Accessibility — Preserve focus rings, semantic headings, and reduced-motion respect (prefers-reduced-motion) when adding motion libraries.
  • Performance — Lazy-load non-critical sections; keep hero images optimized and aligned with next/image patterns already in the app.
  • Diaspora narrative — The homepage should gradually reflect Malayalees in the USA / Keralites in America as an underlying theme (not a one-line slogan only). Plan this via research and design cues below.

Malayalee / Keralite diaspora in America — underlying homepage theme (research backlog)

Status: to be researched and validated with stakeholders. The points below are a design brief, not final copy or art direction. Community leaders should review tone, terminology (Malayalee vs Keralite vs region-specific), and imagery so the site feels authentic and inclusive rather than decorative or stereotypical.
Why this theme matters

MCEFEE / MOSC serves a community that spans heritage rooted in Kerala and everyday life in the United States. The homepage should signal—quietly and consistently—that visitors belong to that bridge: cultural programs, families, professionals, and donors who identify with Malayali/Keralite culture while living in America.

Research to schedule (before locking visuals)
  • Language & naming — Preferred terms in your audience (Malayalee, Malayali, Keralite, NRI, diaspora); bilingual headline or subcopy strategy (English primary; optional Malayalam accents for names or taglines—accessibility and readability first).
  • Audience segments — First-generation vs second-generation; geographic chapters in the US; what “home” means in messaging without excluding anyone.
  • Visual authenticity — Photo sources: real community events vs stock; representation of generations, genders, and US contexts (not only Kerala tourism imagery).
  • Competitive / peer scan — How other Malayalee associations and South Asian nonprofits in the US express identity on the hero and above the fold.
  • Legal / cultural sensitivity — Flags, maps, religious motifs: use only with clear org policy; prefer neutral cultural cues (arts, language, food as event context, community gathering) over clichés.

How to project the idea in homepage design (once research is done)

  • Copy architecture — Hero subtitle and supporting line that name the two anchors (e.g. cultural heritage × American community life) without long essays; repeat the thread in one section below the fold (e.g. “Who we serve”).
  • Imagery & collage — Side-by-side or layered storytelling: US locale cues + cultural programming; consistent color grading so MOSC earth tones still dominate per mosc_styling_standards.mdc.
  • Programs as proof — Feature event types, language classes, youth, charity work—concrete modules that show Malayalee life in America in action.
  • Typography & micro-details — Optional Malayalam-friendly font stack for short phrases if design system allows; ensure Latin text remains primary for WCAG and non-Malayalam readers.
  • Motion & rhythm — Subtle transitions between hero panels can suggest “connection” or “journey”; avoid gimmicks that trivialize identity.
  • SEO & metadata — Page title and description can include phrases like “Malayalee community in [region]” where accurate—coordinate with content owners.
Traceability: When this research completes, add a short documentation/ memo (or PRD excerpt) with agreed vocabulary and image guidelines, and link it from this guide so future redesigns stay consistent.

7. Step-by-step: getting familiar

Week 1 — Orientation

  1. Read .cursor/rules/mosc_styling_standards.mdc, skim HeroSection.tsx, and review diaspora theme research backlog with stakeholders.
  2. Watch one overview video from the curated list (pick “premium SaaS UX” or “Impeccable” based on your focus).
  3. Run a UI/UX Pro Max design-system query for your niche (see commands below).

Week 2 — Prototype

  1. Branch in Git; optionally install Framer Motion and animate a single client subcomponent.
  2. Pull one inspiration block from 21st.dev or section 5 / section 6; redraw it using MOSC colors (do not paste vendor CSS blindly).
  3. Run npm run build to catch SSR/client boundary issues.

Week 3 — Integrate

  1. Peer review against icons_buttons_styles.mdc and image_containment_prevention_hero_images_cards.mdc.
  2. Test on real mobile devices (middleware and public proxy rules already document mobile pitfalls).

10. Sample agent prompt (homepage facelift)

Copy the block below into Cursor or Claude Code when you want a scoped redesign that still obeys repo rules.

Homepage facelift prompt
You are working in the mosc-temp Next.js App Router repo. Goal: improve the public homepage look and feel (especially HeroSection and first screen) while STRICTLY preserving MOSC design tokens from .cursor/rules/mosc_styling_standards.mdc and hero mobile rules in .cursor/rules/hero_mobile_image_containment_aspect_ratio.mdc. Underlying narrative (coordinate with stakeholders; do not invent sensitive cultural claims): The organization serves Malayalees / Keralites in the United States. Propose how the homepage can subtly project that bridge—heritage rooted in Kerala and community life in America—through copy hierarchy, imagery direction, and section storytelling. Avoid stereotypes and decorative clichés; prefer authentic community programming, inclusive representation, and clear English-first readability (optional Malayalam only where agreed and accessible). Constraints: - Do not break Clerk, proxy routes, or server/client boundaries (client components only for motion). - Prefer existing Tailwind and tailwindcss-animate; add framer-motion only if justified and tree-shaken. - Maintain accessibility: focus states, contrast, semantic headings, prefers-reduced-motion. - Align buttons/icons with .cursor/rules/icons_buttons_styles.mdc. Deliverables: 1) Short audit of src/components/HeroSection.tsx and src/app/page.tsx (what to improve). 2) Concrete UI changes (spacing, typography, CTA hierarchy, optional subtle motion). 3) Brief “diaspora theme” recommendations: headline/subcopy ideas, imagery notes, and one below-the-fold section concept—explicitly marked as needing leadership/content review. 4) List of files touched and manual test steps (desktop + mobile).
