Date: March 2026
Status: Draft

Executive Summary

This document defines (1) end-user join: a logged-in user can join a focus group, with the mapping between focus group and user profile stored in the existing focus_group_members table; and (2) executive/organising committee: a list of members with role EXECUTIVE or ORGANISER (and optionally LEAD, ADMIN) per focus group, using the same table’s role column. No new tables are required; only documentation and optional SQL comments on role and status.

Key Concept: The mapping focus group ↔ user profile already exists in public.focus_group_members (focus_group_id, user_profile_id, tenant_id, role, status). End-user “join” = insert one row; “committee” = members where role IN (EXECUTIVE, ORGANISER).

1. Data Model

focus_group_members (existing table):

Column Type Description
id bigint Primary key
tenant_id varchar(255) Tenant isolation
focus_group_id bigint FK to focus_group(id)
user_profile_id bigint FK to user_profile(id)
role varchar(50) MEMBER, LEAD, ADMIN, EXECUTIVE, ORGANISER (EXECUTIVE/ORGANISER = committee)
status varchar(50) PENDING, ACTIVE, INACTIVE
created_at, updated_at timestamp Audit fields

Unique constraint: (tenant_id, focus_group_id, user_profile_id). One membership row per user per focus group per tenant.

2. Role and Status Values

role (allowed values):

  • MEMBER – Regular member (default for end-user join).
  • LEAD – Lead (optional committee display).
  • ADMIN – Admin (optional committee display).
  • EXECUTIVE – Executive committee member.
  • ORGANISER – Organising committee member.

The executive/organising committee list for a focus group is defined as all focus_group_members rows for that focus group where role IN ('EXECUTIVE', 'ORGANISER'). Optionally include LEAD and ADMIN per product decision.

status (allowed values):

  • PENDING – Awaiting approval (default for self-service join if policy requires approval).
  • ACTIVE – Active membership.
  • INACTIVE – Inactive (e.g. left or revoked).

3. Out of Scope

Event–focus group association (event_focus_groups) and event media focus group tagging (event_media.event_focus_group_id) are covered in the separate PRDs under documentation/focus_group_requirements/event_media_integration/. This PRD covers only focus group membership (join) and committee (role-based list).
