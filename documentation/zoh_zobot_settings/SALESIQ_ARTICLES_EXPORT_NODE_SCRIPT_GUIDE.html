SalesIQ Articles Export — Node Script (Step 2)

This script automates Step 2 of the Zoho SalesIQ knowledge-base pipeline: it reads your URL list, fetches each public /mosc-redesign page, extracts title and main content, and writes salesiq-articles-import.csv. Recommended next step: post articles via REST API — Operations Runbook §9 Bulk upload (npm run zobot:bulk-upload-articles -- --confirm). Portal CSV import is documented as an alternative in the same runbook (§10) and main guide Section 7. See also: Zobot knowledge base guide.

1. What it does

InputOutputScript
documentation/zoh_zobot_settings/mosc-redesign-urls.txt documentation/zoh_zobot_settings/salesiq-articles-import.csv scripts/zoh-zobot-settings/export-salesiq-articles.mjs

CSV columns: Title, Description, Content, Category, Language, Source URL, Tags. Match these to Zoho’s sample import spreadsheet in your portal (column names may differ slightly — rename columns in Excel if needed).

Malformed URL list? If lines look like https://www.mosc-temp.comF:/project_..., the script still normalizes them to https://www.mosc-temp.com/mosc-redesign/.... Re-run the PowerShell generator in section 5 of the main guide for a clean source file.

2. Prerequisites

  • Node.js 18+ (uses built-in fetch)
  • Repository cloned with npm install already run (uses project package.json scripts only — no extra npm packages)
  • URL list file present at documentation/zoh_zobot_settings/mosc-redesign-urls.txt
  • Live site reachable at https://www.mosc-temp.com (script fetches production HTML)

3. Where to run

Repository root required

Run all commands from the repository root (folder containing package.json), e.g. f:\project_workspace\mosc-temp.

Use PowerShell or Windows Terminal — not CMD — for the cd ...; npm run ... lines below.

4. Commands

Test export (first 10 URLs)
PowerShell — run at repo root
cd f:\project_workspace\mosc-temp; npm run zobot:export-articles:test
Full export (~265 URLs; 7 dynamic templates auto-skipped)
PowerShell — run at repo root (several minutes; 400ms delay between requests)
cd f:\project_workspace\mosc-temp; npm run zobot:export-articles
Dry run (parse URLs only, no HTTP)
cd f:\project_workspace\mosc-temp; node scripts/zoh-zobot-settings/export-salesiq-articles.mjs --dry-run --limit 5
Optional flags (append to node scripts/zoh-zobot-settings/export-salesiq-articles.mjs):
  • --limit 25 — process first N URLs
  • --delay-ms 800 — slower pacing between requests
  • --input path\to\urls.txt — custom URL list
  • --output path\to\out.csv — custom CSV path
  • --dry-run — list normalized URLs without fetching
  • --include-templates — also fetch placeholder routes like .../news/[slug] (usually not needed for SalesIQ)

After a full run you should see about 265 articles in the CSV (not 272 — dynamic App Router folders are skipped). If you previously got 250 articles and 22 errors, pull the latest exporter and run again; the 16 real-page failures were fixed by stripping RSC scripts before parsing <main>.

7. Alternative: portal CSV import

Settings → Articles → Add
  1. Sign in at salesiq.zoho.com/giventainc/onboard
  2. Confirm brand language: Settings → Brands → Giventa Inc. → General configurations → Languages — English selected and default (main guide #import-brand-language)
  3. Create article Categories matching the CSV before bulk import (main guide #import-categories)
  4. Settings → Articles (Settings sidebar) → Add → upload salesiq-articles-import.csv
  5. On Add Articles mapping: map every column 1:1; set Article status to Publish (or Status in sheet + map Status). See main guide section 7 mapping table.
  6. Click Validate, then finish import
  7. Import a test batch first (use --limit 10 when exporting)
  8. Review a few articles in the portal, then run the full export and import again

8. Troubleshooting

IssueCauseFix
ENOENT on input file Missing URL list Run PowerShell URL generator (main guide section 5) or create mosc-redesign-urls.txt
Many rows in errors JSON Site down, 404, Next.js inline scripts breaking <main> parsing, or page has little text Re-run with latest exporter (strips scripts before parsing); check URL in browser; increase --delay-ms
Extracted content too short on directory/news/calendar Fixed in exporter v2 — RSC <script> tags inside <main> caused early </main> match Pull latest script and run npm run zobot:export-articles again
URLs like .../news/[slug] in errors Dynamic route placeholders from file scan — not real pages Re-run PowerShell generator (skips [...] folders) or let exporter skip them automatically
Validate: "Invalid article language" CSV used en or brand has no English Patch: npm run zobot:patch-import-csv -- --department "MOSC Connect" --language English --channels Web; enable English under Settings → Brands → Giventa Inc. → General configurations → Languages
Edit info: Channels blank / globe not selected CSV used Website instead of Web Patch with --channels Web and re-upload; map Channels → Channels
Edit info: Category blank / Validate fails Category string not in SalesIQ Create categories in Settings → Articles → Categories, or patch with --flatten-categories and create shorter names
Edit info: Resource ID Expectation that new rows need numeric IDs For Action = Create, leave Resource ID empty in CSV and in Edit info — only existing articles use numeric IDs
Validate fails on Department / Is Public Required SalesIQ fields not in our CSV Set Publish (not Status in sheet); pick MOSC Department in dropdown; set Is Public default if offered
CSV import rejected by SalesIQ Column names don’t match portal template Align headers with Zoho sample XLS; save as CSV UTF-8
Bot still has no answers Articles not linked to Answer Bot Complete Answer Bot setup (main guide section 8), not Zobot URL import
