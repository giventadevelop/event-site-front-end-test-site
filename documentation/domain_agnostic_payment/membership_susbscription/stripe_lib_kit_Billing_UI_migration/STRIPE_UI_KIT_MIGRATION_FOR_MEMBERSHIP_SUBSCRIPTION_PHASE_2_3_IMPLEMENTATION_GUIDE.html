Overview

Phase 2/3 makes Stripe Checkout the primary and only payment method when enabled. This differs from Phase 1 (hybrid approach) where both Stripe Elements and Stripe Checkout were available.

✅ Phase 2/3 Goals:
  • Make Stripe Checkout the ONLY payment option (no inline payment)
  • Hide Stripe Elements components when useCheckoutFlow === true
  • Simplify user experience (single payment path)
  • Reduce code complexity (no dual payment flows)

Enabling Checkout-Only Flow (Phase 2/3)

Step 1: Set Environment Variable

Set NEXT_PUBLIC_USE_STRIPE_CHECKOUT=true to enable Phase 2/3 (Checkout-only flow).

Local Development (.env.local)

NEXT_PUBLIC_USE_STRIPE_CHECKOUT=true

Production (AWS Amplify Console)

  1. Go to App Settings → Environment Variables
  2. Add or update: NEXT_PUBLIC_USE_STRIPE_CHECKOUT = true
  3. Redeploy application

Rollback Instructions

If you need to rollback to Phase 1 (hybrid approach), simply set:

NEXT_PUBLIC_USE_STRIPE_CHECKOUT=false

This will restore the hybrid approach where both Stripe Elements and Stripe Checkout are available.

⚠️ Important: The Payment Intent code is still in place (just hidden), so rollback is immediate with no code changes needed. This is by design for safety.
