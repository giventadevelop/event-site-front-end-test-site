Event Registration Enhancements - Frontend Changes

Frontend implementation details for the Next.js project.

Pages Updated

  • src/app/events/[id]/register/page.tsx
  • src/app/events/[id]/register/success/page.tsx

Register Page Changes

  • Added notes textarea with maxLength=1500 and live character counter.
  • Added attachment picker for max 2 files with remove action.
  • Client validation for file type and size (10MB each).
  • Submit flow changed to:
    1. Create attendee via /api/proxy/event-attendees
    2. Upload each file via /api/proxy/event-attendee-attachments/upload
  • UI styling improved with branded background gradients, card surfaces, and stronger action button treatments.

Success Page Changes

  • Added notes display section.
  • Added uploaded attachments section with metadata and file link.
  • Added attachment fetch: /api/proxy/event-attendee-attachments?attendeeId.equals={attendeeId}
  • Improved visual hierarchy and CTA button styles.

Type and API Proxy Changes

  • Added EventAttendeeAttachmentDTO in src/types/index.ts.
  • Added proxy routes:
    • src/pages/api/proxy/event-attendee-attachments/index.ts
    • src/pages/api/proxy/event-attendee-attachments/[...slug].ts
    • src/pages/api/proxy/event-attendee-attachments/upload.ts
