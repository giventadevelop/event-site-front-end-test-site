Prevention Strategy

1. Environment Variable Management

✅ DO ❌ DON'T
  • Use AWS Secrets Manager or Parameter Store for sensitive keys
  • Use environment variables only for non-sensitive configuration
  • Never log environment variables in build scripts
  • Use AWS IAM roles instead of hardcoded keys when possible
  • Copy/paste keys directly into AWS Amplify console
  • Store keys in properties files (even if in .gitignore)
  • Log environment variables in build logs
  • Share keys via email/chat

2. Secure Key Storage

# Use AWS Secrets Manager
aws secretsmanager create-secret \
  --name "stripe/live/secret-key" \
  --secret-string "sk_live_..."

# Reference in application
application:
  payment:
    stripe:
      secret-key: ${aws.secretsmanager:stripe/live/secret-key}

3. Build Script Security

# Never log sensitive values
function Write-SecureValue {
    param($value)
    if ($value -and $value.Length -gt 8) {
        return $value.Substring(0, 4) + "***" + $value.Substring($value.Length - 4)
    }
    return "***"
}

# Use masked logging
Write-Host "Secret Key: $(Write-SecureValue $secretKey)"
