2. Date string parser — parseManifestDateString()

New: scripts/gallery-porting/parse-manifest-album-date.mjs

Input: manifest date string. Output:

{
  eventDateStart: '2015-10-24' | null,  // ISO date
  eventDateEnd: '2015-10-26' | null,
  eventLocation: 'Indore' | null,
  albumYear: 2015                       // always from year in string or start date
}

Parse rules (order matters)

  1. Trim input; empty → all null, keep existing albumYear.
  2. Location split: If string matches /,\s*([A-Za-z][A-Za-z\s.-]{0,100})$/, extract location and remove from date portion. Example: October 24-26, 2015, Indore → date part October 24-26, 2015, location Indore.
  3. Year only: /^\d{4}$/ → no event dates; albumYear = that year.
  4. Same-month range: /^([A-Za-z]+)\s+(\d{1,2})-(\d{1,2}),\s*(\d{4})$/ → start = month day1 year, end = month day2 year.
  5. Single date: Date.parse or new Date(datePart) with en-US month names → start only.
  6. albumYear = year from start date or parsed year.

3. Extend import script

File: scripts/gallery-porting/import-static-gallery-to-api.mjs

  • In createAlbum(), call parser on albumMeta.date and include eventDateStart, eventDateEnd, eventLocation in POST payload.
  • In category backfill PATCH path, include same fields when patching from manifest.

4. New backfill script

scripts/gallery-porting/backfill-gallery-album-event-dates.mjs
  • Load static-gallery-manifest.json
  • GET all gallery albums for tenant
  • Match by static_slug= in description
  • PATCH each album with parsed event fields + albumYear if needed
  • Write report JSON: event-date-backfill-report.json
  • Support --dry-run
