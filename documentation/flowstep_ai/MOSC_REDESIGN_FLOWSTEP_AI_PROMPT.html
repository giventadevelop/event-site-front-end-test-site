1. How to use this with Flowstep

Recommended workflow
  1. Open Flowstep.ai and start a new design session.
  2. Attach a reference: paste https://www.mosc-temp.com/mosc-redesign as a design reference URL (Flowstep supports PRD links, inspiration images, and reference URLs).
  3. Copy the Master prompt from Section 6 into the chat. Optionally attach a screenshot of the live homepage or a subpage (e.g. Catholicate).
  4. Ask Flowstep to generate multiple screens in one pass: homepage, Catholicate hub, Administration hub, content inner page with sidebar, and mobile header.
  5. Export or copy designs to Figma (⌘C / ⌘V per Flowstep docs), then translate approved patterns back into the Next.js codebase using the repo design tokens.
Tip: Flowstep outputs React + TypeScript + Tailwind. This project already uses that stack under src/app/mosc-redesign/. Treat Flowstep output as visual direction, not drop-in production code — preserve existing routes, CMS content, and server components.

2. Project context & goals

Organization: Malankara Orthodox Syrian Church (MOSC) — ancient Indian Orthodox Christian tradition; apostolic mission of St. Thomas.

Product: Modern public website redesign (Next.js 15, React, TypeScript, Tailwind CSS) replacing legacy mosc.in layout while preserving sitemap, content hierarchy, and reverent tone.

Design inspiration: Layout clarity inspired by syromalabarchurch.in (structure only). Color palette: let Flowstep propose a cohesive scheme — do not lock specific hex values in the prompt.

Live reference: https://www.mosc-temp.com/mosc-redesign

Enhancement goals (what Flowstep should improve)

  • Modernize while staying Orthodox: fresher grid, whitespace, typography scale, and subtle motion — but always reverent, warm, and unmistakably Malankara Orthodox (liturgical dignity, no startup/SaaS clichés).
  • Homepage rearrangement: rethink section order and visual weight on the homepage — e.g. hero + shortcuts, About Us, Saints, Catholicos, calendar, world map, contact — so the flow feels contemporary without dropping any content block. Adjust image sizes, crops, and placement (side-by-side vs stacked, portrait vs landscape frames) for better balance.
  • Visual polish: stronger hierarchy, spacing rhythm, card elevation, and CTA clarity without losing sacred/reverent character.
  • Color exploration: Flowstep should choose its own harmonious palette (primary accent, backgrounds, text, footer) suited to an Orthodox church site — warm, sacred, readable — and document the hex values it picks.
  • Header & navigation: logo prominence, single-line church title on desktop, hamburger + drawer on mobile, consistent nav bar styling across tiers.
  • Hero & banners: full-bleed imagery with dignified treatment; no accidental cropping of episcopal portraits or liturgical art.
  • Component consistency: unify primary buttons, outline buttons, cards, icon shortcut bars, sidebars, breadcrumbs, and footer across all subpages.
  • Content-heavy inner pages: improve readability for long theological/historical text (The Church, Catholicate history, Saints, Lectionary).
  • Directory & interactive sections: world map, liturgical calendar widget, contact form — clearer layout and touch targets on mobile.
  • Accessibility: WCAG 2.1 AA contrast, focus rings, semantic headings, prefers-reduced-motion respect.

3. Design direction (mood & components)

Wisest approach for Flowstep: Do not prescribe exact colors (e.g. burgundy hex codes) in the prompt. Instead, describe the mood and constraints and let the tool propose a full palette. Lock colors only after you approve a direction and map them into the codebase (mosc-redesign-design-tokens.ts). The live site and repo JSON may use burgundy/parchment today — treat that as reference, not a Flowstep requirement.

Optional repo reference: design_systems/design_samples/mosc_re_design_latest_design_system.json

Color mood (guide the tool — not fixed hex values)

Primary accentDeep, dignified tone — e.g. wine, maroon, earth red, or muted gold — for nav, CTAs, active states (tool’s choice)
BackgroundsWarm light neutrals (cream, parchment, soft stone) — avoid cold gray or pure white slabs
TextWarm dark browns or deep burgundy-blacks; strong contrast for long reading
FooterDarker warm anchor color that pairs with the chosen accent
AvoidNeon, generic SaaS blue/gray, playful gradients, dark “gaming” UI

Typography & shape (flexible)

Token Guidance
TypographyClean humanist sans (DM Sans or similar); headings semibold/bold; body 14–16px, comfortable line-height
Border radiusCards ~12px; buttons rounded-lg or pill for outline variants

Key components to keep consistent

  • Three-tier header: logo/title bar → main nav → quick links bar (tool picks colors; keep clear hierarchy).
  • Hero carousel: ~400px desktop / ~300px mobile; dignified transitions; optional shortcut icons below.
  • CTA icon bar: Catholicate News, Downloads, E-Mail, Gallery, Contact — icon + label, hover lift.
  • Section cards: About Us, Saints carousel, Catholicos profile, liturgical calendar, world map, contact form.
  • Inner page pattern: page banner + breadcrumb + optional left sidebar + main content column.
  • Footer: darker warm footer, quick links, contact blurb, copyright.

5. Constraints & non-goals

  • Modern + Orthodox balance — contemporary layout and components welcome; sacred/reverent tone and dignified imagery are non-negotiable.
  • Sacred / reverent aesthetic — warm earth tones and soft backgrounds; not neon, not startup SaaS, not overly playful.
  • Color palette — Flowstep chooses the palette; prompt specifies mood only (warm, Orthodox, readable). Avoid locking hex codes unless you are re-running for token alignment with an approved design.
  • Homepage layout freedom — you may reorder sections and resize/reposition images on the homepage for clarity and modern feel; keep every existing section type, just improve composition.
  • No hero click-through links on rotating banners unless the existing site already has them (prefer visual dignity over aggressive marketing).
  • Image containment: portraits, episcopal vestments, and liturgical art must not be awkwardly cropped; use object-contain or aspect-ratio-aware frames where faces matter.
  • Responsive: mobile-first stacking; hamburger menu; touch-friendly 44px+ targets.
  • Framework: designs should translate to Tailwind utility classes; after approval, map chosen colors into project tokens.
  • Non-goals: changing URL structure, removing sections, or adding unrelated features (e-commerce, social feeds). Do not force the current live-site burgundy if a better Orthodox-appropriate palette emerges.

Master prompt — MOSC redesign enhancement
You are an expert product designer enhancing an existing church website redesign. Reference site: https://www.mosc-temp.com/mosc-redesign (Malankara Orthodox Syrian Church). Also study https://www.mosc-temp.com/mosc-redesign/catholicate as the best current subpage example. PROJECT - Organization: Malankara Orthodox Syrian Church (MOSC), Indian Orthodox tradition, apostolic heritage. - Goal: MODERNIZE the current redesign while preserving its Orthodox character — make it feel more contemporary (cleaner grids, better whitespace, refined typography, thoughtful image composition) without losing reverence, warmth, or liturgical dignity. Polish layout, navigation, cards, and mobile experience. Preserve all existing content sections and information architecture (same sections, better presentation). - Stack target: Next.js + React + TypeScript + Tailwind CSS (designs should use Tailwind-friendly spacing and color tokens). HOMEPAGE RE-LAYOUT (important) - Reference the live homepage: https://www.mosc-temp.com/mosc-redesign - You MAY rearrange homepage sections for a stronger visual story — e.g. lead with hero + shortcut icons, then About Us, Saints, Catholicos profile, liturgical calendar, worldwide map, contact — but keep every section type that exists today. - Adjust images around the page: rebalance portrait vs landscape frames, improve side-by-side vs stacked layouts on desktop/mobile, fix awkward crops, and give episcopal portraits and church photography more intentional placement (not just stacked full-width blocks). - Aim for a slightly more modern editorial feel (magazine-style section breaks, asymmetric two-column blocks where helpful) while staying unmistakably Orthodox — no generic corporate or tech landing-page tropes. DESIGN SYSTEM — COLOR (you choose) - Propose a complete, cohesive color palette for an Orthodox church website: primary accent, secondary, page backgrounds, card surfaces, body text, headings, footer, borders, and hover states. - Mood: sacred, reverent, warm, contemporary-but-timeless. Suitable for Malankara Orthodox Syrian Church (Indian Orthodox tradition). - Do NOT default to generic blue/gray SaaS UI. Warm creams, deep wine/maroon/gold/earth tones are all acceptable if harmonious and WCAG AA compliant. - Document every hex (or Tailwind arbitrary value) you use on each screen. Use the same palette consistently across homepage, inner pages, and components. Typography: DM Sans or similar clean humanist sans. Headings: semibold/bold, reverent not flashy. Body: 14–16px, generous line-height for long theological articles. Components to unify across ALL pages: 1) Three-tier header: (a) logo + church name on light background, (b) main nav bar with strong accent, (c) quick links bar. Desktop: horizontal nav. Mobile: hamburger opening full-height drawer; logo remains visible; Search Directory accessible. 2) Hero: rotating banner ~400px desktop; dignified imagery (Patriarch, Aramana, bishops); optional CTA icon row below (News, Downloads, Email, Gallery, Contact). 3) Cards: light surface bg, ~12px radius, subtle border, hover lift + shadow. 4) Primary button: accent fill, white or high-contrast text, rounded-lg, hover lighten + slight translateY. 5) Outline button: 2px accent border, accent text, pill or rounded-lg. 6) Inner pages: top banner image + breadcrumb + left sidebar (section tree) + main reading column. Sidebar active state uses primary accent. 7) Footer: dark warm anchor color from your palette, quick links, contact snippet, copyright. PAGES IN SCOPE (apply same design language to each): - Homepage /mosc-redesign - The Catholicate + historical Catholicos pages - Administration, Holy Synod, Ecumenical, Dioceses, Saints (+ saint biography pages) - The Church mega-section (theology, liturgy, spirituality, church history, calendar) - Spiritual Organisations, Publications, Institutions, Training, Theological Seminaries - Lectionary, Downloads, Calendar, Gallery, News - Directory search (bishops, priests, parishes, dioceses) - Contact / email pages ENHANCEMENT PRIORITIES 1. Homepage modernization: rearrange sections, rebalance images, improve spacing and section rhythm — contemporary but reverent. 2. Stronger visual hierarchy on homepage — hero, About Us, Saints, Catholicos section, calendar, world map, contact form. 3. Consistent button styles (replace any legacy flat/outdated buttons with design system primary/outline). 4. Improve mobile: header compression, readable nav drawer, stacked sections, touch-friendly controls. 5. Content pages: better readability (max-width ~720px for prose), sidebar spacing, heading scale, pull quotes optional. 6. Directory & map: cleaner filters, clearer region highlights on world map. 7. Accessibility: WCAG AA contrast, visible focus states, semantic heading order. CONSTRAINTS - Modernize the UI, but keep the Orthodox nature — sacred, reverent, warm; NOT tech startup, NOT dark mode gaming aesthetic, NOT excessive gradients. - Homepage: rearranging layout and adjusting images is encouraged; do not delete content sections. - Do not crop faces in portraits; use contain or careful object-position. - Do not remove sections or change church doctrine/copy without labeling as "draft copy". - Do not change URL paths. DELIVERABLES (generate multiple screens in one session if possible) A) Enhanced homepage desktop + mobile B) Catholicate hub page desktop C) Generic inner content page with sidebar (The Church article layout) D) Administration card grid page E) Component sheet: buttons, cards, nav states, form fields, footer For each screen: annotate spacing, list the color palette you chose (with hex), and show consistent token usage.
