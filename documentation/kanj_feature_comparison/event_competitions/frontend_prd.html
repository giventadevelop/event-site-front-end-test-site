All backend access via proxy routes (/api/proxy/...) and server actions using fetchWithJwtRetry. Client components never call the backend URL directly. DTOs live in src/types/index.ts.

1. Proxy routes

Each resource: src/pages/api/proxy/{resource}/index.ts + [...slug].ts using createProxyHandler. PATCH slug routes require export const config = { api: { bodyParser: false } }.

6. Winner photo upload (admin)

Follow existing event media upload flow (admin event media pages). After upload, PATCH result with winnerMediaId and optional denormalized winnerPhotoUrl from media fileUrl. Same S3 bucket patterns as gallery.

10. Stripe webhook

On payment_intent.succeeded, when metadata.registrationType === 'event_competition':

metadata: {
  registrationType: 'event_competition',
  eventId: '123',
  competitionRegistrationIds: '1,2,3',
  tenantId: 'tenant_demo_002'
}

PATCH registrations to CONFIRMED via service JWT. Do not alter standard ticket metadata paths.
