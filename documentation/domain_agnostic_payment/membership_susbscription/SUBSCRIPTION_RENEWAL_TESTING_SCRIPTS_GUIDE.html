Overview

These scripts allow you to test subscription renewal functionality without waiting for the actual billing period to end. They work with Stripe test mode subscriptions to:

  1. View subscription details - Retrieve and display comprehensive Stripe subscription information
  2. Expedite renewal - Advance subscription period dates in Stripe test mode
  3. Verify database - Compare database state with Stripe subscription state
  4. Trigger batch job - Execute batch job on demand for testing
  5. Full test suite - Run all tests in sequence

Scripts

  • 1. View Stripe Subscription Details

    File: view-stripe-subscription.js

    Retrieves and displays detailed information about a Stripe subscription including dates, status, billing information, and other attributes.

    📋 Script Usage
    Verify Script
    🔧 Customize Parameters:
    node scripts/test-subscription-renewal/view-stripe-subscription.js --subscription-id=sub_1SeifsK5BrggeAHMBvg2XE93
    Windows Single-Line:
    node scripts/test-subscription-renewal/view-stripe-subscription.js --subscription-id=sub_1SeifsK5BrggeAHMBvg2XE93
    ⚙️ Script Parameters

    Parameters:

    • --subscription-id (required): Stripe subscription ID
    📊 What it displays
    • Subscription status and customer information
    • Current billing period dates (start, end, days until renewal)
    • Billing cycle anchor
    • Billing information (currency, collection method, interval, amount)
    • Subscription items and pricing
    • Cancellation settings
    • Trial information (if applicable)
    • Latest invoice details
    • Payment method information
    • Metadata
    • Additional subscription attributes
    📊 Script Output
    • Comprehensive subscription details formatted for easy reading
    • All dates in both ISO format and localized format
    • Days until renewal calculation

  • 2. Expedite Stripe Renewal

    File: expedite-stripe-renewal.js

    Advances a Stripe test mode subscription's billing period to trigger renewal immediately.

    📋 Script Usage
    Expedite Script
    🔧 Customize Parameters:
    node scripts/test-subscription-renewal/expedite-stripe-renewal.js --subscription-id=sub_1SeifsK5BrggeAHMBvg2XE93 --days-to-advance=30
    Windows Single-Line:
    node scripts/test-subscription-renewal/expedite-stripe-renewal.js --subscription-id=sub_1SeifsK5BrggeAHMBvg2XE93 --days-to-advance=30
    ⚙️ Script Parameters

    Parameters:

    • --subscription-id (required): Stripe subscription ID
    • --days-to-advance (optional): Number of days to advance (default: 30)
    📊 What it does
    1. Retrieves current subscription from Stripe
    2. Calculates new period dates
    3. Updates subscription billing cycle anchor
    4. Creates and pays invoice for new period
    5. Verifies final subscription state
    📊 Script Output
    • Current and new period dates
    • Invoice creation status
    • Final subscription state
    Note: Running this script multiple times will advance the subscription month by month. Each run reads the current Stripe date and advances by one billing period (typically 30 days).

  • 3. Verify Database

    File: verify-database.js

    Compares database subscription record with Stripe subscription state.

    📋 Script Usage
    Verify Script
    🔧 Customize Parameters:
    node scripts/test-subscription-renewal/verify-database.js --subscription-id=sub_1SeifsK5BrggeAHMBvg2XE93 --tenant-id=tenant_demo_002
    Windows Single-Line:
    node scripts/test-subscription-renewal/verify-database.js --subscription-id=sub_1SeifsK5BrggeAHMBvg2XE93 --tenant-id=tenant_demo_002
    ⚙️ Script Parameters

    Parameters:

    • --subscription-id (required): Stripe subscription ID
    • --tenant-id (optional): Tenant ID (defaults to NEXT_PUBLIC_TENANT_ID)
    📊 What it does
    1. Fetches subscription from Stripe
    2. Fetches subscription from database via API proxy
    3. Compares:
      • current_period_start
      • current_period_end
      • subscription_status
      • stripe_customer_id
    4. Reports discrepancies
    5. Checks reconciliation status
    📊 Script Output
    • Comparison results (matches, warnings, issues)
    • Reconciliation status
    • Recommendations

Troubleshooting

Stripe API Errors

Error Solution
"No such subscription" Check subscription ID is correct and in test mode
"Invalid API key" Verify STRIPE_SECRET_KEY is set and is a test mode key
"Cannot update subscription" Subscription may be canceled or in invalid state

Database Connection Errors

Error Solution
"Connection refused" Check DATABASE_URL and database is running
"Authentication failed" Verify database credentials
"Subscription not found" Check subscription exists in database with correct tenant_id

Batch Job Errors

Error Solution
"401 Unauthorized" Check API_JWT_USER and API_JWT_PASS
"404 Not Found" Verify batch job endpoint exists in backend
"500 Internal Server Error" Check backend logs for details

Important Notes

⚠️ Test Mode Only

These scripts only work with Stripe test mode subscriptions. Do not use with production subscriptions.

📋 Database Safety

Scripts are read-only for database (except batch job which updates). The batch job will update subscription dates to match Stripe.

⏱️ Timing Considerations

  • Webhook Timing: Allow 5-10 seconds for webhooks to process after expediting
  • Batch Job Timing: Allow 10-30 seconds for batch job to complete
  • Multiple Runs: Running expedite script multiple times will advance subscription month by month
