1. Architecture — per-table id sequences

Three-layer alignment
LayerRequirement
PostgreSQLCREATE SEQUENCE public.{table}_id_seq INCREMENT BY 1
Hibernate@SequenceGenerator(..., sequenceName = "public.{table}_id_seq", allocationSize = 1)
After importsync_all_table_sequences.sql and/or POST /api/admin/sync-sequence

Most application tables use id bigint DEFAULT nextval('public.{table}_id_seq'::regclass).

Separate sequences (do not merge):

  • batch_job_seq, batch_job_execution_seq, batch_step_execution_seq — Spring Batch framework
  • batch_job_execution_log_id_seq — batch_job_execution_log.id (BIGSERIAL / IDENTITY)

2. Why next IDs start at 600000+

CauseExplanation
MOSC seed blockcorrected_event_media_inserts.ordered.sql includes tenant mosc_malankara_orthodox_2 rows at 600001+
Per-table sequencesEach table uses {table}_id_seq; new ids are MAX(id)+1 for that table only
Import without syncManual INSERTs do not update last_value
Sync after ordered importsetval sets sequence to global MAX(id) — ~600k is correct for that file
Legacy increment 50Old JHipster DBs used INCREMENT BY 50 + allocationSize = 50
Lower new IDs: Import sequence_fix_inserts/corrected_event_media_inserts.renumbered.sql, then run sync_all_table_sequences.sql. Executive committee next id = that table's max + 1 (not inflated by event_media).

3. SQL scripts — reorder, renumber, sync

3.1 Reorder — FK-safe table order

reorder_sql_inserts_final.cjs

Path: code_html_template/SQLS/reorder_sql_inserts_final.cjs

Input: export.sql · Output: corrected_event_media_inserts.ordered.sql

Maintain TABLE_ORDER when export adds tables. Console must report Extra tables (not in order): none.

Spring Batch tables must appear in dependency order: instance → execution → context → params → step → step_context → log.

3.2 Optional renumber — compact PKs + trim batch

renumber_sql_insert_ids.cjs

Path: code_html_template/SQLS/sequence_fix_inserts/renumber_sql_insert_ids.cjs

  • Per-table PKs starting at 1
  • Column-aware FK rewrite (schema + manual overrides)
  • Trim Spring Batch to first N executions + N log rows (default 3)
  • Output: corrected_event_media_inserts.renumbered.sql, id-mapping-report.json

Does not replace canonical SQLS/corrected_event_media_inserts.ordered.sql until import is validated.

3.3 Sync after import (mandatory)

Step 7 — sync_all_table_sequences.sql
docker exec -i CONTAINER_ID psql -U event_site_admin -d event_site_manager_db -v ON_ERROR_STOP=1 < code_html_template/SQLS/Current_Sqls/sync_all_table_sequences.sql
Windows single-line:
docker exec -i CONTAINER_ID psql -U event_site_admin -d event_site_manager_db -v ON_ERROR_STOP=1 < code_html_template/SQLS/Current_Sqls/sync_all_table_sequences.sql
Step 7b — sync_spring_batch_sequences.sql
docker exec -i CONTAINER_ID psql -U event_site_admin -d event_site_manager_db -v ON_ERROR_STOP=1 < code_html_template/SQLS/Current_Sqls/sync_spring_batch_sequences.sql

docker exec -i CONTAINER_ID psql -U event_site_admin -d event_site_manager_db -v ON_ERROR_STOP=1 < code_html_template/SQLS/Current_Sqls/sync_spring_batch_sequences.sql

Run on every machine: Schema + import on a local DB does not sync sequences automatically. Steps 7 and 7b must run on that database before relying on the app for new IDs.

4. Backend — event-site-manager-service

Repo: F:/project_workspace/event-site-manager-service (active API; not legacy malayalees-us-site-boot).

Expected entity annotation
@GeneratedValue(strategy = GenerationType.SEQUENCE, generator = "sequenceGenerator")
@SequenceGenerator(
    name = "sequenceGenerator",
    sequenceName = "public.{table}_id_seq",
    allocationSize = 1

All domain entities under src/main/java/com/eventsitemanager/domain/ (~78 files) must use allocationSize = 1.

6. Run order (every database / machine)

Standard pipeline
  1. Latest_Schema_Post__Blob_Claude_12.sql
  2. node reorder_sql_inserts_final.cjs
  3. Optional: node renumber_sql_insert_ids.cjs
  4. Import ordered or renumbered INSERT file
  5. sync_all_table_sequences.sql
  6. sync_spring_batch_sequences.sql (if batch data present)
  7. Deploy backends with allocationSize = 1
  8. Legacy once: fix_per-table id sequences_increment_by_one.sql if increment_by = 50
