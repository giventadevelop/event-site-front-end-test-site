3. GalleryAlbum entity / mapper changes

  • Add @ManyToOne GalleryCategory galleryCategory on GalleryAlbum (lazy)
  • Map albumYear as Integer column
  • On GET list/detail: optionally eager-fetch category for public gallery (or separate join in query service)
  • POST/PATCH: accept galleryCategoryId and albumYear; validate year range 1900–2100
  • On PATCH: allow galleryCategoryId: null to clear category

6. Optional: media count on album list (v1.0 nice-to-have)

Frontend currently N+1 fetches media per album in fetchAlbumsForGallery. Backend optional enhancement:

  • Option A (defer): Keep frontend loop — no backend change
  • Option B: Add read-only DTO field mediaCount populated in query service via subquery/COUNT join
  • Option C: Dedicated GET /api/gallery-albums/with-stats composite endpoint

Recommendation: Option A for v1.0; Option B in v1.1 if performance matters.
