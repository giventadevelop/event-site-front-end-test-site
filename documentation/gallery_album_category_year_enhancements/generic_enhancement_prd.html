This pack extends the existing Gallery Album feature so public cards can be driven from admin/API data instead of hardcoded arrays in mosc-redesign/(syro)/gallery/page.tsx. It adds a tenant-scoped gallery category lookup and an album year field on each album for display (e.g. “Ecumenical Visits”, “2019”).

Non-breaking rule: New columns and tables are nullable or have safe defaults. Existing albums without category/year continue to render with fallbacks.

Executive summary

Problem and solution

The MOSC redesign gallery shows rich album cards (cover image, “60 photos” badge, category pill “Ecumenical Visits”, title, year “2019”, View Album CTA). That data is hardcoded in src/app/mosc-redesign/(syro)/gallery/page.tsx. The database-backed gallery at /gallery uses GalleryAlbumCard with title, cover, photo count, and createdAt — but no category tag and no visit/event year.

Solution (v1.0): Add gallery_category (tenant lookup) and album_year on gallery_album; expose via REST + proxy; extend admin album create/edit; update public cards to match redesign styling; optionally wire MOSC redesign gallery to the same API.

Open questions — recommended defaults

  1. Category model: Use normalized gallery_category table (recommended) — not free-text on album — for consistent pills and admin dropdown.
  2. Year semantics: album_year = calendar year of the visit/event, not record created_at.
  3. Album detail URL (MOSC): Keep existing static routes (/mosc-redesign/gallery/russia-visit) until CMS slug field is added; v1.0 can use optional slug column deferred to v1.1.
  4. Category admin: v1.0 — seed fixed list + admin dropdown only; full category CRUD page optional in G2.
  5. Filter on gallery page: Defer category/year filter chips to v1.1; v1.0 display-only on cards.

Success criteria

  • Admin can assign category and album year when creating/editing a gallery album.
  • Public gallery album cards show category pill and year matching the redesign mockup.
  • Photo count and cover image continue to work from existing media association.
  • MOSC redesign gallery can load albums from API (or shared server action) with same card shape.
  • Existing albums without category/year render without errors (graceful fallbacks).
