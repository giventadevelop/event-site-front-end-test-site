Auto Tenant ID — Backend Enhancements PRD

Project: malayalees-us-site-boot / Rust API  |  Status: Optional (recommended)  |  Index: generic_enhancement_prd.html

v1 frontend computes the next tenant ID by listing organizations. For production-grade concurrency, add a backend endpoint that returns the next sequence atomically (or assigns ID on create).

1. Recommended REST endpoint

GET next tenant ID preview
MethodPathAuth
GET/api/tenant-organizations/next-tenant-idService JWT + super-admin role

Query: prefix (required) — slug before the numeric suffix.

Response 200:
{
  "prefix": "ford_motors",
  "sequenceNumber": 10000,
  "formattedSequence": "10000",
  "tenantId": "ford_motors_10000"
}

2. Server logic

  1. Normalize prefix (lowercase, underscores, strip invalid chars).
  2. Query: SELECT MAX(CAST(substring(tenant_id from '_(\\d+)$') AS BIGINT)) FROM tenant_organization WHERE tenant_id ~ '_(\\d+)$' (PostgreSQL).
  3. If null → next = 1; else next = max + 1.
  4. Format suffix with minimum 5 digits.
  5. Return DTO; do not consume sequence until POST create (preview is advisory) OR use DB sequence table for reservation.

3. Optional: assign on POST

Extend POST /api/tenant-organizations:

  • If body includes tenantIdPrefix and omits tenantId, server assigns next ID inside a transaction.
  • Reject if computed ID already exists (retry with +1).

4. Criteria / list

No change required. Existing GET /api/tenant-organizations with pagination remains the data source for admin lists.

5. Frontend integration (future)

  • Replace previewNextTenantIdServer internal pagination with proxy call to /api/proxy/tenant-organizations/next-tenant-id?prefix=....
  • Add Pages Router proxy slug or dedicated handler in src/pages/api/proxy/tenant-organizations/.

Not required for v1: Frontend-only generation works when tenant creation volume is low. Adopt backend endpoint before high-volume or multi-admin concurrent onboarding.
