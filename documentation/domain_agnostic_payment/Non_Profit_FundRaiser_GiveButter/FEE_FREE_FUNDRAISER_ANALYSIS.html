Zeffy Integration Options

⚠️ CRITICAL LIMITATION: Zeffy API Availability

Important Finding: According to Zeffy's Help Center, Zeffy does NOT have an open-source API. This means:

  • ❌ No traditional API secret keys for direct integration
  • ❌ Cannot build custom ZeffyPaymentAdapter with REST API calls
  • ✅ Integration available exclusively through Zapier
  • ✅ May support embedded widgets/iframes (needs verification)
  • ✅ May support redirect-based flows (needs verification)

Impact: This significantly changes the implementation approach. See "Alternative Integration Strategies" below.

What is Zeffy?

Zeffy is a zero-fee fundraising platform designed exclusively for non-profit organizations. It covers all processing fees by offering donors optional tips to support platform operations.

Zeffy Features

  • Ticketing: Sell and manage event tickets with zero fees
  • Donations: Customizable donation forms
  • eCommerce: Online store for merchandise
  • Memberships: Membership program management
  • Peer-to-Peer Campaigns: Enable supporters to fundraise
  • Raffles & Lotteries: Organize fundraising raffles
  • Auctions: Host online auctions
  • Donor Management: Maintain donor database
  • Tap to Pay: In-person payments via smartphone

Zeffy Integration Requirements

Eligibility Requirements

  • Organization must be a registered non-profit/charity
  • Must provide proof of charitable status (EIN, CRA, etc.)
  • Currently supports US, Canada, and EU regions (2025)
  • KYC verification required

Alternative Integration Strategies

Since Zeffy doesn't have an open API, here are viable integration approaches:

Option 1: Zapier Integration (Recommended if using Zeffy)

// ZeffyZapierIntegrationService.java @Service public class ZeffyZapierIntegrationService { /** * Initialize Zeffy donation via Zapier webhook * Flow: Our System → Zapier → Zeffy → Zapier Webhook → Our System */ public PaymentSession initializeZeffyViaZapier(PaymentInitializeRequest request) { // 1. Create donation record in our system with status PENDING PaymentTransaction transaction = createPendingTransaction(request); // 2. Trigger Zapier webhook with donation details // Zapier will create Zeffy donation form/campaign String zapierWebhookUrl = getZapierWebhookUrl(request.getTenantId()); Map<String, Object> zapierPayload = buildZapierPayload(request, transaction); // 3. Zapier creates Zeffy donation and returns Zeffy URL String zeffyDonationUrl = triggerZapierWebhook(zapierWebhookUrl, zapierPayload); // 4. Return redirect URL for frontend return PaymentSession.builder() .providerType(ProviderType.ZEFFY) .sessionUrl(zeffyDonationUrl) // Redirect to Zeffy .providerTransactionId(transaction.getId().toString()) .build(); } /** * Handle Zapier webhook callback when Zeffy donation completes */ public void handleZapierCallback(ZapierWebhookPayload payload) { // Zapier sends donation completion data // Update payment_transaction status // Record zero-fee transaction } }

Zapier Integration Considerations

  • Pros: Only way to programmatically integrate with Zeffy
  • Cons: Requires Zapier account and webhook setup
  • Cost: Zapier may have subscription costs
  • Reliability: Depends on Zapier uptime
  • Latency: Additional hop adds delay

Fallback Strategy

Recommended Fallback Flow

  1. Check if event is configured for Zeffy zero-fee fundraising
  2. Attempt to redirect to Zeffy donation page (or embed widget if available)
  3. If Zeffy unavailable or user prefers, fallback to Stripe with DONATION type
  4. Display message: "Zero-fee option unavailable, using standard donation (fees apply)"
  5. Allow donor to cover fees option in Stripe checkout

Givebutter is an excellent alternative to Zeffy with significant advantages:

  • API: ✅ Open-source REST API with full documentation

  • Integration: RESTful API with JSON responses - ideal for custom integration
  • Documentation: Comprehensive API docs with code samples

  • Tenant Support: ✅ Perfect fit for multi-tenant architecture

Since Zeffy has API limitations, consider TicketStripe as an alternative:

// GivebutterPaymentAdapter.java @Service @RequiredArgsConstructor public class GivebutterPaymentAdapter implements PaymentAdapter { private final RestTemplate restTemplate; private final EncryptionService encryptionService; @Override public PaymentSession initializePayment(PaymentInitializeRequest request) { // 1. Get tenant's Givebutter configuration PaymentProviderConfig config = getProviderConfig( request.getTenantId(), ProviderType.GIVEBUTTER ); // 2. Decrypt API key from config_data String apiKey = decryptApiKey(config.getConfigData()); // 3. Create donation/campaign in Givebutter GivebutterDonationRequest gbRequest = GivebutterDonationRequest.builder() .amount(request.getAmount()) .campaignId(getCampaignId(request.getEventId(), config)) .donorEmail(request.getCustomerEmail()) .donorName(request.getCustomerFirstName() + " " + request.getCustomerLastName()) .metadata(Map.of( "event_id", request.getEventId().toString(), "tenant_id", request.getTenantId(), "payment_type", "DONATION_ZERO_FEE" )) .build(); // 4. Call Givebutter API HttpHeaders headers = new HttpHeaders(); headers.set("Authorization", "Bearer " + apiKey); headers.setContentType(MediaType.APPLICATION_JSON); HttpEntity<GivebutterDonationRequest> entity = new HttpEntity<>(gbRequest, headers); ResponseEntity<GivebutterDonationResponse> response = restTemplate.postForEntity( "https://api.givebutter.com/v1/donations", entity, GivebutterDonationResponse.class ); // 5. Return payment session return PaymentSession.builder() .providerType(ProviderType.GIVEBUTTER) .providerTransactionId(response.getBody().getDonationId()) .sessionUrl(response.getBody().getCheckoutUrl()) // If redirect needed .clientSecret(null) // Givebutter uses different flow .metadata(Map.of( "givebutter_donation_id", response.getBody().getDonationId(), "checkout_url", response.getBody().getCheckoutUrl() )) .build(); } @Override public PaymentStatusResponse checkStatus(String providerTransactionId) { // Poll Givebutter API for donation status // Or handle via webhook } private String decryptApiKey(String encryptedConfig) { // Decrypt config_data JSON and extract API key String decrypted = encryptionService.decrypt(encryptedConfig); JsonNode config = objectMapper.readTree(decrypted); return config.get("api_key").asText(); } }

// GivebutterWebhookHandler.java @RestController @RequestMapping("/api/webhooks/givebutter") public class GivebutterWebhookHandler { @PostMapping public ResponseEntity<?> handleWebhook( @RequestHeader("X-Givebutter-Signature") String signature, @RequestBody String payload ) { // 1. Givebutter for Church Mass Offerings & Collections

Sacred Context: The Offertory at Mass

The Offertory, or Collection, at Mass is an integral part of the Sunday liturgy, not just because it is the primary means by which the parish receives financial support from the parishioners, but because it is also an expression of our gratitude to God for the blessings He has given us to return a portion of them back.

Givebutter provides an ideal solution for modernizing this sacred tradition while maintaining reverence and ease of use.

Use Cases for Mass Offerings

  • In-Person Mass: QR codes displayed during collection, quick mobile donations
  • Live-Streamed Mass: Real-time donation widget embedded in live stream
  • Sunday Collection: Weekly recurring offerings with optional prayer intentions
  • Special Collections: Designated offerings for specific causes (building fund, missions, etc.)
  • Votive Offerings: Spontaneous offerings with prayer intentions

1. Stripe Non-Profit Discount Program

Eligibility & Application

  • Eligibility: Registered non-profit with 80%+ tax-deductible donations
  • Discount Rate: 2.2% + $0.30 per transaction (vs. standard 2.9% + $0.30)

    • EIN or IRS 501(c)(3) determination letter
    • Primary email associated with Stripe account
    • Statement confirming 80%+ tax-deductible donations
