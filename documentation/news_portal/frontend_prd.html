Executive Summary

This document defines the frontend requirements for the News Portal feature in the Next.js application (mosc-temp). It covers public news pages (landing, category, article detail), navigation integration, and the admin dashboard for editors to manage articles, categories, section images, and flash news. All API access follows nextjs_api_routes.mdc (proxy + server actions). UI must follow .cursor/rules/ui_style_guide.mdc and mosc_styling_standards.mdc.

2. Public News Pages

Landing (/news)

  • Fetch: section config (for section images/titles), sidebar promotions (active rotating banners for right column slideshow), flash items, and recent articles per section/category.
  • Layout: Flash News carousel at top; then blocks for Main News, Featured News, Press Release – each with optional section banner image and list of articles (image, title, date, excerpt). Right sidebar: rotating banner ads/promotions as a slideshow with timer (each slide shown for its display_duration_seconds), plus optional social/video block.
  • Links: Section titles link to /news/category/[slug]; article cards link to /news/[slug].

Category (/news/category/[slug])

  • Fetch: articles for category (by slug), paginated; optional section config for that category’s block.
  • Layout: Paginated list/grid of articles; pagination controls per project pagination footer styling.

Article (/news/[slug])

  • Fetch: single article by slug (title, body, featured image, date, author, categories).
  • Layout: Article header (image, title, date, author), body content; optional “Back to News” / category links.

Live Video (/news/live)

  • Fetch: current live stream config (embed_url, title, is_active) from backend (e.g. /api/proxy/news-live-stream-config or single active record).
  • Layout: Full-width or centred embed container (e.g. YouTube iframe from embed_url), optional title above; if no active stream, show a friendly “No live stream at the moment” message.

Public pages may use direct fetch to proxy endpoints if routes are in middleware public/ignored lists; otherwise use server components that fetch via server actions or proxy with JWT where required.

3. Admin Dashboard

Admin news area must be user-friendly and self-guiding so editors can update news and section images regularly without developer help.

Admin Hub (/admin/news)

  • Button group or card links to: Articles, Categories, Section Images, Sidebar Promotions (rotating banners), Flash News, Live Stream (per admin home button groups styling).

Articles

  • List articles with filters (category, status); create/edit form with title, slug, excerpt, body, featured image, category selection, published_at, status.
  • Validation and form styling per form_validation_styling.mdc; server actions in ApiServerActions.ts for create/update/delete.

Categories

  • List categories; create/edit name, slug, description, display_order, is_active.

Section Images (/admin/news/sections)

  • One row/card per section key: flash_news, main_news, featured_news, most_read, press_release, sidebar_promotional_block.
  • Each row: section label, optional title override, image upload or URL input, save button. Editors update each section’s image independently. The sidebar_promotional_block row controls the right sidebar’s section-level title/banner override; rotating banner ads are managed on /admin/news/sidebar-promotions.
  • Use PATCH to backend for section config; follow PATCH direct-backend pattern from nextjs_api_routes (service JWT) where applicable.

Sidebar Promotions – Rotating Banners (/admin/news/sidebar-promotions)

  • List promotional items (banner image, link, display order, display duration (seconds) per slide, valid_from/valid_to, is_active). Add/edit/delete items. Display duration controls how long each slide is shown in the slideshow with timer on the public news page right sidebar.
  • Public news landing fetches active promotions (filtered by tenant, is_active, valid date range), orders by display_order, and renders a slideshow that auto-advances after display_duration_seconds per slide.

Flash News

  • List flash items; add/edit headline, link (URL or article), display order, valid_from/valid_to, is_active.

Live Stream (/admin/news/live)

  • Single form or card: embed URL (e.g. YouTube embed src for live stream), optional title, is_active toggle. Saving updates the current live stream config so /news/live shows this embed. User-friendly labels (e.g. “Paste YouTube Live embed URL here”) so editors can update without developer help.

All admin API calls must use server actions and fetchWithJwtRetry for backend; no direct fetch from client components to backend (nextjs_api_routes.mdc).

4. API Integration

  • Proxy routes: Add /api/proxy/news-articles, /api/proxy/news-categories, /api/proxy/news-section-config, /api/proxy/news-flash, /api/proxy/news-live-stream-config using createProxyHandler from src/lib/proxyHandler.ts. Do not add tenantId.equals in client/server when calling proxy (proxy injects tenant).
  • Server actions: Place in ApiServerActions.ts under the relevant app segment (e.g. src/app/news/ for public, src/app/admin/news/ for admin). Use fetchWithJwtRetry for any direct backend call from server actions.
  • Public reads: For public pages, if proxy is in public/ignored routes, client components may fetch proxy URLs with getAppUrl() for absolute URL in server context; relative URLs in client.
  • DTOs: Import types from @/types; no local DTO declaration in API routes (nextjs_api_routes.mdc).

5. UI Standards

  • General: Follow .cursor/rules/ui_style_guide.mdc and mosc_styling_standards.mdc (colors, typography, spacing, sacred design elements).
  • Admin: Pagination per pagination_footer_styling.mdc; buttons per admin_action_buttons_styling.mdc and admin_home_button_groups_styling.mdc; icons per icons_buttons_styles.mdc (inline SVG, no react-icons for medium action icons).
  • Forms: Validation, error display, scroll-to-error, onBlur validation per form_validation_styling.mdc.
  • Images: Section/hero images that must not be cropped: use object-contain and h-auto container per image_containment_prevention.mdc.
  • Media grids: If news admin uses media grids, follow media_gallery_grid_style.mdc.

6. Menu Integration

Add “News” as a menu or submenu item in the main site navigation (e.g. in src/components/Header.tsx or equivalent) linking to /news. Placement (top-level vs under a parent) to be decided; ensure route is accessible to public (public or ignored as needed in middleware).
