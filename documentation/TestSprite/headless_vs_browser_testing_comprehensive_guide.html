1. Understanding Headless vs Browser-Based Testing

1.1 What is Headless Testing?

Headless testing runs a browser engine without a visible UI. Playwright/Chromium runs in the background, executing tests programmatically.

  • No visible browser window - Runs in background
  • Faster execution - No rendering overhead
  • CI/CD friendly - Works in server environments without display
  • Full browser capabilities - Still uses real Chromium engine
  • JavaScript execution - Full JS runtime, can detect errors

1.2 What is Browser-Based Testing?

Browser-based testing (also called "headed" or "non-headless") opens a visible browser window that you can watch and interact with.

  • Visible browser window - You can see what's happening
  • Slower execution - Rendering overhead
  • Better for debugging - See exactly what the test is doing
  • Manual intervention possible - Can pause and inspect
  • Same capabilities - Uses same browser engine as headless

1.3 Key Misconception: "Headless Doesn't Catch JavaScript Errors"

⚠️ This is FALSE

Headless testing CAN and DOES detect JavaScript errors. Playwright's headless mode uses the same Chromium engine as headed mode, with full JavaScript execution.

What headless testing CAN detect:

  • ✅ JavaScript runtime errors (ReferenceError, TypeError, etc.)
  • ✅ Console errors and warnings
  • ✅ Network errors (404, 500, etc.)
  • ✅ Page load failures
  • ✅ Unhandled promise rejections
  • ✅ React component errors

What headless testing CANNOT do:

  • ❌ Show you a visual representation (you can't "see" the page)
  • ❌ Allow manual interaction during test execution
  • ❌ Debug visually (but screenshots help)

  • Higher resource usage
  • Still won't catch errors if pages aren't visited

💡 Best Use Case

Use browser-based mode for debugging specific test failures, not for comprehensive testing.

✅ Recommended Strategy

    • Visual debugging helps identify the issue

  3. Manual Override: Force Event IDs

// Always set up error detection BEFORE navigating to pages async function runTest(page, test, config) { const jsErrors = []; const consoleErrors = []; // Set up error listeners BEFORE navigation page.on('pageerror', (error) => { jsErrors.push({ message: error.message, stack: error.stack, timestamp: new Date().toISOString() }); console.error(` ⚠️ JavaScript Error: ${error.message}`); }); page.on('console', (msg) => { if (msg.type() === 'error') { consoleErrors.push({ text: msg.text(), timestamp: new Date().toISOString() }); console.error(` ⚠️ Console Error: ${msg.text()}`); } }); // NOW navigate to the page await page.goto(`${config.baseUrl}${test.url}`, { waitUntil: 'networkidle', timeout: test.timeout || config.timeout }); // Wait for page to fully load await page.waitForLoadState('networkidle'); // Check for errors if (jsErrors.length > 0 || consoleErrors.length > 0) { return { ...test, status: 'failed', jsErrors, consoleErrors, error: 'JavaScript errors detected' }; } // Continue with element verification... }

// Verify page actually loaded before testing async function verifyPageLoad(page, test, config) { // Check HTTP status const response = page.response(); if (response && response.status() !== 200) { return { status: 'failed', error: `Page returned ${response.status()} instead of 200` }; } // Wait for React to hydrate await page.waitForLoadState('domcontentloaded'); await page.waitForLoadState('networkidle'); // Check for React error boundaries const reactError = await page.evaluate(() => { // Check for React error boundary indicators const errorBoundary = document.querySelector('[data-react-error-boundary]'); if (errorBoundary) { return errorBoundary.textContent; } return null; }).catch(() => null); if (reactError) { return { status: 'failed', error: `React error detected: ${reactError}` }; } // Check for Next.js error page const isErrorPage = await page.evaluate(() => { return document.title.includes('Error') || document.body.textContent.includes('Application error'); }); if (isErrorPage) { return { status: 'failed', error: 'Next.js error page detected' }; } return { status: 'passed' }; }
