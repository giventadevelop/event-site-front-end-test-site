What is the project goal?

We want to refresh the homepage so it looks more modern and professional, but we still follow the organization’s existing design rules (colors, type, and layout standards in the codebase). We also want the site to quietly show who the community is: Malayalees and Keralites living in the United States. That second part needs research and leadership input before we lock in exact words and pictures.

Framer Motion and 21st.dev — do you need an account?

Framer Motion (in your Next.js app)

Framer Motion is an open-source React animation library published on npm (see framer.com/motion). You do not register on a website or create an API key to use it in code. You install it in the repo (for example npm install framer-motion), read the official docs, and copy or adapt examples into your own components—typically inside files marked 'use client' because animation hooks run in the browser. Licensing follows the library’s open-source terms; there is no separate “Framer Motion login” for the package itself.

21st.dev — two different ways to use it

Browsing the site (inspiration / copy snippets) — You can usually open 21st.dev in the browser, browse components, and copy code by hand without setting up an API key. Anything that is clearly “log in to see this” or “Pro” follows their product rules on the site.

Magic MCP, CLI, or other integrated flows — If you wire 21st.dev into Cursor via MCP or use their official CLI installers, you typically need a 21st.dev account and an API key (often created in their Magic Console). Treat that key like any secret: local MCP config only, not committed to git.

Optional: clone or scrape a website manually (wget)

Sometimes you like a site’s layout or structure and want a local copy to study in the browser or folder, without using a paid “clone” service. A common free tool is GNU wget. This is not a substitute for rebuilding your real site in React/Next.js—it is mainly for reference (HTML, CSS, assets) while you redesign.

Legal and respectful use. Only mirror sites you own, have permission to copy, or are allowed to fetch under their terms. Mass downloading can burden servers, break terms of service, and raise copyright issues. Using -e robots=off ignores robots rules—use it only when you understand the risk and have a legitimate reason (for example your own staging site). Do not republish someone else’s site as your own.

Install wget on Windows (Git Bash)

  1. Download a Windows build of wget.exe (many teams use the build from Eternally Bored).
  2. Copy wget.exe into your Git for Windows folder, for example C:\Program Files\Git\usr\bin (same place as other Unix tools).
  3. Close and reopen Git Bash, then run wget --version to confirm it works.

First commands to try (full mirror, single line for Git Bash)

Run from the folder where you want the downloaded site folder to appear. A full mirror can take a long time; the terminal may look idle—wait until wget finishes. If you prefer line breaks in Bash, you can split with \ at the end of each line; below uses one line for easy copy-paste.

Example 1 — mosc.in (with a common desktop user-agent):

wget --mirror --convert-links --adjust-extension --page-requisites --no-parent --user-agent="Mozilla/5.0 (Windows NT 10.0; Win64; x64)" https://mosc.in

Example 2 — directory.mosc.in:

wget --mirror --convert-links --adjust-extension --page-requisites --no-parent --user-agent="Mozilla/5.0 (Windows NT 10.0; Win64; x64)" https://directory.mosc.in

Simpler variants (no custom user-agent):

wget --mirror --convert-links --adjust-extension --page-requisites --no-parent https://mosc.in wget --mirror --convert-links --adjust-extension --page-requisites --no-parent https://www.syromalabarchurch.in

Why did wget only download one file?

  • The page may use nofollow or other hints so wget does not follow links.
  • robots.txt may block crawlers; wget respects that by default.

Possible fix: add -e robots=off so wget does not stop for robots rules. Only do this when it is appropriate for the target site and your situation.

Heavier “full mirror” example (single line—replace the URL and --domains= list with your real host and any CDNs you need, e.g. Google Fonts):

wget --mirror --page-requisites --convert-links --adjust-extension --no-parent --span-hosts -e robots=off --domains=reputable-scarab-5b76b9.qwery-test.themerex.net,fonts.googleapis.com,fonts.gstatic.com --reject-regex='wp-json|xmlrpc\.php|feed|comments/feed|ical=1' "https://reputable-scarab-5b76b9.qwery-test.themerex.net/"

Lighter pass (roughly the home page plus one level of links and assets—still single line; adjust URL and domains):

wget -r -l1 -H --page-requisites --convert-links --adjust-extension --no-parent -e robots=off --domains=reputable-scarab-5b76b9.qwery-test.themerex.net,fonts.googleapis.com,fonts.gstatic.com "https://reputable-scarab-5b76b9.qwery-test.themerex.net/"

Check how many files were saved (Git Bash)

Wget usually creates a folder named like the site’s host. Replace the path with your folder name.

find reputable-scarab-5b76b9.qwery-test.themerex.net -type f | wc -l

Force a clean re-download (optional)

rm -rf reputable-scarab-5b76b9.qwery-test.themerex.net

Then cd to the directory where you want the mirror (example: cd /e/project_workspace/your-folder) and run your wget command again from that folder.
