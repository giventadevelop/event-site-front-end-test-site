Banner Image Specification for Sponsors — Summary

Documentation folder: Banner_Image_Specification_for_Sponsors

1. Rule used for global sponsor images (banner/logo) on the home screen

Display rule: Image Containment Prevention (.cursor/rules/image_containment_prevention_hero_images_cards.mdc).

  • It applies to “banner images (sponsor banners, event banners)” and hero/logo images that must stay fully visible.
  • Pattern: container relative w-full h-auto rounded-t-2xl overflow-hidden; image w-full h-auto object-contain with flexible height so nothing is cropped.

Where sponsor images appear on the home screen:

  • Home page: OurSponsorsSection → SponsorCard shows the sponsor banner via sponsor.bannerImageUrl (no logo-specific “hero” on the main homepage hero).
  • SponsorCard (src/components/sponsors/SponsorCard.tsx): Renders the banner with <Image width={800} height={600} className="w-full h-auto object-contain ..." /> inside a relative w-full h-auto rounded-t-2xl overflow-hidden container — i.e. it follows the containment rule (no cropping, aspect ratio preserved).

Banner hero and “logo” on the home screen:

  • Main hero (rotating slideshow): Comes from event/standalone hero media, not from sponsors. Rule: hero_section_image_rotation.mdc (and related hero specs). Recommended: 1200×800 px (3:2); no sponsor-specific size.
  • Left-panel “logo”/banner (e.g. Kerala / MALAYALEES.US): Hero Section 1 image; documented in MediaImageSpecHelpContent and docs/hero-section-1-image-resize-prompt.md (e.g. 1000×1200, 5:6). Not sponsor-specific.

So: global sponsor images on the home screen (in Our Sponsors) use the image containment rule for how they’re displayed; they are not part of the banner hero or logo rules, which apply to hero/event imagery.

2. Is there any document for “banner image – size and dimensions”?

Hero / event banner sizes: Yes. Examples:

  • documentation/HERO_SECTION_IMAGE_SPECIFICATIONS.md — hero (e.g. 1200×800), logo (e.g. 240×240), Unite India, buttons; charity-theme hero.
  • hero_section_image_rotation.mdc — hero 1200×800 (3:2), “Keep admin/upload copy and docs (e.g. HERO_SECTION_IMAGE_SPECIFICATIONS.md, MediaClientPage hero tip) aligned with 1200×800 (3:2).”
  • MediaImageSpecHelpContent + docs/hero-section-1-image-resize-prompt.md — Section 1 (left panel), Section 2 (slideshow), Section 3 (Mission card), Featured event image.

Sponsor banner image – size and dimensions: No. There is no dedicated doc that specifies recommended size/dimensions (e.g. pixels or aspect ratio) for sponsor banner images. SPONSOR_IMAGE_UPLOAD_ANALYSIS.md and event_sponsors docs describe upload flow, S3 paths, and types (logo/hero/banner) but do not define recommended dimensions or aspect ratios for sponsor banners.

3. Is there any rule specified for the banner image in the sponsor?

Display (how to show, no crop): Yes. The image containment rule explicitly includes “Banner images (sponsor banners, event banners)” and points to:

  • src/app/events/[id]/page.tsx (event page sponsor banner images),
  • src/components/OurSponsorsSection.tsx (home sponsor section images),

with the pattern: flexible height container + object-contain + w-full h-auto.

Sponsor banner – size/dimensions rule: No. No cursor rule or doc specifies recommended pixel size or aspect ratio for sponsor banner uploads. MediaImageSpecHelpContent covers hero sections 1–3 and featured event image only; it does not include a “sponsor banner” size/dimension section.
