1. Current State Analysis

1.1 Understanding the Date Display

✅ Current Behavior is CORRECT: The dates shown (12/5/2025 to 1/4/2026) represent the first billing period of a monthly subscription. This is the expected behavior for Stripe subscriptions.

What's happening:

  • When a user subscribes, Stripe creates a subscription with a billing period (e.g., monthly)
  • The currentPeriodStart and currentPeriodEnd fields in the database represent the current active billing period
  • For a monthly subscription starting on 12/5/2025, the first period ends on 1/4/2026 (30 days)
  • Stripe automatically handles recurring billing - it will charge the customer again on 1/4/2026

1.2 Current Implementation Flow

Step 1: User subscribes via /membership/subscribe/[planId]
Step 2: Frontend creates Stripe Checkout Session (mode: 'subscription')
Step 3: User completes payment on Stripe Checkout
Step 4: User redirected to /membership/success
Step 5: Frontend calls processMembershipSubscriptionSessionServer()
Step 6: Subscription created in database with:
  • currentPeriodStart = Today's date
  • currentPeriodEnd = Today + billing interval (e.g., 30 days for monthly)
  • stripeSubscriptionId = Stripe subscription ID
  • subscriptionStatus = 'ACTIVE' or 'TRIAL'

2. Identified Issues

2.1 Issue #1: Recurring Payment Period Updates

⚠️ PROBLEM: When Stripe automatically charges the customer for the next billing period (e.g., on 1/4/2026), the currentPeriodStart and currentPeriodEnd dates in the database are NOT automatically updated.

Expected Behavior:

  • On 1/4/2026, Stripe charges the customer automatically
  • Stripe sends invoice.payment_succeeded webhook
  • Backend should update:
    • currentPeriodStart = 1/4/2026
    • currentPeriodEnd = 2/3/2026 (next month)
    • subscriptionStatus = 'ACTIVE'

Current State:

  • ❌ No webhook handler for invoice.payment_succeeded
  • ❌ Dates remain at initial values (12/5/2025 to 1/4/2026)
  • ❌ Admin page shows outdated period dates

❌ PROBLEM: No email is sent to the subscriber after successful subscription creation or recurring payment.

Expected Behavior:

  • After initial subscription: Send welcome email with subscription details
  • After recurring payment: Send payment confirmation email
  • On payment failure: Send payment failure notification
  • On cancellation: Send cancellation confirmation

Current State:

  • ❌ No email sending logic in subscription creation flow
  • ❌ No email template for subscription confirmations
  • ❌ No email sending in webhook handlers

3.2 Webhook Handler Implementation Details

3.2.1 invoice.payment_succeeded Handler

@PostMapping("/api/webhooks/stripe")
public ResponseEntity<String> handleStripeWebhook(
    @RequestBody String payload,
    @RequestHeader("Stripe-Signature") String sigHeader
) {
    Event event = Webhook.constructEvent(payload, sigHeader, webhookSecret);

    if ("invoice.payment_succeeded".equals(event.getType())) {
        Invoice invoice = (Invoice) event.getDataObjectDeserializer()
            .getObject().orElse(null);

        if (invoice != null && invoice.getSubscription() != null) {
            handleInvoicePaymentSucceeded(invoice);
        }
    }

    return ResponseEntity.ok("Webhook processed");
}

private void handleInvoicePaymentSucceeded(Invoice invoice) {
    String stripeSubscriptionId = invoice.getSubscription();

    // Find subscription by Stripe subscription ID
    MembershipSubscription subscription = membershipSubscriptionRepository
        .findByStripeSubscriptionId(stripeSubscriptionId)
        .orElse(null);

    if (subscription == null) {
        log.warn("Subscription not found for Stripe subscription: {}", stripeSubscriptionId);
        return;
    }

    // Retrieve updated subscription from Stripe
    Subscription stripeSubscription = stripeService
        .retrieveSubscription(stripeSubscriptionId);

    // Update period dates from Stripe subscription
    subscription.setCurrentPeriodStart(
        LocalDate.ofEpochDay(stripeSubscription.getCurrentPeriodStart())
    );
    subscription.setCurrentPeriodEnd(
        LocalDate.ofEpochDay(stripeSubscription.getCurrentPeriodEnd())
    );
    subscription.setSubscriptionStatus("ACTIVE");
    subscription.setUpdatedAt(LocalDateTime.now());

    membershipSubscriptionRepository.save(subscription);

    // Send payment confirmation email
    emailService.sendSubscriptionPaymentConfirmation(
        subscription.getUserProfile(),
        subscription,
        invoice.getAmountPaid()
    );

    log.info("Updated subscription {} for recurring payment", subscription.getId());
}

3.2.2 invoice.payment_failed Handler

private void handleInvoicePaymentFailed(Invoice invoice) {
    String stripeSubscriptionId = invoice.getSubscription();

    MembershipSubscription subscription = membershipSubscriptionRepository
        .findByStripeSubscriptionId(stripeSubscriptionId)
        .orElse(null);

    if (subscription == null) {
        return;
    }

    // Update status to PAST_DUE
    subscription.setSubscriptionStatus("PAST_DUE");
    subscription.setUpdatedAt(LocalDateTime.now());
    membershipSubscriptionRepository.save(subscription);

    // Send payment failure email
    emailService.sendSubscriptionPaymentFailure(
        subscription.getUserProfile(),
        subscription,
        invoice.getAmountDue()
    );

    log.warn("Subscription {} payment failed", subscription.getId());
}

async function sendSubscriptionWelcomeEmail(params: {
  subscriptionId: number;
  userEmail: string;
  planName: string;
  amount: number;
  currency: string;
  periodStart: string;
  periodEnd: string;
}) {
  try {
    const response = await fetch('/api/proxy/membership-subscriptions/send-welcome-email', {
      method: 'POST',
      headers: { 'Content-Type': 'application/json' },
      body: JSON.stringify(params)
    });

    if (response.ok) {
      console.log('[MEMBERSHIP-SUCCESS] Welcome email sent');
    }
  } catch (error) {
    console.error('[MEMBERSHIP-SUCCESS] Failed to send welcome email:', error);
  }
}

10. Error Handling & Edge Cases

10.1 Webhook Processing Errors

  • Subscription not found: Log warning and return 200 (idempotent)
  • Stripe API failure: Retry with exponential backoff
  • Database update failure: Log error and return 500 (Stripe will retry)
  • Email sending failure: Log error but don't fail webhook (non-critical)

10.2 Edge Cases

  • Duplicate webhook events: Make handlers idempotent
  • Out-of-order webhooks: Use timestamps to handle correctly
  • Subscription already cancelled: Skip processing
  • Missing user profile: Log error and skip email
