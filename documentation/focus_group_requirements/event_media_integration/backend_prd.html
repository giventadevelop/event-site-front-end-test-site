PRD: Focus Group – Event Media Integration (Backend)

Requirement

Add optional event_focus_group_id to the event_media table and expose it in REST and DTOs so the frontend can associate media with an event’s focus group when uploading or editing. The canonical schema change is already applied in the shared SQL file; the backend must apply an equivalent migration (e.g. Liquibase) and update entity, DTO, criteria, mapper, and REST layer.

Schema / Migration

  • Column: event_focus_group_id bigint NULL
  • FK: REFERENCES public.event_focus_groups(id) ON DELETE SET NULL
  • Index: idx_event_media_event_focus_group_id on event_media(event_focus_group_id) WHERE event_focus_group_id IS NOT NULL
  • Add a Liquibase changelog (or equivalent) for the new column and constraint. Reference the canonical file: code_html_template/SQLS/Current_Sqls/Latest_Schema_Post__Blob_Claude_12.sql (in the frontend repo or sync from it).

Entity

In EventMedia.java: add eventFocusGroupId (Long, nullable) mapped to column event_focus_group_id. Optionally add @ManyToOne(fetch = LAZY) EventFocusGroup eventFocusGroup with @JoinColumn(name = "event_focus_group_id") for navigation.

DTO

In EventMediaDTO: add eventFocusGroupId (Long, nullable). Optionally include a nested EventFocusGroupDTO for responses when the client needs focus group details with the media.

Criteria

In EventMediaCriteria: add filter(s) such as eventFocusGroupId.equals (and eventFocusGroupId.in if needed) so the frontend can query media by focus group.

Mapper

In EventMediaMapper: map the new field between entity and DTO (and nested EventFocusGroup if present).

REST

No URL change. Existing EventMediaResource and query service should persist and return the new field. Regenerate or update Swagger / api-docs to include eventFocusGroupId on the EventMedia schema.

Validation

When eventFocusGroupId is set, ensure the referenced event_focus_groups row’s event_id matches the media’s event_id (service-level check or DB constraint). Reject or correct invalid combinations to keep data consistent.

Backward compatibility: The field is optional. Existing API clients that do not send or expect eventFocusGroupId remain unchanged.
