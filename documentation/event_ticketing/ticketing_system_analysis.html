2. Current QR Scanning Implementation Analysis

2.1 Current Implementation

Location: /admin/qrcode-scan/tickets/events/[eventId]/transactions/[transactionId]

Current Features:

  • QR Code Generation: ✅ Implemented - QR codes are generated server-side and stored in qrCodeImageUrl field
  • QR Code Display: ✅ Implemented - QR codes are displayed on ticket success pages (desktop and mobile)
  • Manual Check-In: ✅ Implemented - Admin can manually check in attendees via form
  • Check-In Status Tracking: ✅ Implemented - Tracks checkInStatus, checkInTime, numberOfGuestsCheckedIn
  • QR Code Validation Page: ✅ Implemented - Page displays ticket details when QR code is scanned

Current Limitations:

  • ❌ No Camera-Based Scanning: Cannot scan QR codes using device camera
  • ❌ No Mobile Scanner App: No dedicated mobile app for scanning
  • ❌ No Browser-Based Scanner: Cannot use web camera to scan QR codes
  • ⚠️ Manual Process: Requires admin to manually enter transaction ID or copy-paste QR code data
  • ⚠️ No Batch Scanning: Cannot scan multiple tickets quickly

2.2 Current Check-In Flow

1 QR Code Generation: When a ticket is purchased, a QR code is generated and stored in the database
2 QR Code Display: QR code is shown to the attendee on the success page and sent via email
3 Manual Check-In: Admin navigates to /admin/qrcode-scan/tickets/events/[eventId]/transactions/[transactionId] and manually enters transaction details
4 Status Update: Admin uses the check-in form to update checkInStatus, numberOfGuestsCheckedIn, and checkInTime

3. Proposed QR Scanning Solution

3.1 Recommended Approach: Browser-Based QR Scanner

Recommended Library: react-qr-reader or @zxing/library with react-qr-scanner

Why Browser-Based?
  • ✅ No need for native mobile app development
  • ✅ Works on both desktop (webcam) and mobile (camera)
  • ✅ Faster to implement than native app
  • ✅ Cross-platform compatibility
  • ✅ Easy to update and maintain

// src/app/api/qr/verify/route.ts import { NextRequest, NextResponse } from 'next/server'; import { fetchWithJwtRetry } from '@/lib/proxyHandler'; export async function GET(req: NextRequest) { const { searchParams } = new URL(req.url); const qrData = searchParams.get('data'); if (!qrData) { return NextResponse.json({ valid: false, error: 'No QR data provided' }, { status: 400 }); } try { // Parse QR data (could be transaction ID, URL, or encoded JSON) // Example: Extract transaction ID from QR code const transactionId = extractTransactionId(qrData); if (!transactionId) { return NextResponse.json({ valid: false, error: 'Invalid QR code format' }, { status: 400 }); } // Fetch transaction details const API_BASE_URL = process.env.NEXT_PUBLIC_API_BASE_URL; const response = await fetchWithJwtRetry( `${API_BASE_URL}/api/event-ticket-transactions/${transactionId}`, { cache: 'no-store' } ); if (!response.ok) { return NextResponse.json({ valid: false, error: 'Ticket not found' }, { status: 404 }); } const transaction = await response.json(); // Validate ticket const isValid = validateTicket(transaction); return NextResponse.json({ valid: isValid, transaction: isValid ? transaction : null, message: isValid ? 'Ticket valid - Access granted' : 'Invalid ticket' }); } catch (error) { console.error('QR Verification Error:', error); return NextResponse.json({ valid: false, error: 'Verification failed' }, { status: 500 }); } } function extractTransactionId(qrData: string): string | null { // Handle different QR code formats: // 1. Direct transaction ID // 2. URL with transaction ID (e.g., /admin/qrcode-scan/.../transactions/123) // 3. Encoded JSON with transaction ID // Try to extract from URL const urlMatch = qrData.match(/transactions\/(\d+)/); if (urlMatch) return urlMatch[1]; // Try to parse as JSON try { const parsed = JSON.parse(qrData); return parsed.transactionId || parsed.id || null; } catch { // Not JSON, try as direct ID return /^\d+$/.test(qrData) ? qrData : null; } } function validateTicket(transaction: any): boolean { // Check if ticket is valid for check-in if (!transaction) return false; if (transaction.status === 'REFUNDED') return false; if (transaction.status === 'CANCELLED') return false; // Add more validation as needed return true; }

3.3 QR Code Format Standardization

Recommendation: Standardize QR code data format to include:
  • Transaction ID
  • Event ID
  • Verification token (optional, for security)
Format Options:
  1. URL Format: https://yourdomain.com/admin/qrcode-scan/tickets/events/{eventId}/transactions/{transactionId}
  2. JSON Format: {"transactionId": 123, "eventId": 456, "token": "abc123"}
  3. Encoded Format: Base64 encoded JSON
