2. REST API Endpoints

Base path suggestion: /api/news-articles, /api/news-categories, /api/news-section-config, /api/news-sidebar-promotions, /api/news-flash, /api/news-live-stream-config. Use JHipster/Spring Data REST style criteria where applicable (e.g. tenantId.equals=, slug.equals=, categoryId.equals=). Align request/response shapes with api-docs.json and frontend DTOs.

News Articles

  • GET /api/news-articles – List (criteria: tenantId, category slug/id, status, page, size, sort).
  • GET /api/news-articles/{id} – By ID.
  • GET /api/news-articles/by-slug/{slug} – By slug (and tenant).
  • POST /api/news-articles – Create (body includes tenantId).
  • PATCH /api/news-articles/{id} – Update (merge-patch).
  • DELETE /api/news-articles/{id} – Delete.

News Categories

  • GET /api/news-categories – List (tenantId, isActive, sort).
  • GET /api/news-categories/{id} – By ID.
  • GET /api/news-categories/by-slug/{slug} – By slug (and tenant).
  • POST /api/news-categories – Create.
  • PATCH /api/news-categories/{id} – Update.
  • DELETE /api/news-categories/{id} – Delete.

News Section Display Config

  • GET /api/news-section-config – List by tenantId (and optionally sectionKey).
  • GET /api/news-section-config/{id} – By ID.
  • POST /api/news-section-config – Create.
  • PATCH /api/news-section-config/{id} – Update (e.g. banner_image_url, section_title_override).

News Sidebar Promotions (Rotating Banners / Slideshow)

  • GET /api/news-sidebar-promotions – List by tenantId (criteria: isActive.equals, valid date range). Return ordered by display_order for slideshow. Public read for active promotions.
  • GET /api/news-sidebar-promotions/{id} – By ID.
  • POST /api/news-sidebar-promotions – Create (body: tenantId, title, image_url, link_url, display_order, display_duration_seconds, valid_from, valid_to, is_active).
  • PATCH /api/news-sidebar-promotions/{id} – Update (merge-patch).
  • DELETE /api/news-sidebar-promotions/{id} – Delete.

Front end uses list response to render a slideshow with timer: each slide shown for display_duration_seconds seconds before advancing to the next.

News Flash

  • GET /api/news-flash – List (tenantId, isActive, valid date range).
  • GET /api/news-flash/{id} – By ID.
  • POST /api/news-flash – Create.
  • PATCH /api/news-flash/{id} – Update.
  • DELETE /api/news-flash/{id} – Delete.

3. Entities & DTOs

Implement JPA entities for each table; provide REST DTOs (or use OpenAPI-generated) matching the frontend @/types. Include tenantId in create/update payloads. PATCH must accept Content-Type: application/merge-patch+json and require id in body where applicable.

4. Security & Multi-Tenancy

  • All endpoints must enforce tenant scope (tenantId from JWT or context).
  • Public read (list published articles, section config, active sidebar promotions for slideshow, active flash, current live stream config) can allow unauthenticated access with tenant resolved from request (e.g. header or default); create/update/delete require authenticated user with appropriate role (e.g. ADMIN or NEWS_EDITOR).
  • Align with existing backend JWT and role checks used by the frontend proxy.

5. Batch Jobs (Optional)

In event-site-manager-batch-jobs: optional jobs for scheduled publish/unpublish of articles (e.g. set status to PUBLISHED at published_at, or ARCHIVED after valid_to). Not required for MVP; can be added in a later phase.
