Overview

This guide walks you through the complete process of setting up Givebutter for development and integration with the MCEFEE platform. By the end of this guide, you will have:

  • A Givebutter account with API access
  • API keys for development and production
  • Test campaigns configured
  • Webhooks set up for donation callbacks
  • A working development environment
  • Test scenarios for validation

Prerequisites

  • Organization with US bank account (for production)
  • US EIN or SSN (for production account verification)
  • Email address for account creation
  • Access to Givebutter dashboard
  • Development environment ready (Spring Boot backend, Next.js frontend)

⚠️ Important: Givebutter Uses a Unified Environment

Givebutter does NOT have separate development and production environments. The same API keys and account are used for both testing and live operations.

Key Points:

  • ✅ Same API keys work for both test and production
  • ✅ Same webhooks can be configured for different URLs (dev/staging/prod)
  • ⚠️ Test transactions appear in the same dashboard as real transactions
  • ⚠️ Use Stripe test card numbers for testing (Givebutter uses Stripe for processing)
  • ✅ Create separate webhooks pointing to different environments
  • ✅ Use clear naming conventions to distinguish test campaigns

Testing Strategy: Use Stripe test cards and create separate webhooks for each environment URL.

Step 1: Create Givebutter Account

1.1 Visit Givebutter Sign-Up Page

URL: https://givebutter.com/signup

You'll see three account type options:

  • Nonprofit: For tax-exempt organizations with EIN (recommended for churches/parishes)
  • Other: For other types of organizations
  • Individual: Cannot create fundraising accounts (as of Dec 2022)

1.2 Choose Account Type

For Church/Parish Organizations:

Select "Nonprofit" if your organization is:

  • Registered as a 501(c)(3) tax-exempt organization
  • A church or religious organization
  • Has an EIN (Employer Identification Number)

Select "Other" if:

  • You don't have tax-exempt status yet
  • You're setting up a test/development account
  • You'll upgrade to nonprofit later

1.3 Provide Organization Details

Fill in the following information:

  • Organization Name: Your church/parish name (e.g., "MOSC Parish")
  • Website: Your organization's website URL (or social media profile if no website)
  • Organization Type: Select appropriate category
  • Location: Your organization's location

⚠️ Important for Development

For development/testing purposes, you can use:

  • Test organization name: "MOSC Development Test"
  • Test website: Your development URL or placeholder
  • Note: Production account will need real organization details

1.4 Set Up Personal Profile

Create your login credentials:

  • Name: Your full name
  • Email: Use a professional email (this will be your login)
  • Password: Create a strong password
Example: Name: John Doe Email: developer@mosc-parish.org Password: [Strong password with 8+ characters]

1.5 Complete Registration

  1. Review all information
  2. Accept Givebutter's terms of service
  3. Click "Sign Up"
  4. You'll be redirected to your Givebutter dashboard

✅ Account Created Successfully

You should now see the Givebutter dashboard. If you selected "Nonprofit", you may need to verify your tax-exempt status later.

1.6 Verify Email (If Required)

Check your email inbox for a verification email from Givebutter:

  • Click the verification link in the email
  • This activates your account

Step 2: Generate API Keys

2.1 Navigate to API Settings

  1. Log in to your Givebutter dashboard
  2. Click on "Settings" in the top navigation
  3. In the left sidebar, click "Integrations"
  4. Click on "API" tab
[Screenshot: Settings → Integrations → API navigation path]

2.2 Create API Key

  1. Click "New API Key" button
  2. Enter a descriptive name for your API key:
    • Example: "MCEFEE Development" or "MOSC Production"
    • Use naming convention: [Environment]_[Purpose]
  3. Click "Create"

⚠️ CRITICAL: Save API Key Immediately

The API key is displayed only once! After you close the dialog, you cannot retrieve it again.

  • ✅ Copy the API key immediately
  • ✅ Store it securely (password manager, encrypted file)
  • ✅ Add to your environment variables
  • ❌ Never commit API keys to version control

2.3 API Key Format

Givebutter API keys typically look like:

gb_live_xxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxx or gb_test_xxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxx
  • gb_live_: Production API key (real transactions)
  • gb_test_: Test/sandbox API key (if available)

2.4 Create Multiple API Keys (Recommended)

Note: While Givebutter uses a unified environment, creating separate API keys helps with tracking and security:

Environment API Key Name Purpose Note
Development MCEFEE_DEV Local development and testing Same account, different key for tracking
Staging MCEFEE_STAGING Staging environment testing Same account, different key for tracking
Production MCEFEE_PROD Live production transactions Same account, different key for tracking

Best Practice

Use different API keys for each environment to:

  • Track usage per environment
  • Revoke access if compromised
  • Monitor API calls separately
  • Maintain security isolation

Important: All keys access the same Givebutter account and data. The separation is for organizational purposes only.

2.5 Store API Keys Securely

Add to your environment variables:

# .env.local (for Next.js frontend) NEXT_PUBLIC_GIVEBUTTER_API_KEY=gb_live_xxx # Only if needed in frontend # Note: Usually API keys should be server-side only # .env (for Spring Boot backend) GIVEBUTTER_API_KEY=gb_live_xxx GIVEBUTTER_WEBHOOK_SECRET=whsec_xxx

⚠️ Security Best Practices

  • ❌ Never expose API keys in frontend code
  • ❌ Never commit API keys to Git
  • ✅ Use environment variables
  • ✅ Encrypt API keys in database (config_data field)
  • ✅ Rotate API keys periodically
  • ✅ Use different keys for dev/staging/production

Step 4: Configure Webhooks - Backend-Driven Architecture

✅ Backend-Driven Webhook Pattern (Matching Stripe)

Webhooks are handled entirely in the backend Spring Boot application:

  • ✅ Webhook URL points to backend - Not Next.js frontend
  • ✅ Webhook secret stored in database - Per tenant in payment_provider_config
  • ✅ Dynamic secret retrieval - Retrieved by tenant_id at runtime
  • ✅ Signature verification - Uses webhook secret from database
  • ✅ Multi-tenant support - Each tenant can have different webhook secrets

Architecture: Givebutter → Backend Spring Boot → Database (webhook secret) → Process event

4.1 Navigate to Webhook Settings

  1. In Givebutter dashboard, go to Settings
  2. Click Integrations in left sidebar

4.2 Webhook URL Configuration - Backend Endpoints

Webhook URLs should point directly to your backend Spring Boot application:

  • ✅ Backend endpoint: POST /api/webhooks/givebutter (Spring Boot)
  • ❌ NOT frontend: /api/webhooks/givebutter (Next.js) - Wrong!
  • ✅ Full URL format: https://your-backend-domain.com/api/webhooks/givebutter

4.3 Get Webhook Secret and Store in Database

After creating the webhook in Givebutter dashboard:

  1. Givebutter will generate a webhook secret
  2. Copy the secret immediately (shown only once)
  3. Store it in database - NOT in environment variables (multi-tenant pattern)

✅ Store Webhook Secret in Database (Per Tenant)

Webhook secret is stored in payment_provider_config table, encrypted:

Update payment_provider_config with Webhook Secret

-- Update existing Givebutter config with webhook secret UPDATE payment_provider_config SET webhook_secret = encrypt('whsec_xxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxx') WHERE tenant_id = 'tenant_demo_001' AND provider_type = 'GIVEBUTTER' AND metadata->>'useCase' = 'OFFERING'; -- Or when inserting new config: INSERT INTO payment_provider_config ( tenant_id, provider_type, provider_name, is_active, is_primary, priority, config_data, -- Encrypted: {"api_key": "gb_live_xxx"} webhook_secret, -- Encrypted: Givebutter webhook secret supports_wallets, supports_subscriptions, supports_refunds, metadata ) VALUES ( 'tenant_demo_001', 'GIVEBUTTER', 'Givebutter Mass Offerings', true, false, 1, encrypt('{"api_key": "gb_live_xxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxx"}'), encrypt('whsec_xxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxx'), -- Webhook secret from Givebutter true, true, true, '{ "supportsZeroFee": true, "useCase": "OFFERING", "apiBaseUrl": "https://api.givebutter.com/v1", "campaignId": "campaign_123abc", "webhookUrl": "https://api.yourdomain.com/api/webhooks/givebutter" }'::jsonb );

⚠️ Do NOT Store Webhook Secret in Environment Variables

For multi-tenant architecture, webhook secrets must be in database:

  • ❌ GIVEBUTTER_WEBHOOK_SECRET=whsec_xxx in .env - WRONG (single tenant)
  • ✅ Store in payment_provider_config.webhook_secret - CORRECT (multi-tenant)
  • ✅ Encrypt before storing
  • ✅ Retrieve dynamically by tenant_id at runtime

✅ Webhook Secret Security

The webhook secret is used to verify that webhook requests are actually from Givebutter. The backend handler will:

  1. Receive webhook request
  2. Extract tenant_id from webhook payload or URL
  3. Retrieve webhook secret from database (per tenant)
  4. Decrypt webhook secret
  5. Verify signature using decrypted secret
  6. Process webhook if signature is valid

4.4 Backend Webhook Handler Implementation

Create webhook handler in Spring Boot backend (matching Stripe pattern):

GivebutterWebhookController.java

// GivebutterWebhookController.java @RestController @RequestMapping("/api/webhooks/givebutter") @RequiredArgsConstructor @Slf4j public class GivebutterWebhookController { private final PaymentProviderConfigRepository configRepository; private final EncryptionService encryptionService; private final PaymentOrchestrationService orchestrationService; private final ObjectMapper objectMapper; @PostMapping public ResponseEntity<?> handleWebhook( @RequestHeader(value = "X-Givebutter-Signature", required = false) String signature, @RequestBody String payload, HttpServletRequest request ) { try { log.info("Received Givebutter webhook: {}", payload); // 1. Parse webhook payload to extract tenant_id GivebutterWebhookEvent event = objectMapper.readValue( payload, GivebutterWebhookEvent.class ); // Extract tenant_id from donation metadata or campaign String tenantId = extractTenantId(event); if (tenantId == null) { log.error("Cannot determine tenant_id from webhook payload"); return ResponseEntity.status(400).body("Missing tenant_id"); } // 2. Retrieve webhook secret from database (per tenant) PaymentProviderConfig config = configRepository .findByTenantIdAndProviderType(tenantId, ProviderType.GIVEBUTTER) .orElse(null); if (config == null) { log.error("Givebutter not configured for tenant: {}", tenantId); return ResponseEntity.status(404).body("Tenant not configured"); } // 3. Decrypt webhook secret String webhookSecret = decryptWebhookSecret(config); if (webhookSecret == null) { log.error("Webhook secret not configured for tenant: {}", tenantId); return ResponseEntity.status(500).body("Webhook secret not configured"); } // 4. Verify webhook signature if (!verifySignature(payload, signature, webhookSecret)) { log.error("Invalid webhook signature for tenant: {}", tenantId); return ResponseEntity.status(401).body("Invalid signature"); } log.info("Webhook signature verified for tenant: {}", tenantId); // 5. Process webhook event orchestrationService.processGivebutterWebhook(event, tenantId); return ResponseEntity.ok().build(); } catch (Exception e) { log.error("Error processing Givebutter webhook", e); return ResponseEntity.status(500).body("Webhook processing failed"); } } /** * Extract tenant_id from webhook payload. * Givebutter webhook includes metadata with tenant_id. */ private String extractTenantId(GivebutterWebhookEvent event) { // Option 1: From donation metadata if (event.getDonation() != null && event.getDonation().getMetadata() != null) { Map<String, Object> metadata = event.getDonation().getMetadata(); String tenantId = (String) metadata.get("tenant_id"); if (tenantId != null) { return tenantId; } } // Option 2: From campaign metadata (if stored) if (event.getCampaign() != null && event.getCampaign().getMetadata() != null) { Map<String, Object> metadata = event.getCampaign().getMetadata(); String tenantId = (String) metadata.get("tenant_id"); if (tenantId != null) { return tenantId; } } // Option 3: Lookup by campaign_id (if campaign_id is tenant-specific) if (event.getDonation() != null && event.getDonation().getCampaignId() != null) { // Query database to find tenant by campaign_id // This requires storing campaign_id -> tenant_id mapping return lookupTenantByCampaignId(event.getDonation().getCampaignId()); } return null; } /** * Decrypt webhook secret from encrypted webhook_secret field. */ private String decryptWebhookSecret(PaymentProviderConfig config) { String encryptedSecret = config.getWebhookSecret(); if (encryptedSecret == null) { return null; } return encryptionService.decrypt(encryptedSecret); } /** * Verify Givebutter webhook signature. * Givebutter uses HMAC-SHA256 signature verification. */ private boolean verifySignature(String payload, String signature, String secret) { if (signature == null || secret == null) { return false; } try { Mac mac = Mac.getInstance("HmacSHA256"); SecretKeySpec secretKey = new SecretKeySpec( secret.getBytes(StandardCharsets.UTF_8), "HmacSHA256" ); mac.init(secretKey); byte[] hash = mac.doFinal(payload.getBytes(StandardCharsets.UTF_8)); String computedSignature = "sha256=" + Hex.encodeHexString(hash); return MessageDigest.isEqual( signature.getBytes(StandardCharsets.UTF_8), computedSignature.getBytes(StandardCharsets.UTF_8) ); } catch (Exception e) { log.error("Error verifying webhook signature", e); return false; } } }

Webhook Event Processing

// PaymentOrchestrationService.java - Add Givebutter webhook processing public void processGivebutterWebhook(GivebutterWebhookEvent event, String tenantId) { log.info("Processing Givebutter webhook: type={}, tenant={}", event.getType(), tenantId); switch (event.getType()) { case "donation.completed": handleDonationCompleted(event, tenantId); break; case "donation.updated": handleDonationUpdated(event, tenantId); break; case "donation.refunded": handleDonationRefunded(event, tenantId); break; default: log.warn("Unhandled webhook event type: {}", event.getType()); } } private void handleDonationCompleted(GivebutterWebhookEvent event, String tenantId) { GivebutterDonation donation = event.getDonation(); // Find transaction by provider_transaction_id PaymentTransaction transaction = transactionRepository .findByProviderTransactionId(donation.getId()) .orElse(null); if (transaction == null) { log.warn("Transaction not found for donation: {}", donation.getId()); return; } // Update transaction transaction.setStatus(PaymentStatus.SUCCEEDED); transaction.setAmountTotal(donation.getAmount()); transaction.setAmountProcessingFee(BigDecimal.ZERO); // Zero fees transaction.setCompletedAt(LocalDateTime.now()); // Update metadata Map<String, Object> metadata = transaction.getMetadata(); if (metadata == null) { metadata = new HashMap<>(); } metadata.put("givebutter_donation_id", donation.getId()); metadata.put("givebutter_campaign_id", donation.getCampaignId()); if (donation.getCustomFields() != null) { metadata.put("prayer_intention", donation.getCustomFields().get("prayer_intention")); } transaction.setMetadata(metadata); transactionRepository.save(transaction); log.info("Updated transaction {} to SUCCEEDED", transaction.getId()); }

Step 5: Development Environment Setup

5.1 Backend Configuration (Spring Boot) - Multi-Tenant Database-Driven

✅ Multi-Tenant Architecture Pattern

Givebutter configuration follows the same pattern as Stripe:

  • ✅ No static configuration - No API keys in application.properties
  • ✅ Per-tenant configuration - Each tenant has their own Givebutter account/keys
  • ✅ Database-driven - Credentials stored in payment_provider_config table
  • ✅ Encrypted storage - API keys and webhook secrets encrypted in database
  • ✅ Dynamic retrieval - Adapter retrieves config by tenant_id at runtime
  • ✅ Domain-agnostic - All payment processing happens in backend
  • ✅ Use case routing - Separate configs for OFFERING vs DONATION_ZERO_FEE

// GivebutterPaymentAdapter.java @Service @RequiredArgsConstructor @Slf4j public class GivebutterPaymentAdapter implements PaymentAdapter { private final PaymentProviderConfigRepository configRepository; private final EncryptionService encryptionService; private final RestTemplate restTemplate; // Standard Spring RestTemplate @Override public PaymentSession initializePayment(PaymentInitializeRequest request) { try { // 1. Resolve tenant's Givebutter configuration dynamically PaymentProviderConfig config = resolveProviderConfig( request.getTenantId(), request.getPaymentType() // OFFERING or DONATION_ZERO_FEE ); if (config == null) { throw new PaymentException( "Givebutter not configured for tenant: " + request.getTenantId() ); } // 2. Decrypt API key from encrypted config_data String apiKey = decryptApiKey(config); String webhookSecret = decryptWebhookSecret(config); // 3. Extract Givebutter-specific config from metadata Map<String, Object> metadata = config.getMetadata(); String campaignId = (String) metadata.get("campaignId"); String apiBaseUrl = (String) metadata.getOrDefault("apiBaseUrl", "https://api.givebutter.com/v1"); // 4. Build Givebutter donation request GivebutterDonationRequest gbRequest = buildDonationRequest(request, campaignId); // 5. Call Givebutter API with tenant-specific credentials HttpHeaders headers = new HttpHeaders(); headers.set("Authorization", "Bearer " + apiKey); headers.setContentType(MediaType.APPLICATION_JSON); HttpEntity<GivebutterDonationRequest> entity = new HttpEntity<>(gbRequest, headers); ResponseEntity<GivebutterDonationResponse> response = restTemplate.postForEntity( apiBaseUrl + "/donations", entity, GivebutterDonationResponse.class ); if (!response.getStatusCode().is2xxSuccessful()) { throw new PaymentException( "Givebutter API error: " + response.getStatusCode() ); } // 6. Return payment session return PaymentSession.builder() .providerType(ProviderType.GIVEBUTTER) .providerTransactionId(response.getBody().getDonationId()) .sessionUrl(response.getBody().getCheckoutUrl()) .metadata(Map.of( "givebutter_donation_id", response.getBody().getDonationId(), "checkout_url", response.getBody().getCheckoutUrl(), "campaign_id", campaignId )) .build(); } catch (Exception e) { log.error("Givebutter payment initialization failed for tenant: {}", request.getTenantId(), e); throw new PaymentException("Failed to initialize Givebutter payment", e); } } /** * Resolve Givebutter configuration based on tenant and payment type. * Supports separate configs for OFFERING vs DONATION_ZERO_FEE. */ private PaymentProviderConfig resolveProviderConfig( String tenantId, PaymentType paymentType ) { // Determine use case based on payment type String useCase = null; if (PaymentType.OFFERING.equals(paymentType)) { useCase = "OFFERING"; } else if (PaymentType.DONATION.equals(paymentType)) { // Check if this is a zero-fee donation useCase = "DONATION_ZERO_FEE"; } // Query for Givebutter config matching tenant and use case List<PaymentProviderConfig> configs = configRepository .findByTenantIdAndProviderType(tenantId, ProviderType.GIVEBUTTER); if (useCase != null) { // Filter by use case in metadata return configs.stream() .filter(c -> { Map<String, Object> meta = c.getMetadata(); return useCase.equals(meta.get("useCase")); }) .findFirst() .orElse(null); } // Fallback: return first active Givebutter config return configs.stream() .filter(PaymentProviderConfig::getIsActive) .findFirst() .orElse(null); } /** * Decrypt API key from encrypted config_data field. * config_data is stored as encrypted JSON: {"api_key": "gb_live_xxx"} */ private String decryptApiKey(PaymentProviderConfig config) { String encryptedConfig = config.getConfigData(); String decryptedJson = encryptionService.decrypt(encryptedConfig); // Parse JSON to extract api_key ObjectMapper objectMapper = new ObjectMapper(); try { Map<String, String> configMap = objectMapper.readValue( decryptedJson, new TypeReference<Map<String, String>>() {} ); return configMap.get("api_key"); } catch (Exception e) { throw new PaymentException("Failed to decrypt Givebutter API key", e); } } /** * Decrypt webhook secret from encrypted webhook_secret field. */ private String decryptWebhookSecret(PaymentProviderConfig config) { String encryptedSecret = config.getWebhookSecret(); if (encryptedSecret == null) { return null; } return encryptionService.decrypt(encryptedSecret); } private GivebutterDonationRequest buildDonationRequest( PaymentInitializeRequest request, String campaignId ) { return GivebutterDonationRequest.builder() .amount(request.getAmount()) .campaignId(campaignId) .donorEmail(request.getCustomerEmail()) .donorName(request.getCustomerFirstName() + " " + request.getCustomerLastName()) .anonymous(Boolean.TRUE.equals(request.getMetadata().get("isAnonymous"))) .recurring(Boolean.TRUE.equals(request.getMetadata().get("isRecurring"))) .recurringFrequency((String) request.getMetadata().get("recurringFrequency")) .customFields(extractCustomFields(request)) .metadata(Map.of( "event_id", String.valueOf(request.getEventId()), "tenant_id", request.getTenantId(), "payment_type", request.getPaymentType().name() )) .build(); } }

// PaymentOrchestrationService.java @Service @RequiredArgsConstructor public class PaymentOrchestrationService { private final PaymentProviderFactory providerFactory; private final PaymentProviderConfigRepository configRepository; public PaymentSession initializePayment(PaymentInitializeRequest request) { // 1. Determine provider based on payment type and event config ProviderType providerType = determineProvider(request); // 2. Get adapter PaymentAdapter adapter = providerFactory.getAdapter(providerType); // 3. Initialize payment (adapter retrieves config dynamically) return adapter.initializePayment(request); } private ProviderType determineProvider(PaymentInitializeRequest request) { // Check if event is configured for zero-fee provider if (request.getEventId() != null) { EventDetails event = eventRepository.findById(request.getEventId()) .orElse(null); if (event != null) { Map<String, Object> metadata = event.getMetadata(); if (metadata != null) { Map<String, Object> donationConfig = (Map<String, Object>) metadata.get("donationConfig"); if (donationConfig != null) { // Check for zero-fee provider preference Boolean useZeroFeeProvider = (Boolean) donationConfig.get("useZeroFeeProvider"); if (Boolean.TRUE.equals(useZeroFeeProvider)) { String zeroFeeProvider = (String) donationConfig.get("zeroFeeProvider"); if ("GIVEBUTTER".equals(zeroFeeProvider)) { // Verify tenant has Givebutter configured boolean hasGivebutter = configRepository .existsByTenantIdAndProviderType( request.getTenantId(), ProviderType.GIVEBUTTER ); if (hasGivebutter) { return ProviderType.GIVEBUTTER; } } } } } } } // For OFFERING payment type, prefer Givebutter if configured if (PaymentType.OFFERING.equals(request.getPaymentType())) { boolean hasGivebutter = configRepository .existsByTenantIdAndProviderType( request.getTenantId(), ProviderType.GIVEBUTTER ); if (hasGivebutter) { return ProviderType.GIVEBUTTER; } } // Default to Stripe (primary provider for tickets) return ProviderType.STRIPE; } }

if (transaction == null) { log.warn("Transaction not found for donation: {}", donation.getId()); return; } // Update transaction transaction.setStatus(PaymentStatus.SUCCEEDED); transaction.setAmountTotal(donation.getAmount()); transaction.setAmountProcessingFee(BigDecimal.ZERO); // Zero fees transaction.setCompletedAt(LocalDateTime.now()); // Update metadata with donation details Map<String, Object> metadata = transaction.getMetadata(); if (metadata == null) { metadata = new HashMap<>(); } metadata.put("givebutter_donation_id", donation.getId()); metadata.put("givebutter_campaign_id", donation.getCampaignId()); if (donation.getCustomFields() != null) { metadata.put("prayer_intention", donation.getCustomFields().get("prayer_intention")); } transaction.setMetadata(metadata); transactionRepository.save(transaction); log.info("Updated transaction {} to SUCCEEDED", transaction.getId()); } private boolean verifySignature(String payload, String signature) { if (signature == null || givebutterConfig.getWebhookSecret() == null) { return false; } // Givebutter webhook signature verification // Implementation depends on Givebutter's signature algorithm // Typically HMAC-SHA256 try { Mac mac = Mac.getInstance("HmacSHA256"); SecretKeySpec secretKey = new SecretKeySpec( givebutterConfig.getWebhookSecret().getBytes(StandardCharsets.UTF_8), "HmacSHA256" ); mac.init(secretKey); byte[] hash = mac.doFinal(payload.getBytes(StandardCharsets.UTF_8)); String computedSignature = "sha256=" + Hex.encodeHexString(hash); return MessageDigest.isEqual( signature.getBytes(StandardCharsets.UTF_8), computedSignature.getBytes(StandardCharsets.UTF_8) ); } catch (Exception e) { log.error("Error verifying webhook signature", e); return false; } } }

Troubleshooting

Common Issues and Solutions

Issue 1: API Key Not Working

Symptoms:

  • 401 Unauthorized errors
  • "Invalid API key" messages

Solutions:

  • Verify API key is correct (no extra spaces)
  • Check if using correct environment (live vs test)
  • Ensure API key hasn't been revoked
  • Verify Authorization header format: Bearer gb_live_xxx

Issue 2: Webhook Not Receiving Events

Symptoms:

  • No webhook calls received
  • Webhook shows as "failed" in Givebutter dashboard
  • All webhooks receive events (unexpected)

Solutions:

  • Verify webhook URL is accessible (use ngrok for local dev)
  • Check webhook endpoint returns 200 OK
  • Verify webhook events are selected correctly
  • Check server logs for errors
  • Test webhook URL manually with curl
  • Remember: All webhooks receive ALL events - implement filtering
  • Use metadata to filter events by environment
  • Disable webhooks you don't need (but they'll still receive events)

Issue 3: Campaign ID Not Found

Symptoms:

  • 404 errors when creating donations
  • "Campaign not found" errors

Solutions:

  • Verify campaign ID is correct
  • Ensure campaign is active in Givebutter
  • Check campaign belongs to your organization
  • Use API to list campaigns and verify IDs

Issue 4: Webhook Signature Verification Fails

Symptoms:

  • 401 errors in webhook handler
  • Signature mismatch errors

Solutions:

  • Verify webhook secret is correct
  • Check signature algorithm matches Givebutter's (usually HMAC-SHA256)
  • Ensure payload is raw (not parsed) when verifying
  • Check header name matches: X-Givebutter-Signature
  • Log signature and computed signature for debugging

Debugging Checklist

  • API key is valid and not expired
  • Campaign ID exists and is active
  • Webhook URL is publicly accessible (use ngrok for local)
  • Webhook endpoint returns 200 OK
  • Webhook signature verification implemented correctly
  • Database connection works
  • Environment variables are set correctly
  • API calls use correct base URL
  • Request headers include Authorization
  • Request body matches Givebutter API schema
  • Using Stripe test cards for testing (4242 4242 4242 4242)
  • Test transactions marked with is_test flag
  • Environment filtering implemented in webhook handler
  • All webhooks configured correctly (dev/staging/prod)
  • Can distinguish test vs production transactions
