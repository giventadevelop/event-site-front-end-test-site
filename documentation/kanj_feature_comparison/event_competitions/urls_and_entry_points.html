1. Prerequisites

Competition mode must be enabled on the event. In admin, edit the event and turn on Competition Event (isCompetitionEvent). Until that flag is true:

  • The public event page does not show the rose Competitions button.
  • The admin event dashboard does not show the Competitions tile.
Typical admin setup order
  1. /admin/events/{eventId}/competitions/settings — audience mode, registration mode
  2. .../competitions/days — schedule days
  3. .../competitions/list — competition catalog (+ list/new to add)
  4. .../competitions/registrations — review registrations
  5. .../competitions/results — scores, winner photos, publish

Public hub, register, winners, and rules pages show content once settings and catalog data exist.

3. Public routes

Implemented under src/app/events/[id]/competitions/. Primary hub: CompetitionHub.tsx.

Route patternAuthPurpose
/events/{eventId} Public Event detail; shows Competitions button (rose) when isCompetitionEvent → links to hub
/events/{eventId}/competitions Public Main hub — schedule, catalog, links to sub-pages
/events/{eventId}/competitions/register Clerk Registration wizard + Stripe checkout
/events/{eventId}/competitions/my-registrations Clerk User / family registrations dashboard
/events/{eventId}/competitions/winners Public Published winners board
/events/{eventId}/competitions/rules Public Rules & content blocks

4. Admin routes

Implemented under src/app/admin/events/[id]/competitions/. In-page nav: CompetitionAdminNav.tsx (Settings → Days → List → Registrations → Results).

Route patternAuthPurpose
/admin/events/{eventId} Admin Event admin dashboard; Competitions tile when competition event → settings
/admin/events/{eventId}/competitions/settings Admin Competition settings (audience mode, registration mode, etc.)
/admin/events/{eventId}/competitions/days Admin Schedule days
/admin/events/{eventId}/competitions/list Admin Competition catalog
/admin/events/{eventId}/competitions/list/new Admin Create competition
/admin/events/{eventId}/competitions/{compId}/edit Admin Edit competition
/admin/events/{eventId}/competitions/registrations Admin Registrations list / export
/admin/events/{eventId}/competitions/results Admin Results, winner photos, publish

From Edit event (EventForm.tsx): when isCompetitionEvent is enabled, use Open competition admin → to jump to .../competitions/settings.

5. Entry points & navigation flow

Public visitor flow
1. Browse events → open /events/{eventId}
2. Click Competitions (visible only if competition event)
3. Land on /events/{eventId}/competitions (hub)
4. From hub: Register (sign in) · My registrations · Winners · Rules
Admin flow
A. /admin/manage-events → select event → /admin/events/{eventId}
B. Click Competitions tile → .../competitions/settings
C. Use top nav: Settings · Schedule Days · Competitions · Registrations · Results
Alt. Enable competition on event form → Open competition admin →

7. Authentication

AreaBehavior
Public hub, winners, rules No sign-in; routes under /events(.*) public routes
register, my-registrations src/middleware.ts calls auth.protect() for these paths; unsigned users → Clerk sign-in
All /admin/events/.../competitions/* Clerk session + user_role ADMIN / SUPER_ADMIN in user_profile for tenant

8. Supporting API (not browser pages)

Do not open these in the browser as end-user pages; the UI calls them via server actions / proxy.

  • Proxy: /api/proxy/event-competition-settings, event-competition-days, event-competitions, event-competition-registrations, event-competition-results, etc.
  • Server actions: src/app/events/[id]/competitions/ApiServerActions.ts, src/app/admin/events/[id]/competitions/ApiServerActions.ts
  • Stripe: /api/stripe/competition-payment-intent

See frontend_prd.html §1–2 for the full proxy and server-action list.
