1. Title and summary

News Portal (Strapi-backed) for MOSC. Add a “News” link in the MOSC top menu next to “Saints” that navigates to a news portal. The news portal is displayed and implemented using data from Strapi CMS. The current MOSC header and hero/photo remain unchanged; the news content (flash, main/featured/press release, most read, sidebar) appears inside the main content area, within the existing header and footer.

2. Goals and non-goals

Goals

  • Strapi-driven news UI: all news content (articles, flash, sidebar, ads) sourced from Strapi.
  • Same MOSC header and footer; news layout confined to the main content region.
  • Sidebar promotional block (e.g. Facebook embed) replicated from legacy behaviour, with content from Strapi.
  • Layout aligned with the reference screenshots (news_template_layout_final.jpeg, news_template_layout_annotated_diagram.jpeg).

Non-goals

  • Authentication required to read news (news is public).
  • Editing or authoring content in the Next.js app (content is managed in Strapi Admin only).

3. User stories

  • As a visitor, I see a “News” item in the MOSC top menu next to “Saints” so that I can open the news portal.
  • As a visitor, I see a news homepage with flash news (if enabled), main news, featured news, press release, and most read sections, so that I can browse the latest content.
  • As a visitor, I see a sidebar with a promotional block (e.g. Facebook) and ad slots, so that the layout matches the legacy Catholicate News experience.
  • As a visitor, I can open an article from the list and read the full content on an article detail page.

4. Layout and content mapping

Each section of the news portal maps to Strapi content types as follows. See catholicatenews_strapi_content_mapping.md for full detail.

Section Strapi content type Retrieval
Flash News (red bar) Homepage (single) GET /api/homepage — flashNewsMessage, flashNewsActive
Main News Article (collection) Filter category.slug=main-news
Featured News Article (collection) Filter isFeatured=true
Press Release Article (collection) Filter category.slug=press-release
Most Read Article (collection) Sort views:desc, limit 5
Sidebar (Facebook, video, promo) Sidebar Promotional Block (single) GET /api/sidebar-promotional-block
Sidebar ads Advertisement Slot (collection) Filter position=sidebar

5. Strapi connectivity

The Next.js application reads Strapi configuration from environment variables and performs all Strapi API calls server-side only (no API token exposure to the client).

  • Strapi URL: NEXT_PUBLIC_STRAPI_URL (e.g. http://localhost:1337 or production Strapi URL).
  • API token: STRAPI_API_TOKEN (server-only; not prefixed with NEXT_PUBLIC_). For public read-only access, Strapi can expose a read-only API token; the Next.js app uses it in server components or API routes only.

A small helper (e.g. src/lib/strapi.ts) reads these env values and exports the base URL and headers for Strapi fetch. All data fetching for the news portal is done in server components or server actions so the token is never sent to the browser.

6. Functional requirements

  • Menu: Add a “News” link in the MOSC top menu, placed next to “Saints”, pointing to the news portal route.
  • Route: News portal at /mosc/news (or as decided; alternative is making news the default MOSC content at /mosc).
  • Main content: Flash news bar (visible only when enabled in Strapi); main news; featured news; press release; most read. All within existing MOSC header and footer.
  • Sidebar: Promotional block (e.g. Facebook embed from Strapi sidebar block); advertisement slots (position=sidebar).
  • Article detail: Dynamic route for single article (e.g. /mosc/news/[slug]), showing title, cover image, body, metadata; optional view-count increment.

7. Data and API

Strapi REST API endpoints and filters (from content mapping document):

  • Homepage (flash): GET {STRAPI_URL}/api/homepage?populate=* → flashNewsMessage, flashNewsActive.
  • Featured articles: GET {STRAPI_URL}/api/articles?filters[isFeatured][$eq]=true&filters[publishedAt][$notNull]=true&populate=cover,category,author&sort=publishedAt:desc&pagination[limit]=6 (add tenant filter if Strapi has tenant).
  • Main news: GET {STRAPI_URL}/api/articles?filters[category][slug][$eq]=main-news&filters[publishedAt][$notNull]=true&populate=cover,category,author&sort=publishedAt:desc&pagination[limit]=10.
  • Press release: GET {STRAPI_URL}/api/articles?filters[category][slug][$eq]=press-release&filters[publishedAt][$notNull]=true&populate=cover,category,author&sort=publishedAt:desc&pagination[limit]=10.
  • Most read: GET {STRAPI_URL}/api/articles?filters[publishedAt][$notNull]=true&populate=cover,category&sort=views:desc&pagination[limit]=5.
  • Sidebar promotional block: GET {STRAPI_URL}/api/sidebar-promotional-block (or equivalent single-type endpoint).
  • Advertisement slots: GET {STRAPI_URL}/api/advertisement-slots?filters[position][$eq]=sidebar (plus tenant filter if applicable).
  • Global (optional): GET {STRAPI_URL}/api/global for site name, favicon, etc. if needed.

8. UI/UX constraints

  • Reuse the MOSC design system (typography, colors, spacing, components) as defined in project styling standards.
  • Responsive layout: main column + sidebar; sidebar stacks or reflows appropriately on small screens.
  • Sidebar behaviour (promo block, ads) should replicate the legacy index.html experience where applicable, with content driven by Strapi (embedCode, videoUrl, etc.).

9. Out of scope / future

  • Full i18n for news content (can be added later via Strapi i18n).
  • Multi-tenant Strapi (single tenant assumed unless specified).
  • View-count increment for “Most Read” can be deferred (custom Strapi controller or PATCH with auth).
