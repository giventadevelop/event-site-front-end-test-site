Extends gallery_album with structured event timing and place so public cards can show legacy strings like “October 24–26, 2015, Indore” instead of year-only (2015). Source data already exists in src/lib/gallery/moscStaticAlbums.ts and scripts/gallery-porting/static-gallery-manifest.json as the date field — it was never persisted to the API.

Non-breaking rule: New columns are nullable. Existing albums keep working; cards fall back to albumYear when event dates are absent.

Prerequisite: Category & year v1.0 deployed (gallery_category, album_year, gallery_category_id).

Porting scripts: scripts/gallery-porting/ — extend import + add backfill (see data migration PRD).

Executive summary

Problem and solution

The legacy MOSC gallery card shows a human-readable date line under the title (e.g. November 25, 2015 or October 24–26, 2015, Indore). After static → dynamic porting, the API stores only albumYear (integer). Full dates and locations were extracted into the manifest but dropped at import time.

Solution (v1.0): Add three nullable columns on gallery_album, expose via REST, extend admin create/edit, format display on MoscGalleryAlbumCard and /gallery cards, and backfill imported albums from the existing manifest (no photo re-import).
