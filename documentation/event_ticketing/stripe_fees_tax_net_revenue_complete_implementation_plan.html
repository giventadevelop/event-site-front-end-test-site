Complete Implementation Plan: Stripe Fees, Tax, and Net Revenue

Document Date: January 11, 2026

Purpose: Master implementation plan for capturing Stripe fees, taxes, and calculating net revenue across frontend, backend, and batch job systems.

Implementation Phase: Single Phase - All Enhancements

Executive Summary

This document provides a comprehensive overview of the complete implementation plan to:

  1. Capture Stripe fees and taxes that are not being captured during purchase flow
  2. Calculate and track net revenue (actual payout to bank account)
  3. Implement proper timing logic (10-day delay for normal batch runs)
  4. Add eventId filtering capability to batch job
  5. Fix purchase flow calculations (total_amount vs final_amount)
  6. Enhance analytics to display net payout and organizer revenue

All changes will be implemented in a single phase with no future enhancements deferred.

Documentation Structure

This implementation plan is split into four comprehensive documents:

📋 Main Plan 💻 Frontend Changes ⚙️ Backend Changes 🔄 Batch Job Changes

1. Problem Statement

1.1 Current Issues

  • Issue 1: total_amount and final_amount are identical because calculations are not being done during purchase
  • Issue 2: stripe_fee_amount and stripe_amount_tax are consistently NULL/0.00
  • Issue 3: Net revenue (payout to bank) is not calculated or displayed
  • Issue 4: Batch job cannot filter by eventId
  • Issue 5: Batch job does not implement 10-day delay logic for normal runs

1.2 Financial Flow Understanding

Example Transaction Flow:

1. Customer Payment: $1.00 (final_amount)

2. Stripe Processing:

  • Stripe Fee: $0.30 (stripe_fee_amount)
  • Stripe Tax: $0.10 (stripe_amount_tax)

3. Net Payout to Bank: $1.00 - $0.30 - $0.10 = $0.60

4. Platform Fee: $0.05 (platform_fee_amount)

5. Event Organizer Receives: $0.60 - $0.05 = $0.55
