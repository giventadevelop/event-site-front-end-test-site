Frontend Requirements: Fee-Free Manual Payments

This document specifies the UI/UX and client requirements for manual payments (Zelle, Venmo, Cash App, Cash, Check, Other) in the Next.js frontend. Align all UI with @ui_style_guide.mdc.

1. Event Configuration & Visibility

  • Respect event-level payment_flow_mode (Stripe-only, Manual-only, Hybrid).
  • Only show manual methods enabled in tenant configuration.
  • Display a banner explaining that manual payments require organizer confirmation.

2. Checkout UI

  • Payment method selector with Stripe + Manual methods.
  • Manual method detail panel showing: handle, memo instructions, and amount due.
  • Checkbox acknowledgement: “I understand tickets are issued after confirmation.”
  • Submit action creates a manual payment request via proxy API.

3. Proof-of-Payment Upload

  • Optional upload after request creation (image/PDF).
  • Display current upload status and allow replace/delete before confirmation.
  • Explain storage location: events/{eventId}/manual-payments/proof-of-payment/.

4. Attendee Status & Tickets

Status UI Behavior
Requested Show “Pending confirmation” banner; QR code hidden.
Received Show QR code and success confirmation.
Voided/Refunded Show cancellation notice and provide re-request action.

5. Admin & Organizer Console

  • Manual Payments tab filtered by event and status.
  • Action buttons: Mark Received, Void/Refund, Add Notes.
  • Download proof-of-payment attachments.
  • Dashboard summary widget reads from manual payment summary report endpoint.
  • Display totals grouped by status and method per event.

6. API Integration Notes

  • All REST calls must use proxy routes per cursor_rules/nextjs_api_routes.mdc.
  • Use absolute URLs in server components; client components use relative proxy URLs.
  • No manual payment expiration; allow user-initiated cancel.
