1. What this controls — Tenant default homepage hero images when no upcoming event hero media exists. When Include with events is on, these slides append after event hero images.
  2. Before you upload (create mode) — Save tenant settings first, then open Edit to enable uploads (same as logo upload).
  3. Upload rules — Drag-and-drop or browse; PNG, JPG, JPEG, WEBP, GIF; max 10 MB per file; recommended 2000×800 (5:2 landscape); max 20 images in the library. Successful uploads auto-save enriched JSON when a settings record exists.
  4. Library vs active — The library holds all uploaded slides. Only slides marked Active participate in homepage selection. New uploads default to Inactive (opt-in).
  5. Mark slides active — Toggle per thumbnail; max 10 active at once. Drag to reorder — order matters for the active subset.
  6. Homepage display count — Select 1–6 (default 6): maximum active slides shown when at least one slide is active (min(active count, display count)).
  7. Zero active fallback — If no slides are marked active but the library is not empty, the homepage shows 3 random images from the library on each page load.
  8. Display mode — slideshow rotates selected URLs; random picks one per visit; single uses the first URL only. Applied after active / random-3 resolution.
  9. Include with events — When checked, tenant default slides append after upcoming event hero images.
  10. Homepage fallback chain — Event hero media → tenant defaults (rules above) → bundled /images/hero_section/hero_images/fallback/default-hero.webp.
  11. Legacy data — Plain URL JSON arrays (["https://..."]) keep showing all URLs on the homepage until an admin saves from this UI (migrates to enriched [{ url, active }]).
  12. Manual URLs (advanced) — HTTPS, one per line; merged as inactive; duplicate URLs are skipped.
  13. Verify — Save settings → open the homepage in a new tab; test with and without upcoming events.
Tip: Use the live preview panel to see how active selection, display count, and random-3 fallback affect what visitors see before you publish.
