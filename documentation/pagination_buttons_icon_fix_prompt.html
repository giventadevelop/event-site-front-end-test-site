Pagination Buttons Icon Fix - Implementation Prompt

🚨 Issue Description

Problem: Pagination buttons (Previous and Next) on the /admin/manage-events page are not displaying SVG icons according to the icon standards rule, even after multiple attempts to update the code.

Affected Elements:

  • Previous Button: /html/body/div[3]/div/div[7]/div[1]/button[1]
  • Next Button: /html/body/div[3]/div/div[7]/div[1]/button[2]

Current Behavior:

  • Buttons appear as blue rounded rectangles with text ("Previous" and "Next")
  • No visible chevron icons (left/right arrows) are displayed
  • Icons should be visible even when buttons are disabled/grayed out
  • Buttons are located in the pagination footer section of the EventList component

Expected Behavior:

  • White chevron-left icon (←) visible in Previous button
  • White chevron-right icon (→) visible in Next button
  • Icons must be visible even when buttons are disabled (grayed out due to insufficient records)
  • Icons should follow the icon standards rule: .cursor/rules/icon_standards.mdc

✅ Solution Requirements

File to Modify:

src/components/EventList.tsx

Icon Standards to Follow:

Reference Rule: .cursor/rules/icon_standards.mdc

Pattern: Small Inline Icons for buttons

  • Icon size: w-5 h-5 (20px × 20px) for button icons
  • Use inline SVG with viewBox="0 0 24 24"
  • Stroke width: strokeWidth={2} or strokeWidth="2"
  • Color: White (stroke="white" or stroke="#FFFFFF") for icons on blue buttons

Implementation Steps:

1 Remove react-icons imports
// ❌ REMOVE this line if it exists: import { FaChevronLeft, FaChevronRight } from 'react-icons/fa';
2 Update Previous Button (button[1])

Replace the button content with inline SVG chevron-left icon:

<button onClick={onPrevPage} disabled={isPrevDisabled} className="px-4 py-2 bg-blue-600 text-white font-semibold rounded-lg shadow hover:bg-blue-700 disabled:opacity-50 disabled:cursor-not-allowed flex items-center gap-2 transition-colors" title="Previous Page" aria-label="Previous Page" type="button" > <svg width="20" height="20" viewBox="0 0 24 24" fill="none" xmlns="http://www.w3.org/2000/svg" style={{ display: 'block', width: '20px', height: '20px', flexShrink: 0, pointerEvents: 'none' }} > <path d="M15 19l-7-7 7-7" stroke="#FFFFFF" strokeWidth="3" strokeLinecap="round" strokeLinejoin="round" fill="none" /> </svg> <span className="text-white">Previous</span> </button>
3 Update Next Button (button[2])

Replace the button content with inline SVG chevron-right icon:

<button onClick={onNextPage} disabled={isNextDisabled} className="px-4 py-2 bg-blue-600 text-white font-semibold rounded-lg shadow hover:bg-blue-700 disabled:opacity-50 disabled:cursor-not-allowed flex items-center gap-2 transition-colors" title="Next Page" aria-label="Next Page" type="button" > <span className="text-white">Next</span> <svg width="20" height="20" viewBox="0 0 24 24" fill="none" xmlns="http://www.w3.org/2000/svg" style={{ display: 'block', width: '20px', height: '20px', flexShrink: 0, pointerEvents: 'none' }} > <path d="M9 5l7 7-7 7" stroke="#FFFFFF" strokeWidth="3" strokeLinecap="round" strokeLinejoin="round" fill="none" /> </svg> </button>
4 Key Implementation Details
  • Explicit white color: Use stroke="#FFFFFF" (hex) instead of stroke="white" or stroke="currentColor"
  • Thicker stroke: Use strokeWidth="3" for better visibility when button is disabled (50% opacity)
  • Block display: Use display: 'block' with explicit pixel dimensions
  • Pointer events: Add pointerEvents: 'none' to prevent interaction issues
  • Flex shrink: Add flexShrink: 0 to prevent icon from shrinking
  • Text color: Ensure span has className="text-white" for consistency

⚠️ Important Notes

  • Disabled State Visibility: The buttons use disabled:opacity-50 which makes the entire button 50% transparent. The SVG icons must use explicit white color (#FFFFFF) and thicker stroke (3) to remain visible when disabled.
  • No react-icons: Do NOT use FaChevronLeft or FaChevronRight from react-icons. Use inline SVG only.
  • File Location: The pagination buttons are in the EventList component at approximately line 602-662 in src/components/EventList.tsx
  • Component Usage: The EventList component is used in src/app/admin/manage-events/page.tsx

🔧 Troubleshooting Steps

If Icons Still Don't Appear:

  1. Clear Next.js build cache:
    # Windows PowerShell Remove-Item -Path .next -Recurse -Force -ErrorAction SilentlyContinue
  2. Restart dev server completely:
    # Stop server (Ctrl+C) # Wait 2-3 seconds # Restart npm run dev
  3. Hard refresh browser:
    • Windows/Linux: Ctrl + Shift + R or Ctrl + F5
    • Mac: Cmd + Shift + R
  4. Check browser DevTools:
    • Open DevTools (F12)
    • Go to Elements/Inspector tab
    • Find the button element: /html/body/div[3]/div/div[7]/div[1]/button[2]
    • Verify the SVG element exists inside the button
    • Check for any CSS that might be hiding the SVG
    • Check Console tab for any JavaScript errors
  5. Verify file was saved:
    # Check file modification time Get-Item src\components\EventList.tsx | Select-Object LastWriteTime
  6. Check for multiple EventList files:
    # Search for all EventList files Get-ChildItem -Recurse -Filter "*EventList*" | Select-Object FullName
