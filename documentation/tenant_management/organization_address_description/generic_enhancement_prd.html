Goals

  • Capture complete organization address + description + website on tenant organization create/edit.
  • Migrate existing tenant_settings address values into tenant_organization where org fields are empty.
  • Remove address and description fields from tenant settings admin UI and stop PATCHing them on settings.
  • Public/footer/contact reads: organization first, settings address fallback for legacy tenants.
  • Align admin UX with user profile address layout on the organization form only.

Out of scope (v2.0)

  • Physical DROP of deprecated tenant_settings address columns (planned v2.1 after migration window).
  • ISO-3166 country dropdown, geocoding, Maps autocomplete.
  • Separate “public contact address” distinct from HQ (future if product requires).
