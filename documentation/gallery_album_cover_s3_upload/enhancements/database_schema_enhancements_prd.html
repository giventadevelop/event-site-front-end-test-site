Gallery Album Cover S3 — Database Schema PRD

Version: 1.0  |  Date: June 18, 2026
Index: generic_enhancement_prd.html  |  Canonical schema: code_html_template/SQLS/Current_Sqls/Latest_Schema_Post__Blob_Claude_12.sql

Verdict for v1.0: No PostgreSQL schema migration is required to store S3 cover URLs. The existing gallery_album.cover_image_url column is sufficient (same pattern as focus_group.cover_image_url).

No DDL required for MVP. Backend upload writes HTTPS S3 URL into cover_image_url. Optional v1.1 adds traceability via cover_media_id.

1. Existing schema (no change)

gallery_album — relevant columns
ColumnTypePurpose

idbigint PKUsed in S3 path segment album-id/{id}
tenant_idvarchar(255) NOT NULLTenant isolation; S3 path segment

Column comment (already in schema):

COMMENT ON COLUMN public.gallery_album.cover_image_url IS
  'URL to cover image (references event_media.file_url). Used as album thumbnail in gallery view.';

2. Related table — event_media

Album photos (and optionally cover uploads) live in event_media with:

  • album_id → FK to gallery_album.id
  • file_url → S3 object URL
  • event_media_type → e.g. gallery (existing album upload flow)
  • storage_type → S3

Cover upload may create a dedicated event_media row OR only update gallery_album.cover_image_url (focus group pattern updates entity URL and may still create media — follow backend implementation in focus-group handler).

3. Optional v1.1 — cover_media_id (not required for MVP)

Defer unless product needs “which media row is the cover?” for delete/replace logic or presign refresh. 
-- Optional future migration (v1.1)
ALTER TABLE public.gallery_album
  ADD COLUMN cover_media_id bigint NULL;

ALTER TABLE public.gallery_album
  ADD CONSTRAINT fk_gallery_album_cover_media
  FOREIGN KEY (cover_media_id) REFERENCES public.event_media(id) ON DELETE SET NULL;

CREATE INDEX idx_gallery_album_cover_media_id
  ON public.gallery_album (cover_media_id) WHERE (cover_media_id IS NOT NULL);

COMMENT ON COLUMN public.gallery_album.cover_media_id IS
  'Optional FK to event_media used as album cover; cover_image_url remains denormalized display URL.';
