Document: GiveButter Widget – Hiding Donation Options (Tickets-Only Checkout) Analysis

Context: Embedded widget on /events/[id]/givebutter-checkout shows donation buttons/upsell and "I'd just like to donate" alongside ticket purchase.

Date: January 2025

Analysis: Removing or Hiding Donation Options in the GiveButter Embed

1. What You’re Seeing

On /events/4202/givebutter-checkout the embedded <givebutter-widget id="..."> shows:

  • Ticket selection (e.g. General Admission, VIP Admission) – desired.
  • Donation step – preset amounts ($25, $55, $100, etc.) and "Would you like to add a donation?" – often unwanted for ticket-only flows.
  • "I'd just like to donate" button – alternative path to donate instead of/in addition to buying tickets – often unwanted.

Goal: optionally remove or hide these donation-related elements so the checkout is tickets-only.

2. Widget ID vs Campaign ID (Form)

Embed typeElementTypical useTickets-only?
Widget ID <givebutter-widget id="j1ek6j"> Event/Page campaign: embeds the full checkout flow (tickets → optional donation step → payment). No – flow is fixed by GiveButter; donation step is part of the flow.
Campaign ID (form) <givebutter-form campaign="FwSx70"> Donation form (suggested amounts, custom amount, etc.). Used for Form campaigns or donation-only embeds. No – form is donation-focused; no ticket selection. Not a substitute for ticket checkout.
Conclusion: Using "form with campaign id" (givebutter-form) does not give more flexibility for tickets-only. It shows a donation form, not ticket purchase. For event ticket checkout you need the widget (or full campaign page). The only way to get tickets-only today is to change what the Event campaign shows in its checkout (see below).

3. What GiveButter Exposes Today

3.1 Documented widget/URL parameters

From GiveButter’s docs (URL and HTML parameters, Widgets):

  • Widget HTML attributes: amount, frequency, fund, promo – e.g. <givebutter-widget id="xLWmpB" amount="25">. These prefill the donation step; they do not remove it.
  • For campaigns with tickets: "URL parameters do not apply to the ticket selection step – they apply to the donation amount step after ticket selection." So there is no documented parameter to skip or hide the donation step.
  • No documented attribute such as data-hide-donations, data-tickets-only, or similar.

3.2 Campaign dashboard (Event)

Under Event campaign Tools → Donation settings you can configure suggested amounts, "Other", minimum donation, etc. There is no documented setting to "disable donation step" or "hide optional donation slide" in checkout.

Under Settings → General, GiveButter does allow hiding other steps (e.g. "Public messages" – the "Show your support" page). So the product supports per-step toggles in some cases, but not (yet) for the donation upsell.

4. GiveButter Roadmap: Donation Button / Slide Customization

On the GiveButter community roadmap there is a high-vote request:

"Ability to customize 'No thanks, I'd just like to donate' button (Enable/Disable + Ability to edit text)" – status in progress (as of late 2025). Merged requests include "Disable Donation Slide" and similar. GiveButter has stated they are working on this feature.

Implication: Once this ships, the fix will likely be a campaign-level setting in the GiveButter dashboard (e.g. disable optional donation step or hide "I'd just like to donate"). No change to our embed code may be required – the same <givebutter-widget id="..."> would then respect the campaign’s new settings.

5. Recommendations

  1. Contact GiveButter support – Ask whether:
    • There is any current workaround (e.g. account/campaign setting or beta) to hide the donation step or "I'd just like to donate" for an Event campaign.
    • When the "customize / disable donation button & optional donation slide" feature will be generally available.
  2. Watch the roadmap – When the feature is released, configure the relevant Event campaign(s) in the GiveButter dashboard to disable the donation step or button as desired.
  3. Keep using the widget ID – For event ticket checkout, the widget embed is correct. Do not switch to givebutter-form + campaign ID expecting tickets-only; that path is donation-only.
  4. Optional: pass extra attributes later – If GiveButter later adds widget attributes (e.g. data-hide-donations="true"), we can pass them from donation_metadata through GivebutterWidget (see section 6).

6. Code: Optional attribute passthrough

So that future GiveButter widget attributes can be used without another code change, GivebutterWidget can accept optional data-* (or other) props and forward them to <givebutter-widget>. If GiveButter documents something like data-hide-donations or data-tickets-only, we can then:

  • Store a flag in donation_metadata (e.g. givebutterHideDonationStep: true), and
  • Pass it through as the corresponding attribute on the widget.

TypeScript declarations for givebutter-widget can be extended to include any new attributes once they are documented.
