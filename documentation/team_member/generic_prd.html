Executive summary

Problem and solution

Organizations need rosters for sports teams and music bands with positions, jersey numbers, instruments, and skills—not only executive leadership bios. The platform already has executive_committee_team_members for org leadership; this feature adds team_groups + team_members with optional user_profile_id.

Data model (confirmed)

  • team_groups — tenant-scoped squad/band (type, name, slug, section labels, CTA).
  • team_members — members belonging to one group; optional FK to user_profile.

Normalization rules

  • When user_profile_id is set, admin may prefill name, email, and image from profile; stored columns are the public display source (overrides allowed).
  • Without profile: first_name and last_name required.
  • expertise — JSON array string of skills (same convention as executive committee).
  • team_type on group (SPORTS | MUSIC | OTHER) drives which admin fields are emphasized.
