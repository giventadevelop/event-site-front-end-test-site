1. Proxy routes

Pages API (createProxyHandler)
  • src/pages/api/proxy/team-groups/index.ts + [...slug].ts
  • src/pages/api/proxy/team-members/index.ts + [...slug].ts (PATCH: bodyParser: false)

Do not add tenantId.equals in client queries; proxy injects tenant for list GETs.

2. Types and parsers

  • src/types/teamGroup.ts, src/types/teamMember.ts
  • src/lib/parseTeamGroupsResponse.ts, src/lib/parseTeamMembersResponse.ts

3. Admin CRUD

RouteFiles
/admin/team-groupsApiServerActions.ts, TeamGroupsClient.tsx, TeamGroupForm.tsx
/admin/team-membersApiServerActions.ts, TeamMembersClient.tsx, TeamMemberForm.tsx, TeamMemberUploadActions.ts

4. Public UI

  • SquadRosterSection — homepage carousel; first active group by displayOrder
  • SquadMemberCard — portrait, jersey badge, subtitle
  • /team/[slug] — full roster page from slug.equals

5. Tenant settings flags

show_team_members_section_in_home_page → SquadRosterSection (showSquadSection in TenantSettingsProvider).

show_executive_committee_section_in_home_page → TeamSection (executive committee). Fallback: legacy tenants use show_team_members_section_in_home_page for exec until migrated.

6. API query examples

GET /api/proxy/team-groups?isActive.equals=true&sort=displayOrder,asc
GET /api/proxy/team-members?teamGroupId.equals={id}&isActive.equals=true&sort=priorityOrder,asc
GET /api/proxy/team-groups?slug.equals={slug}&isActive.equals=true

7. Homepage wiring

HomePageClient.tsx renders SquadRosterSection when showSquadSection and TeamSection when showExecutiveCommitteeSection.
