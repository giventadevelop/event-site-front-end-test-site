Backend Requirements: Fee-Free Manual Payments

This document specifies backend service and API requirements to support manual payments in the Spring Boot service (malayalees-us-site-boot).

1. Services & Domain Logic

  • ManualPaymentService to create/update manual payment requests.
  • ManualPaymentAttachmentService for proof-of-payment files.
  • Integrate with ticket inventory and QR issuance after status = Received.
  • No partial payment tracking in v1.

2. REST Endpoints (Examples)

  • POST /api/manual-payments
  • GET /api/manual-payments?eventId.equals=
  • PATCH /api/manual-payments/{id} (status update)
  • POST /api/manual-payments/{id}/proof (upload proof)
  • GET /api/manual-payment-methods?tenantId.equals=
  • GET /api/manual-payment-summary?eventId.equals=&snapshotDate.equals=

3. Storage & File Handling

  • Store proof-of-payment files in S3:
  • events/{eventId}/manual-payments/proof-of-payment/{paymentId}/
  • Persist file key, filename, content type, and upload timestamp.
  • Provide signed URLs for secure download.

4. Reporting & Aggregation

  • Batch job writes daily snapshots into manual_payment_summary_report.
  • Expose read-only summary endpoint for admin dashboard consumption.
  • Retention policy: keep daily snapshots for 18 months, then archive.

5. Security & Authorization

  • Only event organizers/admins can mark Received or Voided.
  • Tenant isolation required on all queries (tenantId.equals).
  • No external webhook triggers for manual actions in v1.

6. Standards

  • Align contracts to documentation/Swagger_API_Docs/api-docs.json.
  • Follow existing proxy patterns for Next.js integration.
