Project: MCEFEE Multi-Tenant Event Management Platform
Document Version: 1.0
Date: January 2026
Status: Draft

Executive Summary

This document defines the generic design and scope for a News Portal feature within the existing MCEFEE platform. The portal replaces the legacy WordPress-hosted static HTML news site (e.g. catholicatenews.in) with a dynamic, database-driven section. News is categorized (Main News, Featured News, Press Release, Flash News, etc.), and each display section can have its own configurable banner/hero image managed by editors via an admin dashboard. The News Portal is a separate page reachable from the main site menu and presents content from the database with proper categorization, dates, and section-level display configuration.

Key Concept: The legacy site is static HTML exported from WordPress. The new system stores articles, categories, section display config, and news flashes in the database and serves them via REST API and Next.js front-end, with an admin UI for editors to update content and section images independently.

2. Goals & Non-Goals

Goals

  • Provide a dedicated News section as a menu item (or submenu) leading to a dynamic news landing page.
  • Store news articles, categories, and section display configuration in the database (multi-tenant).
  • Allow editors to update section images (e.g. Main News, Featured News, Flash News banner) independently per section.
  • Support categorization consistent with legacy: Main News, Featured News, Press Release, Most Read, Flash News, and extensible categories.
  • Display news by category with pagination and publication dates; support “flash” carousel/spotlight items.
  • Admin dashboard that is user-friendly and self-guiding so editors can update news and section images regularly without developer help.

Non-Goals

  • Migrating legacy WordPress content automatically (can be a separate migration task).
  • Replacing or duplicating existing event or gallery features; News is an additive section.

3. Legacy News Structure (Reference)

From catholicatenews_in/index.html, category pages, and documentation/news_portal/catholicatenews_in_full_screen_shot.png:

  • Flash News: Carousel/ticker at top; short headlines linking to full article.
  • Main News: Section with title “Main News” linking to category/main-news; grid/list of articles with image, title, date, excerpt.
  • Featured News: Section with title “Featured News” linking to category/featured-news; same card pattern.
  • Most Read: Section listing popular or highly-read articles; two-column list with headlines, dates, and optional small thumbnails.
  • Press Release: Section linking to category/press-release.
  • Right Sidebar (Promotional / Advertisement area): Dedicated right column containing: social/profile widget, embedded video block with play button, “More Videos” list, and space suitable for rotating banner ads or sponsored content. This block is configurable (e.g. main banner/video thumbnail image, optional ad slots).
  • LIVE (menu item): A dedicated “LIVE” menu item in the primary navigation that links to a page for embedding live videos (e.g. YouTube Live, other streaming embeds). Editors can set the current live stream URL or embed code so visitors can watch live events directly from the news portal.

Each article has: featured image (e.g. uploads/YYYY/MM/...), title, publication date (post-info), excerpt. Categories used: main-news, featured-news, press-release, most-read, and equivalents for flash/uncategorized. Pagination: category/main-news/page/1, etc.

4. Scope & Sections

Display Sections (Configurable)

All sections below are visible on the legacy news portal (see catholicatenews_in_full_screen_shot.png). Each has an optional section-level image (or promotional creative) editable independently in the admin dashboard.

Section Key Purpose Editable Image
flash_news Carousel/ticker at top; short headlines Optional banner/background for flash area
main_news Main news listing block Section header/banner image
featured_news Featured news listing block Section header/banner image
most_read Most-read/popular articles block (headlines, dates, optional small thumbnails) Section header/banner image
press_release Press release listing block Section header/banner image
sidebar_promotional_block Right sidebar: embedded video, “More Videos” list, social widget, or rotating banner ads / sponsored content Main banner or video thumbnail image; rotating banners stored in news_sidebar_promotion, retrieved and shown as slideshow with configurable timer

Each section can have an optional image (URL or media reference) managed in the admin dashboard independently. The sidebar_promotional_block is the primary area for promotional or advertisement content.

Sidebar & Advertisement Areas

  • Sidebar promotional block: The right column is modeled as a configurable section (sidebar_promotional_block) in news_section_display_config (section-level title/banner override) plus a dedicated table news_sidebar_promotion for rotating banner ads. Each promotion row stores: image URL, optional link URL, display order, and display_duration_seconds (timer per slide). The front end retrieves active promotions for the tenant (filtered by is_active and optional valid_from / valid_to), orders by display_order, and renders them as a slideshow with timer (auto-advance after display_duration_seconds seconds per slide). Editors manage promotions in the admin dashboard (add, reorder, set duration and validity).
  • Generic ad slots (future): Additional ad slots (e.g. top banner, between sections, in-article) can be added later via extra config or a dedicated ad-slot table if needed.

Live Video (LIVE Menu)

  • Purpose: The legacy portal includes a “LIVE” menu item that takes users to a page where live videos (e.g. YouTube Live streams) are embedded. The new system must support this.
  • Behaviour: A dedicated public page (e.g. /news/live) displays the currently configured live stream embed. Editors set the active live stream URL (or embed URL) and optional title in the admin dashboard; the page renders the embed (e.g. YouTube iframe) so visitors can watch without leaving the site.
  • Config: One “current” live stream per tenant: embed URL (e.g. YouTube embed src), optional title, and active flag. Stored in a small config table or in section display config (e.g. section_key live_stream with an embed_url field).

News Categories

Categories align with legacy and are stored in DB: e.g. Main News, Featured News, Press Release, Most Read, Flash News, Achievements, English News, Uncategorized. New categories can be added by admins.

5. Stakeholders & Users

  • Public visitors: View news landing page, category pages, and article detail; no auth required for reading.
  • News editors / admins: Use admin dashboard to create/edit articles, manage categories, and set section images for each section (flash, main, featured, most read, press release, sidebar promotional block).
  • Developers: Integrate backend APIs and front-end pages per nextjs_api_routes and UI standards.

6. Database Design

New tables follow existing schema patterns: tenant_id, created_at, updated_at where applicable (see Latest_Schema_Post__Blob_Claude_12.sql).

Proposed Tables

  • news_category – Categories for articles (e.g. Main News, Featured News, Press Release). Fields: id, tenant_id, name, slug, description, display_order, is_active, created_at, updated_at.
  • news_article – Single news item. Fields: id, tenant_id, title, slug, excerpt, body (or body_html), featured_image_url (or media_id), published_at, status (DRAFT/PUBLISHED/ARCHIVED), author_id (FK user_profile), created_at, updated_at.
  • news_article_category – Many-to-many between news_article and news_category. Fields: id, news_article_id, news_category_id, is_primary (boolean), created_at.
  • news_section_display_config – Per-section display settings (one row per section per tenant). Fields: id, tenant_id, section_key (e.g. flash_news, main_news, featured_news, most_read, press_release, sidebar_promotional_block), section_title_override, banner_image_url (or media_id), display_order, is_active, created_at, updated_at.
  • news_sidebar_promotion – Rotating banner ads / promotional items for the right sidebar. Stored in the database, retrieved by tenant (and optional valid_from/valid_to), ordered by display_order, and displayed as a slideshow with configurable timer (display_duration_seconds per slide). Fields: id, tenant_id, title (optional), image_url, link_url (optional), display_order, display_duration_seconds (seconds per slide in slideshow), valid_from, valid_to, is_active, created_at, updated_at.
  • news_flash – Short “flash” items for carousel/ticker. Fields: id, tenant_id, headline, link_url (or news_article_id), display_order, valid_from, valid_to, is_active, created_at, updated_at.
  • news_live_stream_config – Current live stream for the LIVE page (one active config per tenant). Fields: id, tenant_id, embed_url (e.g. YouTube embed src), title (optional), is_active, created_at, updated_at.

Author can reference user_profile.id. Media can reference existing event_media or a dedicated news_media table if needed; otherwise URL fields suffice for MVP.

7. Integration Points

  • Frontend: New route(s) under app (e.g. /news, /news/category/[slug], /news/[slug]). Menu item added in Header/navigation to “News” or under an existing submenu.
  • Backend: REST APIs for news articles, categories, section config, and news flash (CRUD + list with filters). See backend_prd.html.
  • Proxy: Next.js proxy routes under /api/proxy/ for news endpoints, using createProxyHandler and JWT; public read can use same proxy with optional tenant scoping.
  • Admin: Admin dashboard under /admin/news (or similar) for articles, categories, section images, flash items, sidebar rotating promotions (slideshow timer), and live stream config; server actions and proxy per nextjs_api_routes.mdc.
  • Batch (optional): Jobs in event-site-manager-batch-jobs for scheduled publish/unpublish or sync; not required for MVP.

8. Open Questions

  • Exact menu placement (top-level “News” and “LIVE” vs under a “News” submenu).
  • Whether to reuse event_media or introduce news-specific media storage.
  • RSS/feed support for the new portal.
