Focus Group Event Association Feature - Requirements

📋 Document Information

Feature: Event Association for Focus Groups

Target Page: http://localhost:3000/admin/focus-groups/[id]/edit

Created:

Status: Requirements Definition (No Implementation)

1. Overview

This document defines the requirements for adding event association functionality to the Focus Group edit page. The feature will allow administrators to search for events and associate them with focus groups, with a paginated table displaying all associated events.

⚠️ Important: This document contains requirements only. Do NOT implement until explicitly requested. Focus on understanding the requirements and creating a comprehensive implementation plan.

2. Feature Requirements

2.1 Event Search Component

Location: Focus Group Edit Page (/admin/focus-groups/[id]/edit)

Reference Implementation: src/app/admin/promotion-emails/components/EventSearchSelect.tsx

Functionality:

  • Search events by Title or ID (toggle between search types)
  • Real-time search filtering as user types
  • Dropdown results showing matching events (max 7 items visible)
  • Display event title, ID, and start date in dropdown
  • Select event to associate with focus group
  • Clear selection functionality
  • Visual indication of selected event

UI Components:

  • Toggle buttons: "Search by Title" / "Search by ID"
  • Search input field with search icon
  • Dropdown results container (z-index: 50)
  • Selected event display box (blue background)
  • Clear button (×) when event is selected

Styling:

  • Match styling from EventSearchSelect.tsx
  • Input: border border-gray-400 rounded-xl focus:border-blue-500 focus:ring-blue-500 pl-10 pr-10 px-4 py-2 text-base
  • Toggle buttons: Blue when active, gray when inactive
  • Dropdown: White background, border, shadow-lg, max-height for scrolling
  • Hover states: Blue background on dropdown items

2.2 Event Association Functionality

Association Action:

  • When user selects an event from search dropdown, create association
  • Persist association to event_focus_groups join table
  • Association includes:
    • event_id - Selected event ID
    • focus_group_id - Current focus group ID
    • tenant_id - Current tenant ID (auto-injected)
    • created_at - Timestamp
    • updated_at - Timestamp
  • Prevent duplicate associations (enforced by unique constraint: tenant_id, event_id, focus_group_id)
  • Show success message after successful association
  • Refresh associated events table after association

API Endpoint:

  • Method: POST
  • Endpoint: /api/proxy/event-focus-groups
  • Request Body: 
    {
  "eventId": number,
  "focusGroupId": number,
  "tenantId": string (auto-injected)
}
  • Response: EventFocusGroupDTO with created association

Error Handling:

  • Handle duplicate association attempts gracefully
  • Show user-friendly error messages
  • Handle network errors
  • Handle validation errors from backend

2.3 Associated Events Table

Location:

Display at the bottom of the Focus Group Edit page, below the form fields and cover image upload section.

Data Source:

  • Read from event_focus_groups join table
  • Join with event_details table to get event information
  • Filter by focus_group_id matching current focus group
  • Filter by tenant_id (auto-injected)

Table Columns:

Column Source Display Notes
Event Title event_details.title Text Primary column, clickable link to event detail page
Start Date event_details.start_date Formatted date Use existing date formatting utilities
End Date event_details.end_date Formatted date Use existing date formatting utilities
Status event_details.is_active Badge/Indicator Active/Inactive badge
Actions N/A Unlink Button Remove association button

Event Filtering:

  • Show all associated events (upcoming and past)
  • Default sort: startDate,desc (newest first)
  • Optional: Add filter toggle for "Upcoming Events" / "Past Events"
  • Filter logic:
    • Upcoming: startDate >= today
    • Past: endDate < today

Unlink Functionality:

  • Each row has an "Unlink" button in Actions column
  • On click, show confirmation dialog
  • On confirm, delete record from event_focus_groups table
  • API Endpoint: DELETE /api/proxy/event-focus-groups/{id}
  • Refresh table after successful unlink

2.4 Pagination

Reference Implementation:

Match pagination style from http://localhost:3000/admin/manage-events

Reference component: src/components/EventList.tsx (lines 569-604)

Pagination Requirements:

  • Page Size: 10 events per page (configurable)
  • Pagination Controls:
    • Previous button (left side)
    • Page indicator: "Page X of Y" (center)
    • Next button (right side)
  • Pagination Details: Below controls, centered
    • Format: "Showing X to Y of Z events"
    • When no events: "No events found" with badge

Pagination Styling:

/* Pagination Container */
<div className="mt-8">
  <div className="flex justify-between items-center">
    {/* Previous Button */}
    <button
      onClick={handlePrevPage}
      disabled={isPrevDisabled}
      className="px-4 py-2 bg-blue-600 text-white font-semibold rounded-lg shadow hover:bg-blue-700 disabled:opacity-50 disabled:cursor-not-allowed flex items-center gap-2 transition-colors"
    >
      <FaChevronLeft className="h-5 w-5" />
      Previous
    </button>

    {/* Page Indicator */}
    <div className="text-sm font-semibold text-gray-700">
      Page {currentPage} of {totalPages}
    </div>

    {/* Next Button */}
    <button
      onClick={handleNextPage}
      disabled={isNextDisabled}
      className="px-4 py-2 bg-blue-600 text-white font-semibold rounded-lg shadow hover:bg-blue-700 disabled:opacity-50 disabled:cursor-not-allowed flex items-center gap-2 transition-colors"
    >
      Next
      <FaChevronRight className="h-5 w-5" />
    </button>
  </div>

  {/* Pagination Details */}
  <div className="text-center text-sm text-gray-600 mt-2">
    {totalCount > 0 ? (
      <>Showing <span className="font-medium">{startItem}</span> to <span className="font-medium">{endItem}</span> of <span className="font-medium">{totalCount}</span> events</>
    ) : (
      <div className="flex items-center justify-center gap-2">
        <span>No events found</span>
        <span className="bg-blue-100 text-blue-800 px-2 py-1 rounded-md text-sm font-medium">
          [No events match your criteria]
        </span>
      </div>
    )}
  </div>
</div>

Pagination Logic:

  • Use 0-based page indexing internally
  • Display as 1-based to users ("Page 1", "Page 2", etc.)
  • Calculate:
    • totalPages = Math.ceil(totalCount / pageSize)
    • startItem = currentPage * pageSize + 1
    • endItem = Math.min((currentPage + 1) * pageSize, totalCount)
  • Disable Previous when on first page
  • Disable Next when on last page
  • Disable both when loading

Icons:

  • Import from react-icons/fa: FaChevronLeft, FaChevronRight
  • Size: h-5 w-5

3. Technical Specifications

3.1 API Routes

Follow Next.js API Routes Rules:

Reference: .cursor/rules/nextjs_api_routes.mdc

Key Rules:

  • No local DTO declarations: Import DTOs from @/types
  • Environment Variables: Use process.env.NEXT_PUBLIC_API_BASE_URL
  • Query String Handling: Use buildQueryString helper
  • Tenant ID: Do NOT add tenantId.equals in client code - proxy handles it
  • JWT Handling: Use fetchWithJwtRetry for backend calls
  • Error Handling: Catch and return 500 with clear error message
  • Method Handling: Only allow GET/POST/PATCH/DELETE with proper 405 responses

API Endpoints Required:

Endpoint Method Purpose Parameters
/api/proxy/event-focus-groups POST Create association eventId, focusGroupId
/api/proxy/event-focus-groups GET List associations focusGroupId.equals={id}, pagination params
/api/proxy/event-focus-groups/{id} DELETE Remove association Association ID in URL
/api/proxy/event-details GET Search events title.contains, id.equals, pagination

3.2 Server Actions

Pattern:

Create server actions file: src/app/admin/focus-groups/[id]/edit/ApiServerActions.ts

Required Functions:

  • associateEventWithFocusGroup(eventId: number, focusGroupId: number)
    • POST to /api/proxy/event-focus-groups
    • Use fetchWithJwtRetry from @/lib/proxyHandler
    • Return EventFocusGroupDTO
  • fetchAssociatedEvents(focusGroupId: number, page: number, pageSize: number, sort?: string)
    • GET /api/proxy/event-details?focusGroupId.equals={id}&page={page}&size={pageSize}&sort={sort}
    • Return { events: EventDetailsDTO[], totalCount: number }
    • Read X-Total-Count header for total count
  • unlinkEventFromFocusGroup(associationId: number)
    • DELETE /api/proxy/event-focus-groups/{id}
    • Return success/error status
  • searchEvents(searchTerm: string, searchType: 'title' | 'id', pageSize?: number)
    • GET /api/proxy/event-details with appropriate filters
    • Return EventDetailsDTO[]

Important:

  • All server actions MUST use fetchWithJwtRetry from @/lib/proxyHandler
  • Use absolute URLs for server-side fetches: getAppUrl() from @/lib/env
  • Handle errors gracefully with try-catch
  • Return meaningful error messages

3.4 Database Schema

Tables Involved:

  • event_focus_groups (Join Table)
    • id (bigint, PK)
    • tenant_id (varchar)
    • event_id (bigint, FK to event_details)
    • focus_group_id (bigint, FK to focus_group)
    • created_at (timestamp)
    • updated_at (timestamp)
    • Unique constraint: (tenant_id, event_id, focus_group_id)
  • event_details (Event Data)
    • id (bigint, PK)
    • title (varchar)
    • start_date (date)
    • end_date (date)
    • is_active (boolean)
    • ... (other fields)
  • focus_group (Focus Group Data)
    • id (bigint, PK)
    • name (varchar)
    • ... (other fields)

Query Pattern:

SELECT ed.*
FROM event_details ed
INNER JOIN event_focus_groups efg ON ed.id = efg.event_id
WHERE efg.focus_group_id = {focusGroupId}
  AND efg.tenant_id = {tenantId}
ORDER BY ed.start_date DESC
LIMIT {pageSize} OFFSET {page * pageSize}
