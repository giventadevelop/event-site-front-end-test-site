Executive Summary

This document provides detailed implementation instructions for the backend changes required to support the manual payment success flow. The implementation must create ticket transactions automatically when manual payment requests are created, manage ticket status transitions, generate QR codes when payments are confirmed, and trigger email sending via batch jobs.

Key Requirement: All manual payment ticket transactions must start in PENDING status and only transition to CONFIRMED when an admin marks the payment as RECEIVED. QR codes and ticket emails are generated ONLY after confirmation.

2.1 POST /api/manual-payments (Enhance Existing)

Current Behavior: Creates a ManualPaymentRequest record only.
Required Enhancement: Also create EventTicketTransaction and EventTicketTransactionItem records.

Request Body Schema

Request Body (JSON) 
{
  "eventId": 52,
  "manualPaymentMethodType": "ZELLE_MANUAL",
  "amount": 150.00,
  "status": "REQUESTED",
  "requesterEmail": "user@example.com",
  "requesterName": "John Doe",
  "requesterPhone": "+1234567890",
  "paymentHandle": "zelle@example.com",
  "paymentInstructions": "Send payment to zelle@example.com with memo: Event 52",
  "proofOfPaymentFileKey": "events/52/manual-payments/proof-123.jpg",
  "proofOfPaymentUrl": "https://s3.amazonaws.com/bucket/events/52/manual-payments/proof-123.jpg",
  "cart": [
    {
      "ticketTypeId": 101,
      "quantity": 2
    },
    {
      "ticketTypeId": 102,
      "quantity": 1
    }
  ],
  "tenantId": "tenant_demo_001"
}

Implementation Steps

  1. Validate Request:
    • Verify eventId exists and is active
    • Validate cart array is not empty
    • Validate each ticket type exists and has sufficient inventory
    • Calculate total amount from cart and verify it matches amount field
    • Verify tenantId matches authenticated tenant
  2. Create ManualPaymentRequest:
    • Create record with status REQUESTED
    • Set ticket_transaction_id to null initially
    • Store all requester information
  3. Create EventTicketTransaction:
    • Status: PENDING (NOT CONFIRMED)
    • Set email, first_name, last_name, phone from requester info
    • Set total_amount from request amount
    • Set final_amount equal to total_amount
    • Set quantity to sum of all cart item quantities
    • Set transaction_reference to format: "MANUAL-{manualPaymentRequestId}"
    • Set purchase_date to current timestamp
    • DO NOT set qr_code_image_url (only set after RECEIVED status)
    • DO NOT set stripe_payment_intent_id (manual payments don't use Stripe)
  4. Create EventTicketTransactionItem Records:
    • For each item in cart array:
    • Fetch EventTicketType by ticketTypeId
    • Create EventTicketTransactionItem with:
      • transaction_id = created ticket transaction ID
      • ticket_type_id = from cart item
      • quantity = from cart item
      • price_per_unit = from ticket type price
      • total_amount = price_per_unit * quantity
    • CRITICAL: Do NOT update ticket type quantity_sold yet (only update when status changes to RECEIVED)
  5. Link Records:
    • Update ManualPaymentRequest.ticket_transaction_id with created transaction ID
    • Save updated payment request
  6. Return Response:
    • Return created ManualPaymentRequest with ticketTransactionId populated
    • Include related ticket transaction in response (optional, for frontend convenience)

Example Java Implementation Structure

ManualPaymentService.createManualPaymentRequest() 
@Transactional
public ManualPaymentRequestDTO createManualPaymentRequest(
    ManualPaymentRequestDTO requestDTO
) {
    // 1. Validate event exists and is active
    EventDetails event = eventDetailsRepository.findById(requestDTO.getEventId())
        .orElseThrow(() -> new EntityNotFoundException("Event not found"));

    if (!event.getIsActive()) {
        throw new BadRequestException("Event is not active");
    }

    // 2. Validate cart and calculate total
    List<CartItemDTO> cart = requestDTO.getCart();
    if (cart == null || cart.isEmpty()) {
        throw new BadRequestException("Cart cannot be empty");
    }

    BigDecimal calculatedTotal = BigDecimal.ZERO;
    for (CartItemDTO item : cart) {
        EventTicketType ticketType = ticketTypeRepository.findById(item.getTicketTypeId())
            .orElseThrow(() -> new EntityNotFoundException("Ticket type not found"));

        // Validate inventory
        int remaining = ticketType.getQuantityAvailable() - ticketType.getQuantitySold();
        if (item.getQuantity() > remaining) {
            throw new BadRequestException("Insufficient tickets available for type: " + ticketType.getName());
        }

        calculatedTotal = calculatedTotal.add(
            ticketType.getPrice().multiply(BigDecimal.valueOf(item.getQuantity()))
        );
    }

    // Verify amount matches
    if (calculatedTotal.compareTo(requestDTO.getAmount()) != 0) {
        throw new BadRequestException("Amount mismatch. Calculated: " + calculatedTotal + ", Provided: " + requestDTO.getAmount());
    }

    // 3. Create ManualPaymentRequest
    ManualPaymentRequest paymentRequest = new ManualPaymentRequest();
    paymentRequest.setEventId(requestDTO.getEventId());
    paymentRequest.setManualPaymentMethodType(requestDTO.getManualPaymentMethodType());
    paymentRequest.setAmount(requestDTO.getAmount());
    paymentRequest.setStatus(ManualPaymentStatus.REQUESTED);
    paymentRequest.setRequesterEmail(requestDTO.getRequesterEmail());
    paymentRequest.setRequesterName(requestDTO.getRequesterName());
    paymentRequest.setRequesterPhone(requestDTO.getRequesterPhone());
    paymentRequest.setPaymentHandle(requestDTO.getPaymentHandle());
    paymentRequest.setPaymentInstructions(requestDTO.getPaymentInstructions());
    paymentRequest.setProofOfPaymentFileKey(requestDTO.getProofOfPaymentFileKey());
    paymentRequest.setProofOfPaymentUrl(requestDTO.getProofOfPaymentUrl());
    paymentRequest.setTenantId(requestDTO.getTenantId());
    paymentRequest.setCreatedAt(ZonedDateTime.now());
    paymentRequest.setUpdatedAt(ZonedDateTime.now());

    // Save payment request (without ticket_transaction_id yet)
    paymentRequest = manualPaymentRequestRepository.save(paymentRequest);

    // 4. Create EventTicketTransaction (PENDING status)
    EventTicketTransaction ticketTransaction = new EventTicketTransaction();
    ticketTransaction.setEventId(requestDTO.getEventId());
    ticketTransaction.setEmail(requestDTO.getRequesterEmail());
    ticketTransaction.setFirstName(extractFirstName(requestDTO.getRequesterName()));
    ticketTransaction.setLastName(extractLastName(requestDTO.getRequesterName()));
    ticketTransaction.setPhone(requestDTO.getRequesterPhone());
    ticketTransaction.setTotalAmount(requestDTO.getAmount());
    ticketTransaction.setFinalAmount(requestDTO.getAmount());
    ticketTransaction.setQuantity(cart.stream().mapToInt(CartItemDTO::getQuantity).sum());
    ticketTransaction.setStatus("PENDING"); // CRITICAL: PENDING, not CONFIRMED
    ticketTransaction.setTransactionReference("MANUAL-" + paymentRequest.getId());
    ticketTransaction.setPurchaseDate(ZonedDateTime.now());
    ticketTransaction.setTenantId(requestDTO.getTenantId());
    ticketTransaction.setCreatedAt(ZonedDateTime.now());
    ticketTransaction.setUpdatedAt(ZonedDateTime.now());
    // DO NOT set qrCodeImageUrl - only set after RECEIVED status

    ticketTransaction = eventTicketTransactionRepository.save(ticketTransaction);

    // 5. Create EventTicketTransactionItem records
    List<EventTicketTransactionItem> transactionItems = new ArrayList<>();
    for (CartItemDTO item : cart) {
        EventTicketType ticketType = ticketTypeRepository.findById(item.getTicketTypeId())
            .orElseThrow(() -> new EntityNotFoundException("Ticket type not found"));

        EventTicketTransactionItem transactionItem = new EventTicketTransactionItem();
        transactionItem.setTransactionId(ticketTransaction.getId());
        transactionItem.setTicketTypeId(item.getTicketTypeId());
        transactionItem.setQuantity(item.getQuantity());
        transactionItem.setPricePerUnit(ticketType.getPrice());
        transactionItem.setTotalAmount(ticketType.getPrice().multiply(BigDecimal.valueOf(item.getQuantity())));
        transactionItem.setTenantId(requestDTO.getTenantId());
        transactionItem.setCreatedAt(ZonedDateTime.now());
        transactionItem.setUpdatedAt(ZonedDateTime.now());

        transactionItems.add(transactionItem);
    }

    eventTicketTransactionItemRepository.saveAll(transactionItems);

    // 6. Link payment request to ticket transaction
    paymentRequest.setTicketTransactionId(ticketTransaction.getId());
    paymentRequest = manualPaymentRequestRepository.save(paymentRequest);

    // 7. Trigger confirmation email via batch job (async, non-blocking)
    triggerConfirmationEmailJob(paymentRequest.getId());

    // 8. Return DTO with ticket transaction ID
    ManualPaymentRequestDTO responseDTO = convertToDTO(paymentRequest);
    responseDTO.setTicketTransactionId(ticketTransaction.getId());
    return responseDTO;
}

2.4 PATCH /api/manual-payments/{id} (Enhance Existing)

Current Behavior: Updates manual payment request status only.
Required Enhancement: When status changes to RECEIVED, also update ticket transaction, generate QR code, and trigger ticket email.

Request Body Schema

Request Body (JSON Merge Patch) 
{
  "id": 123,

  "receivedBy": "admin@example.com",
  "receivedAt": "2026-01-14T15:30:00Z",
  "notes": "Payment confirmed via Zelle"

Implementation Steps

  1. Fetch and Validate:
    • Fetch ManualPaymentRequest by ID
    • Verify current status is REQUESTED (cannot change from RECEIVED/VOIDED)
    • Verify ticketTransactionId is not null
  2. If Status Changed to RECEIVED:
    • Update ManualPaymentRequest:
      • Set status = RECEIVED
      • Set received_at = current timestamp (or from request)
      • Set received_by = admin user identifier
      • Update updated_at
    • Fetch EventTicketTransaction by ticketTransactionId
    • Update ticket transaction:
      • Set status = CONFIRMED
      • Update updated_at
    • Update ticket inventory:
      • For each EventTicketTransactionItem:
      • Fetch corresponding EventTicketType
      • Increment quantity_sold by item quantity
      • Save ticket type
    • Generate QR code:
      • Call existing QR code generation service/endpoint
      • Endpoint: POST /api/events/{eventId}/transactions/{transactionId}/qrcode
      • Or use internal service method: qrCodeService.generateQrCode(eventId, transactionId, emailHostUrlPrefix)
      • Update EventTicketTransaction.qr_code_image_url with generated URL
    • Trigger ticket email batch job:
      • Create batch job task with type MANUAL_PAYMENT_TICKET_EMAIL
      • Include event ID, ticket transaction ID, requester email
      • Set job status to PENDING
  3. If Status Changed to VOIDED/CANCELLED:
    • Update ManualPaymentRequest status
    • Update EventTicketTransaction.status to CANCELLED
    • DO NOT update ticket inventory (tickets were never confirmed)
    • DO NOT generate QR code
    • DO NOT send ticket email
  4. Return updated ManualPaymentRequestDTO

Example Java Implementation Structure

ManualPaymentService.updateManualPaymentStatus() 
@Transactional
public ManualPaymentRequestDTO updateManualPaymentStatus(
    Long paymentRequestId,
    ManualPaymentStatus newStatus,
    String receivedBy,
    String voidReason
) {
    ManualPaymentRequest paymentRequest = manualPaymentRequestRepository
        .findById(paymentRequestId)
        .orElseThrow(() -> new EntityNotFoundException("Manual payment request not found"));

    // Validate status transition
    if (paymentRequest.getStatus() != ManualPaymentStatus.REQUESTED &&
        newStatus == ManualPaymentStatus.RECEIVED) {
        throw new BadRequestException("Cannot change status to RECEIVED from " + paymentRequest.getStatus());
    }

    ManualPaymentStatus oldStatus = paymentRequest.getStatus();
    paymentRequest.setStatus(newStatus);
    paymentRequest.setUpdatedAt(ZonedDateTime.now());

    if (newStatus == ManualPaymentStatus.RECEIVED) {
        // Update payment request
        paymentRequest.setReceivedAt(ZonedDateTime.now());
        paymentRequest.setReceivedBy(receivedBy);

        // Update ticket transaction to CONFIRMED
        if (paymentRequest.getTicketTransactionId() != null) {
            EventTicketTransaction ticketTransaction = eventTicketTransactionRepository
                .findById(paymentRequest.getTicketTransactionId())
                .orElseThrow(() -> new EntityNotFoundException("Ticket transaction not found"));

            ticketTransaction.setStatus("CONFIRMED");
            ticketTransaction.setUpdatedAt(ZonedDateTime.now());
            ticketTransaction = eventTicketTransactionRepository.save(ticketTransaction);

            // Update ticket inventory
            List<EventTicketTransactionItem> items = eventTicketTransactionItemRepository
                .findByTransactionId(ticketTransaction.getId());

            for (EventTicketTransactionItem item : items) {
                EventTicketType ticketType = ticketTypeRepository.findById(item.getTicketTypeId())
                    .orElseThrow(() -> new EntityNotFoundException("Ticket type not found"));

                ticketType.setQuantitySold(ticketType.getQuantitySold() + item.getQuantity());
                ticketTypeRepository.save(ticketType);
            }

            // Generate QR code
            String emailHostUrlPrefix = getEmailHostUrlPrefix(); // From config
            String qrCodeUrl = qrCodeService.generateQrCode(
                paymentRequest.getEventId(),
                ticketTransaction.getId(),
                emailHostUrlPrefix
            );

            ticketTransaction.setQrCodeImageUrl(qrCodeUrl);
            ticketTransaction = eventTicketTransactionRepository.save(ticketTransaction);

            // Trigger ticket email batch job
            triggerTicketEmailJob(
                paymentRequest.getEventId(),
                ticketTransaction.getId(),
                paymentRequest.getRequesterEmail()
            );
        }
    } else if (newStatus == ManualPaymentStatus.VOIDED ||
               newStatus == ManualPaymentStatus.CANCELLED) {
        paymentRequest.setVoidReason(voidReason);

        // Update ticket transaction to CANCELLED (but don't update inventory)
        if (paymentRequest.getTicketTransactionId() != null) {
            EventTicketTransaction ticketTransaction = eventTicketTransactionRepository
                .findById(paymentRequest.getTicketTransactionId())
                .orElseThrow(() -> new EntityNotFoundException("Ticket transaction not found"));

            ticketTransaction.setStatus("CANCELLED");
            ticketTransaction.setUpdatedAt(ZonedDateTime.now());
            eventTicketTransactionRepository.save(ticketTransaction);
        }
    }

    paymentRequest = manualPaymentRequestRepository.save(paymentRequest);
    return convertToDTO(paymentRequest);
}

4. Ticket Transaction Creation Logic

4.1 Transaction Reference Format

Manual payment ticket transactions must use a distinct reference format to differentiate from Stripe transactions:

  • Format: "MANUAL-{manualPaymentRequestId}"
  • Example: "MANUAL-123" for payment request ID 123
  • Purpose: Allows filtering and identification of manual payment transactions

4.2 Status Flow

Stage ManualPaymentRequest Status EventTicketTransaction Status QR Code Ticket Inventory
Request Created REQUESTED PENDING Not Generated Not Updated
Admin Confirms Payment RECEIVED CONFIRMED Generated Updated (quantity_sold incremented)
Admin Voids Request VOIDED CANCELLED Not Generated Not Updated

4.3 Inventory Management

CRITICAL: Ticket inventory (quantity_sold) must ONLY be updated when payment status changes to RECEIVED, NOT when the request is created. This prevents inventory from being held by unconfirmed payment requests.

Inventory Update Logic

  1. When payment request is created: DO NOT update quantity_sold
  2. When payment status changes to RECEIVED:
    • For each EventTicketTransactionItem:
    • Fetch corresponding EventTicketType
    • Increment quantity_sold by item quantity
    • Save ticket type
  3. When payment status changes to VOIDED/CANCELLED:
    • DO NOT decrement quantity_sold (was never incremented)

5. Status Management & QR Code Generation

5.1 QR Code Generation Trigger

QR codes are generated ONLY when payment status changes from REQUESTED to RECEIVED.

QR Code Generation Steps

  1. Verify ticket transaction status is CONFIRMED
  2. Call QR code generation service with:
    • eventId: From payment request
    • transactionId: Ticket transaction ID
    • emailHostUrlPrefix: From application configuration
  4. Store generated QR code URL in EventTicketTransaction.qr_code_image_url
  5. Save updated ticket transaction

5.2 Status Transition Validation

From Status To Status Allowed? Actions
REQUESTED RECEIVED Yes Update ticket to CONFIRMED, generate QR, update inventory, send ticket email
REQUESTED VOIDED Yes Update ticket to CANCELLED, no QR, no inventory update, no email
REQUESTED CANCELLED Yes Update ticket to CANCELLED, no QR, no inventory update, no email
RECEIVED VOIDED No Not allowed (payment already confirmed)
RECEIVED CANCELLED No Not allowed (payment already confirmed)

7. Error Handling & Validation

7.1 Validation Rules

Validation Error Code HTTP Status
Event not found or inactive EVENT_NOT_FOUND 404
Cart is empty CART_EMPTY 400
Ticket type not found TICKET_TYPE_NOT_FOUND 404
Insufficient ticket inventory INSUFFICIENT_INVENTORY 400
Amount mismatch (calculated vs provided) AMOUNT_MISMATCH 400
Invalid status transition INVALID_STATUS_TRANSITION 400
Tenant ID mismatch TENANT_MISMATCH 403

7.2 Transaction Rollback

All operations must be wrapped in database transactions. If any step fails, roll back all changes:

  • Payment request creation
  • Ticket transaction creation
  • Transaction item creation
  • Status updates
  • Inventory updates

7.3 Idempotency

The following operations must be idempotent (safe to retry):

  • Confirmation email sending endpoint
  • Ticket email sending (batch job should check if email already sent)
  • QR code generation (should return existing URL if already generated)
