GiveButter Donation Flow - Batch Job Implementation PRD
Spring Batch Jobs for Email and QR Code Processing

Critical Requirements

  • Email Sending: Batch job to send donation confirmation emails (triggered by API calls from frontend, similar to ticket email job)
  • QR Code Generation: Batch job to generate QR codes for donations (triggered by API calls from frontend)
  • Reconciliation: Batch job to reconcile GiveButter donations with database records (optional, for data consistency)
  • Statistics Aggregation: Batch job to aggregate donation statistics for admin dashboard
  • NO Webhook Triggering: Jobs are triggered by API endpoints called from frontend, not by webhook events

1. Executive Summary

This PRD outlines the batch job implementation for GiveButter donation flow. The batch jobs handle asynchronous processing tasks such as email sending, QR code generation, and data reconciliation, similar to existing ticket-related batch jobs.

Key Objectives

  • Send donation confirmation emails asynchronously (triggered by API calls from frontend)
  • Generate QR codes for donations in batch (triggered by API calls from frontend)
  • Reconcile GiveButter donations with database records (optional, for data consistency)
  • Aggregate donation statistics for admin dashboard
  • Handle failed email/QR code generation with retry logic
  • Follow existing pattern: Jobs triggered by API endpoints, not webhook events

3.2 Email Reader

// File: com.yourpackage.batch.donation.DonationEmailItemReader.java public class DonationEmailItemReader implements ItemReader<DonationTransaction> { private final DonationTransactionRepository repository; private Iterator<DonationTransaction> iterator; public DonationEmailItemReader(DonationTransactionRepository repository) { this.repository = repository; } @Override public DonationTransaction read() throws Exception { if (iterator == null) { // Find donations that need email sent: // - Status is COMPLETED // - Email is not null // - email_sent is false List<DonationTransaction> donations = repository .findByStatusAndEmailIsNotNullAndEmailSentFalse("COMPLETED"); iterator = donations.iterator(); } return iterator.hasNext() ? iterator.next() : null; } }

3.3 Email Writer

// File: com.yourpackage.batch.donation.writer.DonationEmailItemWriter.java public class DonationEmailItemWriter implements ItemWriter<DonationTransaction> { private final EmailService emailService; public DonationEmailItemWriter(EmailService emailService) { this.emailService = emailService; } @Override public void write(List<? extends DonationTransaction> donations) throws Exception { for (DonationTransaction donation : donations) { try { // Send donation confirmation email emailService.sendDonationConfirmationEmail( donation.getEventId(), donation.getId(), donation.getEmail() ); // Mark email as sent donation.setEmailSent(true); donation.setUpdatedAt(ZonedDateTime.now()); } catch (Exception e) { log.error("Failed to send email for donation: " + donation.getId(), e); throw e; // Re-throw to trigger retry } } } }

// File: com.yourpackage.batch.donation.processor.DonationQrCodeItemProcessor.java public class DonationQrCodeItemProcessor implements ItemProcessor<DonationTransaction, DonationTransaction> { private final QrCodeService qrCodeService; @Override public DonationTransaction process(DonationTransaction donation) throws Exception { try { String emailHostUrlPrefix = getEmailHostUrlPrefix(); String qrCodeUrl = qrCodeService.generateQrCode( donation.getEventId(), donation.getId(), emailHostUrlPrefix, "DONATION" ); donation.setQrCodeUrl(qrCodeUrl); donation.setQrCodeImageUrl(qrCodeUrl); return donation; } catch (Exception e) { log.error("Failed to generate QR code for donation: " + donation.getId(), e); throw e; } } }

5.1 Purpose

  • Fetch donations from GiveButter API

  • Update donation status if changed in GiveButter
  • Create missing donation records if found in GiveButter but not in database

6. Donation Statistics Job

6.1 Purpose

Aggregate donation statistics for admin dashboard:

  • Total donations count by event
  • Total donation amount by event
  • Average donation amount
  • Donations by date range
  • Top donors list

6.2 Statistics Aggregation

// File: com.yourpackage.batch.donation.DonationStatisticsJob.java @Configuration @EnableBatchProcessing public class DonationStatisticsJobConfig { @Bean public Job donationStatisticsJob() { return jobBuilderFactory.get("donationStatisticsJob") .incrementer(new RunIdIncrementer()) .flow(donationStatisticsStep()) .end() .build(); } @Bean public Step donationStatisticsStep() { return stepBuilderFactory.get("donationStatisticsStep") .tasklet(donationStatisticsTasklet()) .build(); } @Bean public Tasklet donationStatisticsTasklet() { return new DonationStatisticsTasklet(donationRepository, statisticsRepository); } } // File: com.yourpackage.batch.donation.DonationStatisticsTasklet.java public class DonationStatisticsTasklet implements Tasklet { private final DonationTransactionRepository donationRepository; private final DonationStatisticsRepository statisticsRepository; @Override public RepeatStatus execute(StepContribution contribution, ChunkContext chunkContext) { // Aggregate statistics by event List<EventDetails> events = eventRepository.findAll(); for (EventDetails event : events) { DonationStatistics stats = new DonationStatistics(); stats.setEventId(event.getId()); stats.setTenantId(event.getTenantId()); // Calculate statistics List<DonationTransaction> donations = donationRepository .findByEventIdAndStatus(event.getId(), "COMPLETED"); stats.setTotalDonations(donations.size()); stats.setTotalAmount(donations.stream() .map(DonationTransaction::getAmount) .reduce(BigDecimal.ZERO, BigDecimal::add)); stats.setAverageAmount(stats.getTotalAmount() .divide(BigDecimal.valueOf(stats.getTotalDonations()), 2, RoundingMode.HALF_UP)); // Save statistics statisticsRepository.save(stats); } return RepeatStatus.FINISHED; } }

7.2 Job Triggering via API Endpoints

Pattern: Following the existing checkout flow, batch jobs are triggered by API endpoints called from the frontend, not by scheduled jobs or webhooks. The frontend calls these endpoints after detecting donation success.
// File: com.yourpackage.controller.DonationJobController.java @RestController @RequestMapping("/api/proxy/events/{eventId}/donations/{transactionId}") public class DonationJobController { @Autowired private JobLauncher jobLauncher; @Autowired @Qualifier("donationEmailJob") private Job donationEmailJob; @Autowired @Qualifier("donationQrCodeJob") private Job donationQrCodeJob; // Trigger email job via API endpoint (called from frontend) @PostMapping("/send-email") public ResponseEntity<Map<String, Boolean>> triggerEmailJob( @PathVariable Long eventId, @PathVariable Long transactionId, @RequestParam String to) { try { JobParameters params = new JobParametersBuilder() .addLong("eventId", eventId) .addLong("transactionId", transactionId) .addString("email", to) .addLong("time", System.currentTimeMillis()) .toJobParameters(); jobLauncher.run(donationEmailJob, params); Map<String, Boolean> response = new HashMap<>(); response.put("emailSent", true); return ResponseEntity.ok(response); } catch (Exception e) { log.error("Error triggering email job", e); return ResponseEntity.status(500).build(); } } // Trigger QR code job via API endpoint (called from frontend) @PostMapping("/qrcode") public ResponseEntity<Map<String, String>> triggerQrCodeJob( @PathVariable Long eventId, @PathVariable Long transactionId) { try { JobParameters params = new JobParametersBuilder() .addLong("eventId", eventId) .addLong("transactionId", transactionId) .addLong("time", System.currentTimeMillis()) .toJobParameters(); jobLauncher.run(donationQrCodeJob, params); // Return QR code URL after job completes // Note: In practice, you may want to wait for job completion or return immediately Map<String, String> response = new HashMap<>(); response.put("qrCodeUrl", "pending"); // Job will update this return ResponseEntity.ok(response); } catch (Exception e) { log.error("Error triggering QR code job", e); return ResponseEntity.status(500).build(); } } }

11. Key Differences from Webhook-Based Approach

Jobs Triggered by API Calls, Not Webhooks

Following the existing checkout flow pattern, batch jobs are triggered by API endpoints called from the frontend, not by webhook events:

  • Email Job: Triggered by POST /api/proxy/events/{eventId}/donations/{transactionId}/send-email
  • QR Code Job: Triggered by POST /api/proxy/events/{eventId}/donations/{transactionId}/qrcode
  • Reconciliation Job: Optional scheduled job for data consistency
  • Statistics Job: Scheduled job for admin dashboard
