Executive Summary

This document provides detailed implementation instructions for batch jobs that send emails for manual payment requests. Two email types are required: (1) Confirmation email sent immediately after payment request creation, and (2) Ticket email sent after admin confirms payment receipt. All email sending must happen asynchronously via batch jobs triggered from the backend.

Key Requirement: All email sending must be done via batch jobs, not directly from backend services. The backend creates batch job records/tasks that are processed asynchronously by the batch job project.

1. Batch Job Types Required

Job Type Purpose When Triggered Email Type
MANUAL_PAYMENT_CONFIRMATION_EMAIL Send confirmation that payment request was received Immediately after payment request creation Confirmation (no QR code)
MANUAL_PAYMENT_TICKET_EMAIL Send ticket email with QR code after payment confirmed When admin marks payment as RECEIVED Ticket email (with QR code)
Important: These are separate from existing Stripe ticket email jobs. Create new job types specifically for manual payments to maintain separation of concerns.

2.3 Email Content Requirements

Subject Line

Example: Payment Request Received - Annual Gala - MANUAL-123

Email Body Structure

  1. Header Section:
    • Greeting: "Dear {recipientName},"
    • Event title and date
    • Confirmation message: "We have received your payment request."
  2. Payment Request Details:
    • Transaction Reference: MANUAL-123
    • Payment Method: Zelle/Venmo/Cash App/etc.
    • Total Amount Due: $150.00
    • Status: Pending Confirmation
  3. Ticket Summary:
    • Table listing each ticket type, quantity, price, and subtotal
    • Total amount
  4. Payment Instructions:
    • Payment method-specific instructions (Zelle handle, Venmo username, etc.)
    • Memo/note format: "Event {eventId} - {transactionReference}"
    • Amount to send

  5. Next Steps:
    • "Please complete your payment using the method you selected."
    • "Once your payment is confirmed by our team, you will receive your tickets via email."
    • "Your tickets will include a QR code for event entry."
  6. Important Notes:
    • "This is a payment request only. Your tickets will be issued after payment confirmation."
    • "If you have any questions, please contact us at [support email]."
  7. Footer:
    • Event organizer contact information
    • Event website link
    • Unsubscribe link (if applicable)

5. Trigger Points from Backend

5.1 Confirmation Email Trigger

When: Immediately after ManualPaymentRequest is created
Backend Endpoint: POST /api/manual-payments/{id}/send-confirmation-email
Backend Action: Create batch job record/task

Backend Job Creation Logic

Backend Service Method 
public void triggerConfirmationEmailJob(Long paymentRequestId) {
    ManualPaymentRequest request = manualPaymentRequestRepository
        .findById(paymentRequestId)
        .orElseThrow(() -> new EntityNotFoundException("Payment request not found"));

    // Check if email already sent (idempotency)
    if (request.getConfirmationEmailSent() != null && request.getConfirmationEmailSent()) {
        log.info("Confirmation email already sent for payment request {}", paymentRequestId);
        return;
    }

    // Create batch job task/record
    BatchJobTask emailJob = new BatchJobTask();
    emailJob.setJobType("MANUAL_PAYMENT_CONFIRMATION_EMAIL");
    emailJob.setStatus("PENDING");
    emailJob.setPaymentRequestId(paymentRequestId);
    emailJob.setEventId(request.getEventId());
    emailJob.setRecipientEmail(request.getRequesterEmail());
    emailJob.setTenantId(request.getTenantId());
    emailJob.setCreatedAt(ZonedDateTime.now());

    // Set job parameters as JSON
    Map<String, Object> jobParams = new HashMap<>();
    jobParams.put("paymentRequestId", paymentRequestId);
    jobParams.put("eventId", request.getEventId());
    jobParams.put("recipientEmail", request.getRequesterEmail());
    jobParams.put("recipientName", request.getRequesterName());
    jobParams.put("paymentMethod", getPaymentMethodName(request.getManualPaymentMethodType()));
    jobParams.put("amount", request.getAmount());
    jobParams.put("paymentHandle", request.getPaymentHandle());
    jobParams.put("paymentInstructions", request.getPaymentInstructions());
    // ... add more parameters

    emailJob.setJobParameters(objectMapper.writeValueAsString(jobParams));
    batchJobTaskRepository.save(emailJob);

    // Mark email as sent (prevent duplicates)
    request.setConfirmationEmailSent(true);
    manualPaymentRequestRepository.save(request);

    log.info("Confirmation email job created for payment request {}", paymentRequestId);
}

5.2 Ticket Email Trigger

When: When admin updates payment status to RECEIVED
Backend Endpoint: PATCH /api/manual-payments/{id}
Backend Action: Create batch job record/task after QR code is generated

Backend Job Creation Logic

Backend Service Method 
public void triggerTicketEmailJob(
    Long eventId,
    Long ticketTransactionId,
    String recipientEmail
) {
    EventTicketTransaction transaction = eventTicketTransactionRepository
        .findById(ticketTransactionId)
        .orElseThrow(() -> new EntityNotFoundException("Ticket transaction not found"));

    // Verify QR code exists
    if (transaction.getQrCodeImageUrl() == null || transaction.getQrCodeImageUrl().isEmpty()) {
        throw new IllegalStateException("QR code not generated for transaction " + ticketTransactionId);
    }

    // Check if email already sent (idempotency)
    if (transaction.getTicketEmailSent() != null && transaction.getTicketEmailSent()) {
        log.info("Ticket email already sent for transaction {}", ticketTransactionId);
        return;
    }

    // Create batch job task/record
    BatchJobTask emailJob = new BatchJobTask();
    emailJob.setJobType("MANUAL_PAYMENT_TICKET_EMAIL");
    emailJob.setStatus("PENDING");
    emailJob.setEventId(eventId);
    emailJob.setTicketTransactionId(ticketTransactionId);
    emailJob.setRecipientEmail(recipientEmail);
    emailJob.setTenantId(transaction.getTenantId());
    emailJob.setCreatedAt(ZonedDateTime.now());

    // Set job parameters as JSON
    Map<String, Object> jobParams = new HashMap<>();
    jobParams.put("eventId", eventId);
    jobParams.put("ticketTransactionId", ticketTransactionId);
    jobParams.put("recipientEmail", recipientEmail);
    jobParams.put("qrCodeImageUrl", transaction.getQrCodeImageUrl());
    // ... add more parameters

    emailJob.setJobParameters(objectMapper.writeValueAsString(jobParams));
    batchJobTaskRepository.save(emailJob);

    log.info("Ticket email job created for transaction {}", ticketTransactionId);
}

7. Error Handling & Retry Logic

7.1 Error Scenarios

Error Action Retry?
Payment request not found Log error, mark job as FAILED No
Event not found Log error, mark job as FAILED No
QR code URL not accessible Log warning, mark job as FAILED Yes (3 retries)
Email service unavailable Log error, mark job as FAILED Yes (5 retries with exponential backoff)
Template rendering error Log error, mark job as FAILED No

7.2 Retry Configuration

  • Max Retries: 3-5 attempts depending on error type
  • Retry Delay: Exponential backoff (1min, 5min, 15min)
  • Final Status: Mark as FAILED after max retries
  • Notification: Send alert to admin if job fails permanently

7.3 Idempotency

Both jobs must be idempotent:

  • Check if email was already sent before sending
  • Use database flags: confirmation_email_sent, ticket_email_sent
  • If email already sent, mark job as COMPLETED without sending duplicate
