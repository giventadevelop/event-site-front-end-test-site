2. Revenue Calculation Definitions

2.1 Gross Revenue

Definition: Sum of totalAmount (base ticket prices before discounts, fees, and taxes)

Formula: Gross Revenue = Σ(totalAmount)

Represents: Total sales value before any deductions

2.2 Total Revenue

Definition: Sum of finalAmount (what the customer actually paid, after discounts, may include taxes)

Formula: Total Revenue = Σ(finalAmount)

Represents: Total amount collected from customers

Note: This is the amount Stripe charged the customer. It may include taxes if Stripe Tax is enabled.

2.3 Net Revenue

Definition: What the event organizer receives after all deductions

Current Formula: Net Revenue = Gross Revenue - Discounts - Refunds - Platform Fees

Represents: Amount available to event organizer after platform takes its cut

⚠️ Important Note: Stripe Fees and Net Revenue

Stripe fees are NOT deducted in Net Revenue calculation because:

  • Stripe automatically deducts processing fees before depositing money to your account
  • The amount you receive in your Stripe account is already net of Stripe fees
  • Net Revenue represents what the event organizer receives from the platform, not what they receive from Stripe
  • Stripe fees are a separate cost that occurs at the payment processing level

However, for accounting and reporting purposes, tracking Stripe fees separately is valuable to understand total costs.

3.2 Why Stripe Fees Are Not Being Captured

Possible Reasons:

  1. Webhook Not Triggering: The charge.succeeded event may not be properly configured or received
  2. Timing Issue: The webhook may fire before the transaction record exists in the database
  3. Balance Transaction Missing: The charge may not have a balance_transaction ID available yet
  4. Webhook Processing Failure: Errors in webhook processing may be silently failing
  5. Test Mode vs Production: Stripe behavior differs between test and live modes

3.3 Tax Capture

Current State: Tax is not being captured from Stripe. The tax_amount field exists but is typically 0.00, and stripe_amount_tax is not being populated.

Stripe Tax Options:

  • Stripe Tax: If enabled, Stripe automatically calculates and collects tax based on customer location
  • Manual Tax: System calculates tax separately and includes it in the payment amount
  • No Tax: Tax is not collected (current implementation appears to use this)

  • Net Revenue represents the event organizer's share after platform fees

5.3 Batch Job Approach

Yes, a batch job can retrieve missing Stripe fees and tax data.

Location: E:\project_workspace\event-site-manager-batch-jobs

5.3.1 Batch Job Implementation Strategy

  1. Query Transactions: Find all transactions where stripe_fee_amount IS NULL and stripe_payment_intent_id IS NOT NULL
  2. Retrieve from Stripe: For each transaction, retrieve the PaymentIntent from Stripe
  3. Get Balance Transaction: Retrieve the charge's balance transaction to get the fee
  4. Get Tax Amount: If Stripe Tax is enabled, retrieve tax from PaymentIntent or CheckoutSession
  5. Update Database: Update transaction records with retrieved values
// Batch Job Pseudo-Code async function updateMissingStripeFees() { // 1. Query transactions missing Stripe fee data const transactions = await db.query(` SELECT id, stripe_payment_intent_id, stripe_checkout_session_id FROM event_ticket_transaction WHERE stripe_fee_amount IS NULL AND stripe_payment_intent_id IS NOT NULL AND status = 'COMPLETED' `); for (const txn of transactions) { try { // 2. Retrieve PaymentIntent from Stripe const pi = await stripe.paymentIntents.retrieve(txn.stripe_payment_intent_id); // 3. Get charge and balance transaction const charges = await stripe.charges.list({ payment_intent: txn.stripe_payment_intent_id, limit: 1 }); if (charges.data.length > 0) { const charge = charges.data[0]; if (charge.balance_transaction) { const balanceTx = await stripe.balanceTransactions.retrieve( charge.balance_transaction ); const stripeFeeAmount = balanceTx.fee / 100; // 4. Get tax amount (if available) let stripeAmountTax = null; if (pi.metadata?.tax_amount) { stripeAmountTax = parseFloat(pi.metadata.tax_amount); } else if (txn.stripe_checkout_session_id) { const session = await stripe.checkout.sessions.retrieve( txn.stripe_checkout_session_id ); stripeAmountTax = session.total_details?.amount_tax ? session.total_details.amount_tax / 100 : null; } // 5. Update database await db.update('event_ticket_transaction', { stripe_fee_amount: stripeFeeAmount, stripe_amount_tax: stripeAmountTax }, { id: txn.id }); } } } catch (error) { console.error(`Failed to update transaction ${txn.id}:`, error); } } }

5.3.2 Batch Job Considerations

  • Rate Limiting: Stripe API has rate limits; implement delays between requests
  • Error Handling: Some transactions may not have charges (refunded, cancelled)
  • Idempotency: Ensure batch job can be run multiple times safely
  • Scheduling: Run daily or weekly to catch any missed webhook updates
  • Historical Data: Batch job can backfill all historical transactions
