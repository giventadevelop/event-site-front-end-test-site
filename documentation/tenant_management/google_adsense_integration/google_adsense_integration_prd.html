Google AdSense — Tenant Settings Integration

Frontend repo: mosc-temp  |  Backend repo: event-site-manager-service  |  Schema baseline: Latest_Schema_Post__Blob_Claude_12.sql

Per-tenant Google AdSense is controlled by tenant_settings. Admin enables ads, sets ca-pub-..., and maps layout regions to AdSense ad unit slot IDs. Public satellite pages render ad regions only when enabled for that tenant.

1. Database (PostgreSQL)

Per database_schema_guidelines: add columns in the canonical CREATE TABLE public.tenant_settings block in Latest_Schema_Post__Blob_Claude_12.sql — do not use standalone ALTER TABLE scripts in the schema repo.

  enable_google_adsense boolean DEFAULT false NOT NULL,
  google_adsense_publisher_id character varying(32),
  google_adsense_placements_json text,

Existing deployments: Liquibase in event-site-manager-service (20260628120000_add_google_adsense_tenant_settings.xml) applies incremental addColumn changesets on startup — not duplicated as raw SQL ALTER files in mosc-temp or batch-jobs.

2. Backend API fields (OpenAPI / DTO)

JSON fieldValidation

Endpoints unchanged: GET/PATCH /api/tenant-settings via existing Next.js proxy.

3. Admin workflow

  1. Sign in as tenant admin → /admin/tenant-management/settings/{id}/edit
  2. Open Integrations tab
  3. Enable Google AdSense
  4. Enter Publisher ID (ca-pub-...)
  5. Paste placements JSON (see region map below)
  6. Save → optional homepage cache refresh (homepageCacheVersion)

To turn off ads for one satellite: uncheck Enable Google AdSense and save. Other tenants are unaffected.

5. Public frontend

  • TenantSettingsProvider — exposes showGoogleAdsense, adsensePublisherId, adsensePlacements

  • GoogleAdSenseRegion.tsx — gates by tenant settings + region slot mapping

6. ads.txt

Static template: public/ads.txt. Replace pub-0000000000000000 with your numeric publisher id before production.

google.com, pub-XXXXXXXXXXXXXXXX, DIRECT, f08c47fec0942fa0

Must be served at https://YOUR-SATELLITE-DOMAIN/ads.txt for each AdSense-approved domain.

7. Content Security Policy (when enforced)

This app does not set a global CSP header today. If you add CSP at CDN, reverse proxy, or next.config.mjs, allow at minimum:

  • script-src: https://pagead2.googlesyndication.com, https://www.googletagservices.com
  • frame-src / child-src: https://googleads.g.doubleclick.net, https://tpc.googlesyndication.com
  • img-src: https://pagead2.googlesyndication.com, https://googleads.g.doubleclick.net
  • connect-src: Google ad / analytics endpoints as required by AdSense diagnostics

8. Per-satellite rollout checklist

  1. Register satellite domain in Google AdSense; wait for approval
  2. Create ad units for each region; note slot IDs
  3. Update public/ads.txt (or domain-specific deploy) with correct pub-... line
  4. Admin: enable AdSense for tenant, set publisher ID + placements JSON
  5. Bump homepageCacheVersion if homepage settings are CDN-cached
  6. Verify homepage/footer regions render; toggle off and confirm regions disappear
  7. Review EU/UK consent requirements before enabling in regulated regions
