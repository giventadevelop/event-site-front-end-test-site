Promotion Email Management System - Frontend PRD

📋 Project Context: This project is already in active development phase. All infrastructure, basic development tools, database schema foundation, JSON schema definitions, and API endpoints are already in place. This PRD focuses exclusively on frontend feature implementation and does not include any infrastructure setup, basic tooling, API development, or foundational work.

1. Overview

This document outlines the frontend implementation requirements for the Promotion Email Management System. The frontend will provide a modern, user-friendly interface for creating, managing, and sending promotion email templates associated with events and discount codes.

2. Objectives

  • Create intuitive UI for template management (CRUD operations)
  • Implement event and discount code selection
  • Add image upload functionality for email headers/footers
  • Build email sending interface (test and bulk)
  • Create email history/audit trail viewer
  • Ensure responsive design and accessibility

4. API Proxy Routes

Task 2.1: Create Template List/Create Proxy

Priority: HIGH

File: src/pages/api/proxy/promotion-email-templates/index.ts

Description: Create proxy handler for GET (list) and POST (create) operations

Requirements:

  • Use createProxyHandler from @/lib/proxyHandler

  • Set backendPath: '/api/promotion-email-templates'
  • Support query parameters: eventId.equals, isActive.equals, sort, page, size
  • Follow cursor rules: no local DTO declarations, use fetchWithJwtRetry

Task 2.2: Create Template Get/Update/Delete Proxy

Priority: HIGH

File: src/pages/api/proxy/promotion-email-templates/[...slug].ts

Description: Create proxy handler for single template operations

Requirements:

  • Use createProxyHandler from @/lib/proxyHandler
  • Set backendPath: '/api/promotion-email-templates'
  • Support GET (by ID), PATCH (update), DELETE operations
  • Handle slug pattern: ['id'] or ['id', 'send-test'] or ['id', 'send-bulk']

Task 2.3: Create Email Sending Proxy Routes

Priority: HIGH

Files:

  • src/pages/api/proxy/promotion-email-templates/[id]/send-test.ts
  • src/pages/api/proxy/promotion-email-templates/[id]/send-bulk.ts

Description: Create proxy handlers for email sending endpoints

Requirements:

  • Use createProxyHandler or custom handler
  • POST method only
  • Forward request body to backend
  • Handle errors gracefully

Task 2.4: Create Email Log Proxy

Priority: MEDIUM

File: src/pages/api/proxy/promotion-email-sent-logs/index.ts

Description: Create proxy handler for email log queries

Requirements:

  • Use createProxyHandler
  • Set backendPath: '/api/promotion-email-sent-logs'
  • Support filtering: eventId.equals, templateId.equals, sentAt.greaterThanOrEqual, sentAt.lessThanOrEqual, emailStatus.equals
  • Support pagination: sort, page, size

5. Server Actions

Task 3.1: Create ApiServerActions.ts

Priority: HIGH

File: src/app/admin/promotion-emails/ApiServerActions.ts

Description: Create server actions for all API calls

Required Functions:

  • fetchPromotionEmailTemplatesServer(params) - List templates
  • fetchPromotionEmailTemplateServer(id) - Get template by ID
  • createPromotionEmailTemplateServer(formData) - Create template
  • updatePromotionEmailTemplateServer(id, formData) - Update template
  • deletePromotionEmailTemplateServer(id) - Delete template
  • sendTestEmailServer(templateId, recipientEmail) - Send test email
  • sendBulkEmailServer(templateId, recipientEmails?) - Send bulk email
  • fetchPromotionEmailSentLogsServer(params) - List email logs

Requirements:

  • All functions must use fetchWithJwtRetry from @/lib/proxyHandler
  • Use getAppUrl() for absolute URLs
  • Handle errors with try-catch
  • Return typed responses
  • Use withTenantId for DTOs

Accessibility Requirements:

  • Focus indicators

12. Cursor Rules Compliance

✅ Must Follow Cursor Rules:
  • REST API Rules: Use fetchWithJwtRetry from @/lib/proxyHandler
  • DTO Import: All DTOs imported from @/types, no local declarations
  • Tenant ID: Use withTenantId utility for all DTOs
  • Query Params: Use JHipster filter syntax (.equals, .contains)
  • Server Actions: All API calls in ApiServerActions.ts
  • Proxy Handlers: Use createProxyHandler for all proxy routes
  • Error Handling: Try-catch blocks with proper error messages
  • Client Components: No direct API calls, use server actions
