Auto Tenant ID — Frontend Enhancements PRD

Project: event-site-manager  |  Status: Implemented (v1)  |  Index: generic_enhancement_prd.html

Replace manual tenant ID entry on Create Organization with a prefix field plus auto-generated sequence. Full ID is persisted to tenant_organization.tenant_id on save.

1. Admin route

RouteAction
/admin/tenant-management/organizations/newCreate organization; auto-generate tenant ID
/admin/tenant-management/settings/newSelect existing org (no ID generation)

2. Files (implemented)

FileRole
src/lib/tenantIdGeneration.tsPure helpers: normalize prefix, parse suffix, next sequence, build ID
src/app/admin/tenant-management/organizations/tenantIdServerActions.tsServer actions: paginate orgs, preview next ID
src/app/admin/tenant-management/components/TenantOrganizationForm.tsxCreate UI: prefix + read-only sequence + preview; re-fetch on submit

3. UX flow

  1. Admin opens New Organization.
  2. Enters organization name; if prefix is empty, slug is suggested from the name.
  3. Admin edits prefix (e.g. ford_motors).
  4. Frontend calls previewNextTenantIdServer(prefix) (debounced 350ms).
  5. Read-only sequence and full generated ID are shown.
  6. On submit, preview is refreshed once more, then createTenantOrganization runs with final tenantId.
  7. Edit mode: tenant ID is read-only.
Server action — previewNextTenantIdServer
Input:  rawPrefix (string)
Steps:  1. normalizeTenantIdPrefix
        2. GET all tenant_organization (paginated via fetchTenantOrganizations)
        3. max trailing _NNNNN + 1 (or 1 if none)
        4. buildTenantId(prefix, sequence)
Output: { prefix, sequenceNumber, formattedSequence, tenantId }

4. Validation

  • Prefix: letters, numbers, underscores after normalization.
  • Full ID regex: /^[a-z0-9]+_\d{5,}$/
  • DB UNIQUE on tenant_id is the final guard for race conditions.

Concurrency: v1 computes next ID in the Next.js server action. Two simultaneous creates may collide; retry or adopt the optional backend endpoint in backend_enhancements_prd.html.
