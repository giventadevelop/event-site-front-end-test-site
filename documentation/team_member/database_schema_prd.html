Two new tables: team_groups (roster/squad metadata) and team_members (members with optional user_profile_id). Baseline columns mirror executive_committee_team_members.

1. Entity relationship

tenant_organization (1) ──< (N) team_groups
team_groups (1) ──< (N) team_members
user_profile (0..1) ──< (N) team_members  [optional user_profile_id]

4. tenant_settings migration

Flag split: Today show_team_members_section_in_home_page drives executive TeamSection. Migration adds show_executive_committee_section_in_home_page backfilled from the existing column so executive visibility is preserved. show_team_members_section_in_home_page then controls the new SquadRosterSection.
