Gallery Album Cover S3 — Frontend Enhancements PRD

Version: 1.0  |  Date: June 18, 2026

Replace manual “Cover Image URL” text entry with an S3 upload control on admin album pages, following FocusGroupCoverImageUpload. Keep “set cover from existing album photo” on the media management page.

1. New component

GalleryAlbumCoverImageUpload.tsx

Location: src/components/admin/gallery/GalleryAlbumCoverImageUpload.tsx (or src/components/GalleryAlbumCoverImageUpload.tsx)

Props:

  • albumId: number — required (album must exist before upload)
  • currentImageUrl?: string
  • onImageUploaded: (url: string) => void
  • onError: (message: string) => void
  • disabled?: boolean

Behavior: Clone FocusGroupCoverImageUpload.tsx — drag/drop, file picker, 10 MB limit, image/* validation, success toast, preview thumbnail.

Upload URL (browser → Next proxy):

/api/proxy/event-medias/upload/gallery-album-cover-image?albumId={id}&title=Gallery Album Cover Image&description=Cover image for gallery album&isPublic=true

On success: read response.fileUrl, call onImageUploaded(fileUrl). Backend already updated gallery_album.cover_image_url; parent form should sync local state.

4. UX requirements

  • Show current cover preview (16:9 or card aspect) with “Replace” affordance.
  • Loading spinner during upload; disable form submit while uploading.
  • On upload success, update form state coverImageUrl without requiring separate Save (backend already persisted) OR auto-save on upload — document chosen behavior in implementation.
  • Recommended dimensions hint: 800×600 or 1200×800 JPG/PNG (match static MOSC gallery art).
  • Error dialog for invalid type/size (reuse ErrorDialog from focus group component).

5. Create-album flow constraint

Upload requires albumId. For “create with cover in one step”, either: (A) create album first via POST then upload cover on returned id, or (B) defer cover upload to edit page only. Recommend (A) in create modal: after successful create, if user selected a file, immediately call upload endpoint before closing modal.

6. DTO / types

No change to GalleryAlbumDTO — coverImageUrl?: string already sufficient.

7. Acceptance tests (manual)

  1. Edit album → upload PNG → preview updates → DB cover_image_url is S3 HTTPS.
  2. Public /gallery shows new cover (hard refresh).
  3. Media page → “Set as cover” on existing photo → album cover updates.
  4. Upload invalid PDF → error message, no DB change.
  5. Album without cover → fallback to first album media or placeholder (existing public behavior).
