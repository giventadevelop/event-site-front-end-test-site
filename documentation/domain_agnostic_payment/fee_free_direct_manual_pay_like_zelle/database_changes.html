Database Changes: Fee-Free Manual Payments

This document proposes schema changes aligned to Latest_Schema_Post__Blob_Claude_12.sql.

1. New Table: manual_payment_request

Column Type Notes
id bigint Primary key
tenant_id varchar Tenant scope
event_id bigint FK to event_details
ticket_transaction_id bigint FK to event_ticket_transaction
amount_due numeric(21,2) Total amount due
payment_method_type varchar(50) ZELLE_MANUAL, VENMO_MANUAL, CASH_APP_MANUAL, CASH, CHECK, OTHER
payment_handle varchar(255) Organizer handle/email/phone/mailing
payment_instructions text Memo text and details
status varchar(30) Requested, Received, Voided/Refunded
proof_of_payment_file_key varchar(512) S3 key
proof_of_payment_file_url varchar(1024) Optional CDN URL
proof_of_payment_uploaded_at timestamp Upload timestamp
received_at timestamp Set on confirmation
received_by varchar(255) Admin/organizer identifier
void_reason text Cancel reason
created_at / updated_at timestamp Audit fields

2. Payment Provider Config Extension

  • Add provider_name values: VENMO_MANUAL, CASH_APP_MANUAL, CASH, CHECK, OTHER_MANUAL.
  • Ensure tenant-level enablement flags for manual methods.

3. Event Configuration

  • Event-level flags: payment_flow_mode and manual_payment_enabled (if missing).

4. Notes

  • No partial payment tracking in v1.
  • Pending requests are open-ended; cancellation is explicit.

5. Copyable DDL Snippets

Create manual_payment_request table 
CREATE TABLE manual_payment_request (
  id BIGINT PRIMARY KEY,
  tenant_id VARCHAR(255) NOT NULL,
  event_id BIGINT REFERENCES event_details(id) ON DELETE CASCADE,
  ticket_transaction_id BIGINT REFERENCES event_ticket_transaction(id) ON DELETE SET NULL,
  amount_due NUMERIC(21,2) NOT NULL,
  payment_method_type VARCHAR(50) NOT NULL,
  payment_handle VARCHAR(255),
  payment_instructions TEXT,
  status VARCHAR(30) NOT NULL DEFAULT 'REQUESTED',
  proof_of_payment_file_key VARCHAR(512),
  proof_of_payment_file_url VARCHAR(1024),
  proof_of_payment_uploaded_at TIMESTAMP,
  received_at TIMESTAMP,
  received_by VARCHAR(255),
  void_reason TEXT,
  created_at TIMESTAMP NOT NULL DEFAULT NOW(),
  updated_at TIMESTAMP NOT NULL DEFAULT NOW()
);
Create manual_payment_method_type enum 
CREATE TYPE public.manual_payment_method_type AS ENUM (
  'ZELLE_MANUAL',
  'VENMO_MANUAL',
  'CASH_APP_MANUAL',
  'CASH',
  'CHECK',
  'OTHER_MANUAL'
);
Event configuration fields 
ALTER TABLE event_details
  ADD COLUMN IF NOT EXISTS payment_flow_mode VARCHAR(30) DEFAULT 'STRIPE_ONLY',
  ADD COLUMN IF NOT EXISTS manual_payment_enabled BOOLEAN DEFAULT false;
