1. The Problem

Issue Description

When implementing horizontal scrolling for responsive tables on mobile and desktop views, we encountered the following issues:

  • Scrollbar track appears - The scrollbar track (the background bar) is visible with arrow buttons on both ends
  • Scrollbar thumb is missing - The draggable thumb (slider) that should appear on the track is not visible
  • Resize issues - When resizing the browser window horizontally, the thumb doesn't appear even when scrolling is needed
  • Mobile usability - Users cannot see or interact with the scrollbar thumb on touch devices

Context

This issue was encountered in the RegistrationManagementClient component at src/app/admin/events/registrations/RegistrationManagementClient.tsx, where a table with multiple columns needed to be horizontally scrollable on smaller screens while maintaining accessibility and usability.

2. What We Tried

Attempt 1: Basic Scrollbar Styling

/* Initial attempt - insufficient */ .table-scroll-container::-webkit-scrollbar { height: 12px; display: block; } .table-scroll-container::-webkit-scrollbar-thumb { background: #9CA3AF; border-radius: 6px; }

Result: ❌ Scrollbar thumb was not visible. The track appeared but the thumb was missing.

Attempt 2: Enhanced Styling with Borders

/* Second attempt - still insufficient */ .table-scroll-container::-webkit-scrollbar-thumb { background: #9CA3AF; border-radius: 7px; border: 2px solid #F3F4F6; min-height: 20px; }

Result: ❌ Thumb still not visible. Border helped but thumb dimensions were not properly set.

Attempt 3: Fixed Table Width

/* Table width approach */ <table style={{ minWidth: '900px', width: '900px' }}> {/* table content */} </table>

Result: ⚠️ Partial success. Table forced scrolling but thumb visibility was inconsistent, especially on resize.

Key Learnings from Failed Attempts

  • Simply setting scrollbar height and background color is not enough
  • The thumb needs explicit minimum dimensions to be visible
  • Table width must be dynamic (max-content) rather than fixed to ensure proper scrollbar behavior
  • Cross-browser compatibility requires multiple CSS properties and vendor prefixes
  • Container overflow properties must be explicitly set with !important flags

3. The Final Solution

Complete Working Implementation

The solution combines proper CSS scrollbar styling with correct container and table width settings to ensure the scrollbar thumb is always visible when scrolling is needed.

CSS Styling (Complete)

<style dangerouslySetInnerHTML={{ __html: ` .table-scroll-container { overflow-x: scroll !important; overflow-y: visible !important; scrollbar-width: thin !important; scrollbar-color: #9CA3AF #F3F4F6 !important; -ms-overflow-style: -ms-autohiding-scrollbar !important; } .table-scroll-container::-webkit-scrollbar { height: 16px !important; display: block !important; -webkit-appearance: none !important; appearance: none !important; } .table-scroll-container::-webkit-scrollbar-track { background: #F3F4F6 !important; border-radius: 8px !important; -webkit-box-shadow: inset 0 0 4px rgba(0,0,0,0.1) !important; box-shadow: inset 0 0 4px rgba(0,0,0,0.1) !important; } .table-scroll-container::-webkit-scrollbar-thumb { background: #9CA3AF !important; border-radius: 8px !important; border: 3px solid #F3F4F6 !important; -webkit-box-shadow: inset 0 0 4px rgba(0,0,0,0.3) !important; box-shadow: inset 0 0 4px rgba(0,0,0,0.3) !important; min-width: 30px !important; background-clip: padding-box !important; } .table-scroll-container::-webkit-scrollbar-thumb:hover { background: #6B7280 !important; border-color: #E5E7EB !important; } .table-scroll-container::-webkit-scrollbar-thumb:active { background: #4B5563 !important; border-color: #D1D5DB !important; } .table-scroll-container::-webkit-scrollbar-button { display: none !important; } .table-scroll-container::-webkit-scrollbar-corner { background: #F3F4F6 !important; } ` }} />

Container HTML Structure

<div className="w-full table-scroll-container" style={{ overflowX: 'scroll', overflowY: 'visible', WebkitOverflowScrolling: 'touch', maxWidth: '100%', display: 'block', position: 'relative', width: '100%', minHeight: '1px', scrollbarGutter: 'stable' }} > <table className="divide-y divide-gray-200" style={{ width: 'max-content', minWidth: '900px' }}> {/* table content */} </table> </div>

4. Implementation Details

Critical CSS Properties Explained

Property Value Purpose
scrollbar-width thin Firefox: Sets scrollbar width to thin (better visibility than auto)
scrollbar-color #9CA3AF #F3F4F6 Firefox: Sets thumb and track colors (thumb track)
::-webkit-scrollbar height 16px Chrome/Safari: Sets scrollbar height (larger = more visible)
::-webkit-scrollbar-thumb min-width 30px CRITICAL: Ensures thumb is always visible (minimum size)
::-webkit-scrollbar-thumb border 3px solid #F3F4F6 Creates contrast between thumb and track for visibility
background-clip padding-box Ensures border renders correctly around the thumb
-webkit-appearance: none none Removes default browser styling for custom appearance
scrollbarGutter stable Reserves space for scrollbar to prevent layout shift

Table Width Strategy

Key Insight: The table width is critical for scrollbar thumb visibility.

❌ Wrong Approach:

<table style={{ minWidth: '900px', width: '900px' }}>

Fixed width prevents proper scrollbar calculation and thumb visibility on resize.

✅ Correct Approach:

<table style={{ width: 'max-content', minWidth: '900px' }}>

Dynamic width ensures table expands naturally while forcing scroll when container is smaller, which triggers proper thumb rendering.

5. General Guidelines

When to Use Horizontal Scrollbars

  • Responsive tables with many columns that need to remain accessible on mobile
  • Data grids where column visibility is important
  • Content containers that exceed viewport width on smaller screens
  • Admin panels with complex data tables

Implementation Checklist

  • Use a dedicated CSS class for the scroll container (e.g., table-scroll-container)
  • Set overflow-x: scroll (not auto) to always show scrollbar
  • Set scrollbar height to at least 16px for visibility
  • Set thumb min-width: 30px to ensure it's always visible
  • Add border to thumb for contrast (3px recommended)
  • Use width: 'max-content' for table instead of fixed width
  • Include hover and active states for better UX
  • Hide scrollbar buttons with display: none
  • Use !important flags to prevent style overrides
  • Test on multiple browsers (Chrome, Firefox, Safari, Edge)
  • Test on mobile devices and during window resize

Color Scheme Recommendations

Element Color Hex Code Usage
Track Background Light Gray #F3F4F6 Scrollbar track background
Thumb Background Medium Gray #9CA3AF Default thumb color
Thumb Hover Dark Gray #6B7280 Hover state for better feedback
Thumb Active Darker Gray #4B5563 Active/dragging state
Thumb Border Light Gray #F3F4F6 Border for contrast and visibility

6. Best Practices

Accessibility

  • Ensure scrollbar thumb has sufficient contrast (WCAG AA: 4.5:1 ratio)
  • Provide keyboard navigation alternatives (arrow keys, Page Up/Down)
  • Test with screen readers to ensure content is accessible
  • Maintain focus indicators for keyboard users

Performance

  • Use CSS transforms for smooth scrolling: WebkitOverflowScrolling: 'touch'
  • Avoid JavaScript-based scrollbar implementations when CSS is sufficient
  • Use scrollbarGutter: 'stable' to prevent layout shifts

Cross-Browser Compatibility

  • Chrome/Safari: Use ::-webkit-scrollbar pseudo-elements
  • Firefox: Use scrollbar-width and scrollbar-color
  • Edge: Supports both WebKit and Firefox properties
  • IE11: Use -ms-overflow-style (limited support)

Mobile Considerations

  • Ensure thumb is large enough for touch interaction (minimum 30px width)
  • Use WebkitOverflowScrolling: 'touch' for momentum scrolling on iOS
  • Test on actual devices, not just browser dev tools
  • Consider swipe gestures as alternative navigation method

7. Troubleshooting

Common Issues and Solutions

Issue: Scrollbar thumb not visible

Symptoms: Track appears but thumb is missing

Solutions:

  • Check that min-width: 30px is set on thumb
  • Verify table width is max-content not fixed
  • Ensure overflow-x: scroll (not auto)
  • Check that !important flags are present

Issue: Thumb disappears on resize

Symptoms: Thumb visible initially but disappears when window is resized

Solutions:

  • Use width: 'max-content' for table
  • Ensure container has scrollbarGutter: 'stable'
  • Check that container width is 100% or max-width: 100%

Issue: Thumb too small to interact with

Symptoms: Thumb is visible but difficult to click/drag

Solutions:

  • Increase min-width to at least 30px
  • Increase scrollbar height to 16px or more
  • Add padding to thumb area for easier touch targets

Issue: Inconsistent appearance across browsers

Symptoms: Works in Chrome but not Firefox, or vice versa

Solutions:

  • Include both WebKit and Firefox scrollbar properties
  • Test in all target browsers during development
  • Use browser-specific prefixes where needed

Debugging Steps

  1. Open browser DevTools and inspect the scroll container
  2. Check computed styles for overflow-x and overflow-y
  3. Verify scrollbar pseudo-elements are being applied (Chrome: Elements → Styles → scrollbar)
  4. Check table width in computed styles (should be max-content or actual pixel value)
  5. Test with different container widths to trigger scrolling
  6. Verify !important flags are not being overridden

Summary

The key to a visible horizontal scrollbar thumb:

  1. Set minimum thumb dimensions (min-width: 30px)
  2. Use dynamic table width (width: 'max-content')
  3. Apply comprehensive CSS (WebKit + Firefox properties)
  4. Use !important flags to prevent overrides
  5. Test on resize to ensure thumb appears when needed
