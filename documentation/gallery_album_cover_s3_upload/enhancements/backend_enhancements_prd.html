Add a dedicated multipart upload endpoint that stores the cover image in S3, persists an event_media record (optional but recommended), and updates gallery_album.cover_image_url with the returned S3 URL.

1. New REST endpoint

POST /api/event-medias/upload/gallery-album-cover-image
ParameterRequiredDescription
fileYesMultipart image (JPEG, PNG, WebP, GIF)
albumIdYesgallery_album.id
tenantIdYesTenant scope; must match album row
titleNoDefault: Gallery Album Cover Image
descriptionNoDefault: Cover image for gallery album
isPublicNoDefault: true

Content-Type: multipart/form-data

Auth: Bearer JWT (service or admin user — same as other upload endpoints)

2. Processing flow

  1. Validate albumId exists and tenantId matches album.
  2. Validate file: image MIME, max size 10 MB (align with focus group / frontend).
  3. Generate S3 key via new S3Service.generateGalleryAlbumCoverPath(...).
  4. Upload to bucket; obtain public/stable fileUrl.
  5. Create EventMedia row:
    • albumId = album id
    • eventId = null or 0 (non-event gallery)
    • eventMediaType = GALLERY or GALLERY_COVER (pick one enum; document in OpenAPI)
    • storageType = S3
    • isPublic = true
    • priorityRanking = 0 (cover first in sort if shown in grid)
  6. PATCH GalleryAlbum.coverImageUrl = fileUrl; update updatedAt.
  7. Return EventMediaDTO (include fileUrl for frontend).

3. S3 path convention

{profilePrefix}/media/tenantId/{tenantId}/gallery-album/album-id/{albumId}/cover_{timestamp}_{uuid}.{ext}

Example:

dev/media/tenantId/tenant_demo_002/gallery-album/album-id/10974/cover_1739900000000_a1b2c3d4.jpg

Mirror focus group pattern for consistency:

dev/media/tenantId/tenant_demo_002/focus-groups/focus-group-id/4002/cover_image_1772540438301_f8b17ade.jpg
