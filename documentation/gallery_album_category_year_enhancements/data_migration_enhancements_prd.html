3. Sample album backfill (static → DB mapping)

Extract from src/app/mosc-redesign/(syro)/gallery/page.tsx. Example UPDATE after albums exist in DB:

-- Example: set year + category on album matched by title
UPDATE public.gallery_album ga
SET album_year = 2019,
    gallery_category_id = (
        SELECT id FROM public.gallery_category gc
        WHERE gc.tenant_id = ga.tenant_id
          AND gc.slug = 'ecumenical-visits'
    ),
    updated_at = now()
WHERE ga.tenant_id = 'tenant_mosc_xxx'
  AND ga.title ILIKE '%Russia visit%';

5. Optional import script approach (M4)

  1. Parse static manifest JSON (generated from gallery page)
  2. POST /api/gallery-albums with title, coverImageUrl, albumYear, galleryCategoryId, isPublic, displayOrder
  3. Upload or link event_media rows with album_id for each photo path under public/images/mosc/gallery/
  4. Store static id → db id map in JSON for href resolution in frontend
Media import scope: Full photo import is large (60+ files per album). v1.0 may import **album metadata only** (cover + count from static) and attach media incrementally via existing admin media UI.

7. Rollback

Implemented: migrations/rollback_gallery_category_v1.sql

  • Drop FK and columns: ALTER TABLE gallery_album DROP COLUMN gallery_category_id, DROP COLUMN album_year;
  • Drop table: DROP TABLE gallery_category;
  • Frontend falls back to pre-v1.0 card (no pill/year)

8. Verification queries

-- Categories per tenant
SELECT slug, display_name, sort_order FROM gallery_category
 WHERE tenant_id = :tenant AND is_active = true ORDER BY sort_order;

-- Albums missing year or category (public)
SELECT id, title, album_year, gallery_category_id FROM gallery_album
 WHERE tenant_id = :tenant AND is_public = true
   AND (album_year IS NULL OR gallery_category_id IS NULL);

-- Album with category join
SELECT ga.title, ga.album_year, gc.display_name, gc.slug
FROM gallery_album ga
LEFT JOIN gallery_category gc ON gc.id = ga.gallery_category_id
WHERE ga.tenant_id = :tenant AND ga.is_public = true;
